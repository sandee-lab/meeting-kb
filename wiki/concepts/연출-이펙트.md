# 연출 & 이펙트

> 출처: [[raw/gdd/05_UX/유닛_승급_강화_연출_기획서]], [[raw/gdd/05_UX/전투_시작_연출]] *(대체됨)*, [[raw/gdd/05_UX/콘텐츠_언락_연출]] *(대체됨)*, [[raw/gdd/05_UX/폴리싱_결과창_MVP유닛_기획서]], [[raw/gdd/05_UX/시스템_메시지_규격]] *(대체됨)*

## 개요

인게임·아웃게임 각종 연출 시퀀스 설계를 정리한 문서이다. 전투 시작 시 비행선·지휘관 드롭 연출, 신규 콘텐츠 언락 연출, 유닛 특별 강화 달성 시 별도 팝업 연출, 결과창 MVP 유닛 표시 연출 등을 포함한다. 연출 중복 발생 시 출력 우선순위와 순차 처리 규칙도 함께 정의한다.

## 상세 내용

### 유닛 특별 강화 연출 (v1.1, 2026.03, 개발 중)

> 출처: [[raw/gdd/05_UX/유닛_승급_강화_연출_기획서]] — 작성: 김나은

특별 강화 등급(3/6/9/12강화) 달성 시 차별화된 피드백으로 성취감 강화 및 해금 효과를 명확히 인지시키는 시스템.

**핵심 설계 원칙:**
- 기존 일반 강화 흐름(`UnitEnchantUpdateSend` → `UpdateUnitData` → 결과 UI) 변경 금지
- 특별 강화 안내를 SystemMessageUI와 독립된 **별도 팝업**으로 구현 (UI 위치 겹침 방지)
- 연출 중 입력 차단(모달) 불가
- 연출은 짧게 (별도 스킵 불필요 수준)
- `special_enchant_type`이 Stat이든 Add_Tactic이든 동일 연출로 처리

**전체 흐름:**
```
강화 버튼 탭
→ WAS: UnitEnchantUpdateSend
  → 서버 응답 → UserDataManager.UpdateUnitData
    → 기존 강화 결과 UI 갱신
      → enchant_type == Special 확인
        ├─ [일반] → SystemMessageUI.AddTotalPower() 즉시 실행
        └─ [특별] → 특별 강화 안내 팝업 재생 (비모달, 자동 등장·퇴장)
                     → (팝업 완료 콜백)
                       → SystemMessageUI.AddTotalPower() 실행
```

**특별 강화 안내 팝업 내용:**
- 타이틀 텍스트 (예: "강화 효과 해제!")
- 해금 효과: `UnitEnchantInfoSlot`과 동일 포맷
  - `Stat` 타입: `"{스탯명} + {수치}"` (예: "공격력 + 15%")
  - `Add_Tactic` 타입: 전술 아이콘 + 전술 카드명
- VFX(기존 에셋 활용, 이펙터 연결 필요), SFX 재생

**우선순위:**

| 우선순위 | 항목 |
|---------|------|
| P0 (필수) | 특별 강화 판별 후 별도 팝업 안내 연출 재생 |
| P0 (필수) | 해금된 특수 효과 내용 텍스트 표시 |
| P1 (중요) | 전용 VFX 및 SFX |
| P2 (선택) | 강화 슬롯 UI에 특별 등급 강조 표시 |

**UX 주의사항:**
- 연속 강화 시 이전 팝업이 남아있으면 Sequence Kill 후 새로 재생
- 일반 강화 시에는 팝업 미표시 (SetActive false 유지)

### 전투 시작 연출 (M7, 대체됨)

> 출처: [[raw/gdd/05_UX/전투_시작_연출]]

비행선 → 지휘관·상자 드롭 → 상자 개봉 → 택틱 플라이 → 'Survive' 텍스트 시퀀스.

**시퀀스:**
1. [Play] 버튼 터치 → 로딩 화면 (챕터#, 챕터 제목, 비행선 이미지)
2. 로딩 종료 시 비행선 형태 활용하여 맵으로 트랜지션
3. 맵 출력 → 비행선에서 지휘관과 상자 드롭
4. 상자 개봉 → 게임 시작(몬스터 스폰 시작)
5. 상자에서 지휘관 보유 택틱 드롭
   - tactic_1~4: 택틱 아이콘이 일시정지 아이콘 방향으로 플라이
   - tactic_5(시작 유닛): 드롭 자리에서 버블이 터지는 연출
6. 상단 메뉴 출력 (위에서 내려오는 연출)
7. 'Survive' 문구 출력

**예외:** Wave 1 시작 시 기존 웨이브 시작 연출 생략

**필요 리소스:** 로딩 화면, 로딩→인게임 트랜지션 연출, 지휘관 모션, 버블 터지는 연출, Wave1 시작 텍스트 연출

### 콘텐츠 언락 연출 (M7, 대체됨)

> 출처: [[raw/gdd/05_UX/콘텐츠_언락_연출]]

신규 콘텐츠 해제 시 순차 연출 처리 설계.

**출력 시점:** 조건 만족 후 최초 로비 복귀 시

**연출 시퀀스:**
1. 딤드 화면 출력
2. 중앙에 흑백 아이콘 + 자물쇠 아이콘 출력 + '탭하여 닫기' (터치 시 연출 종료)
3. 자물쇠 흔들거리다 풀어지는 연출
4. 아이콘 컬러 전환 (스케일 효과 + 반짝임)
5. 타이틀 "새 콘텐츠 해제" 출력
6. 콘텐츠 이름 출력 (스케일 효과)
7. 아이콘 지정 위치로 플라이 → 목표 지점 강조

**데이터 규칙:**
- `ContentsUnlockData@ui_popup=true`인 경우에만 연출 적용
- 동시 해제 시 `ContentsUnlockData@order` 순서로 순차 출력
- `show_popup_on_unlock=true`: 연출 종료 후 해당 UI 강제 오픈 (뒤에 대기 연출 있으면 UI 종료 후 다음 연출)

**아이콘 표시 여부:**
- 잠금 상태 아이콘 출력: 하단 메뉴(상점/유닛/장비/골드 성장), 햄버거 메뉴
- 아이콘 미출력: 메인 화면 양쪽 배치 콘텐츠(출석부, 퀘스트, 우편함, 친구 등)
- 2뎁스 이상 콘텐츠: 뎁스 최상위 아이콘으로 연출 진행 + 해당 콘텐츠 아이콘에 [N]마크
- 인게임 콘텐츠: 아웃게임에서 진행하지 않음 (아머리 진입 시 진행)

### 시스템 메시지 출력 우선순위 (M6, 대체됨)

> 출처: [[raw/gdd/05_UX/시스템_메시지_규격]]

연출이 겹칠 경우 아래 순서로 출력:
1. 레벨업
2. 콘텐츠 언락
3. 시스템 메시지

**인게임 보상 연출 (몬스터 킬 달성):**
1. 화면 상단에서 내려오는 메시지 창 출력
2. 인게임에 오브젝트 드롭
3. 메시지 창 점멸 시점에 드롭 오브젝트가 인벤토리로 플라이
4. 인벤토리 아이콘 점멸
- 시스템 메시지와 동시 발생 시 동시 출력

**룰렛 보상 연출:**
1. 룰렛 연출
2. 결과 출력 후 인게임에 오브젝트 드롭
3. 오브젝트 지정 위치로 플라이 연출
