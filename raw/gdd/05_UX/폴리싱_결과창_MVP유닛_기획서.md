# 폴리싱 결과창 개선 (MVP유닛 & 신규광고)

> **요약:** 챕터 결과창에 누적 피해량 1위 유닛을 MVP로 표시하고, 추가 보상(x2) 획득을 위한 광고 시청 구조를 추가하는 폴리싱 기획이다. MVP는 클리어 시에만 노출되며, 2배 보상은 광고 제거 구매 여부에 따라 광고 시청 또는 즉시 적용으로 분기한다. 일일 5회 제한이며, 순찰 보상 정렬 및 부스트 팩 표기 수정도 포함한다.
> **현재 상태:** 미적용

탭 1

**M8\_폴리싱\_결과창개선(신규광고&MVP유닛)**

# 1. 개요

* 피드백
  + 상위 유닛이 강하다는 체감이 어려움
  + 광고 시청 창구 확대 필요
* 피드백 대응을 위해 챕터 결과창에 **[MVP 유닛 표시]** 및 **[추가 보상(x2) 획득 구조]**를 반영한다.

# 2. MVP 유닛 표시

## 노출 조건

* + 챕터 돌파 클리어(최종 웨이브 성공) 시

## 선정 규칙

* + 해당 챕터 누적 피해량 1위 유닛
    - 동률 발생 시: 누적 피해량 → 초당 피해량 높은 순

## 표시 방식

* + 아이콘: UnitCostumeData@icon 컬럼
  + 위치: 결과창 상단
  + 연출: (상의)

## 예외 처리

* + 튜토리얼/연출 전투와 같이 피해량 데이터가 유효하지 않은 경우에는 MVP 유닛을 표시하지 않는다.
    - 챕터0 해당

## UI 시안

* + 텍스트
    - MVP : local\_stage\_result\_mvp\_unit

# 3. 추가 보상 (x2) 획득 구조

## 노출 조건

* + 챕터 돌파 모드 정상 종료 (패배 및 승리 무관)
    - 훈련 모드는 적용하지 않는다.
    - 비정상 종료 시에는 적용하지 않는다.

## 적용 대상

* + 챕터 종료 시 획득한 반복 보상
    - 인게임에서 획득한 골드가 포함된다.
    - 직전 결과창에서 결정된 반복 보상 수량을 기준으로 한다.

## 동작

* + 광고 제거 미구매 유저
    - 챕터 종료
    - 결과창 노출
    - 보상 버튼 선택
    - 광고 시청
    - 광고 종료
    - 반복 보상 1회 추가 지급 / UI 갱신
  + 광고 제거 구매 유저
    - 챕터 종료
    - 결과창 노출
    - 보상 버튼 선택
    - 반복 보상 1회 추가 지급 / UI 갱신

## 적용 타이밍

* + 추가 보상
    - 광고 제거 미구매
      * 광고 시청 완료 시 반복 보상 한 번 더 지급
      * 광고 시청 후 결과창 복귀 시 UI 갱신
        + 직전 반복 보상 지급량에 배수(x2)를 적용하여 다시 숫자 찍어주는 연출
    - 광고 제거 구매
      * 2배 보상 받기 선택 시 반복 보상 한 번 더 지급
      * 결과창 UI 즉시 갱신
        + 직전 반복 보상 지급량에 배수(x2)를 적용하여 다시 숫자 찍어주는 연출
  + 예외
    - 광고 시청 실패 또는 취소 시에는 적용되지 않는다.
  + 서버 작업 필요
    - 현재 초회 보상과 반복 보상을 구분하지 않고 내려줌
    - 초회 보상과 반복 보상 구분 작업 필요요

## 처리

* + 데이터 처리
    - AdsSlotData
      * ads\_type
        + Stage\_Result
      * reset\_type
        + Daily
      * max\_count
        + 5
      * ready\_type
        + Battle\_End (챕터 기준)
      * ready\_type\_value
        + 1 (1챕터 당 1회)
      * ads\_value
        + 2 (보상 배수)

## 표시 연출

* + 아래 2가지 방식 중 1개를 택합니다.
    - 광고 시청 후 보상 수량을 2배로 갱신하여 재표시 (cf. 메카 어셈블) **[추천]**
      * 보상 수량을 명확히 전달할 수 있어, 전달력이 가장 높다고 판단됩니다.
    - 보상 아이콘에 'x2' 스티커 오버레이 (이미지 참조)
