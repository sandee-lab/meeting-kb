# 이벤트 고도화 / 재화 경쟁 이벤트

> **요약:** 이벤트 계층 구조(이벤트 홈 + 하위 이벤트)를 도입하여 운영 복잡도를 줄이고 유저 이해도를 높이는 기획이다. EventHubEntryData, EventHubData 신규 테이블을 추가하고, 재화 소모 경쟁 이벤트(ResourceRank) 시스템을 신규 도입한다. 프리뷰-본편-교환의 시리즈 구조, 채널 랭킹, 영구 보상(랭킹 코인), 마일스톤 보상, 교환 상점 등 다층적 이벤트 운영 체계를 정의한다.
> **현재 상태:** 미적용

🎪 이벤트

**M8\_이벤트 고도화 / 재화 경쟁 이벤트**

# 개요

* 이벤트 계층 구조를 구현한다.
* 재화 경쟁 이벤트를 추가하여, 유저가 돈을 쓰면서도 대접받고 있다는 느낌을 받게 하자.
  + 사전 집계 기반 재화 소모 경쟁 이벤트로 유저의 참여를 유도하고, 영구 재화를 통해 반복 참여와 매출을 동시에 확보한다.

# 이벤트 계층 구조

## 계층 구조?

### 이벤트 간 계층을 설정한다.

* + - 상위
      * 여러 이벤트를 담는 컨테이너 역할
      * 유저가 처음 진입하는 이벤트 묶음의 진입점
      * 이하 [이벤트 홈]으로 칭합니다 (신규 개념)
    - 하위
      * 컨테이너에 담기는 개별 이벤트

### 예시

* + - 서버 오픈 3주 페스타 ◀ 이벤트 홈
      * 이벤트 출석부 ◀ 하위 이벤트
      * 이벤트 미션 ◀ 하위 이벤트
      * … ◀ 하위 이벤트

### 운영 관점

* + - 현재의 한계점
      * 이벤트 하나 끄면 관련 이벤트가 살아있음 > 유령 이벤트 발생
      * 실수는 곧 CS + 신뢰 하락
    - 계층 구조 적용 후 기대 효과
      * 이벤트 홈 하나만 보면 전체 상태가 보임
      * 이벤트 홈 종료 → 하위 이벤트 자동 정리 가능
      * 운영 체크 포인트가 N개에서 1개로 줄어듦

### 유저 이해도 향상

* + - 현재의 한계점
      * 이벤트 마다 전용 진입 아이콘 배치 > 핵심 이벤트가 묻힘
      * 이벤트 간 관계 인식 불가 > 유저 입장에서 “이게 무슨 이벤트지?”
    - 계층 구조 적용 후 기대 효과
      * 이벤트 홈에서 관련 이벤트를 한 번에 확인 가능

### 기획 의도 전달

* + - 현재의 한계점
      * 이벤트는 전부 독립된 기능 > 실제 게임에서는 단절됨
    - 계층 구조 적용 후 기대 효과
      * 이벤트 홈이 컨텍스트를 제공 > “이 이벤트는 이 페스타의 일부”라는 인식 형성
      * 기획 의도가 UI에 자연스럽게 반영

### 개발 유지보수 비용 절감

* + - 현재의 한계점
      * 이벤트 늘어날수록 예외 처리 증가
    - 계층 구조 적용 후 기대 효과
      * 판단 주체가 이벤트 → 이벤트 홈

* + - entry\_id : 고유 아이디

      * MainIcon : 메인 화면 독립 아이콘

    - icon\_name (MainIcon인 경우)
    - is\_visible : 관리용

### EventHubData

* + - 방향
      * EventManagerData 구조와 최대한 비슷하게 구성한다.
    - 구조
      * hub\_id : 고유 아이디
      * order : 순서
      * event\_scope\_type
        + Global : 시작일시~종료일시 진행
        + Personal : 계정생성 n일차 기준 운영
        + Channel : 채널오픈 n일차 기준 운영
      * event\_duration\_type
        + fixed\_range : 고정 기간
        + access\_triggered : 기간 중 접속 시 [period]동안
      * start\_date : 시작일 [event\_scope\_type=Global 전용]
      * end\_date : 종료일 [event\_scope\_type=Global 전용]
      * start\_day\_offset : 접속 인정 기간 시작일(n일차)
        + [event\_scope\_type=Personal OR Channel 전용]
      * end\_day\_offset : 접속 인정 기간 종료일(n일차)
        + [event\_scope\_type=Personal OR Channel 전용]
      * period
      * unlock\_condition\_type
      * unlock\_condition\_value1
      * unlock\_condition\_value2
        + 이벤트 접근 조건 추가 (챕터 클리어, 계정 레벨 달성)
      * local\_name
      * local\_desc
      * hub\_type
        + 홈 구조를 n가지로 나누어 타입으로 구분한다.
        + 라이브 중 타입이 추가될 가능성이 있다.

# 재화 소모 경쟁 이벤트 시스템 [신규 이벤트]

## 특징

### 사전 집계 지원

* + - 사전 예고 기간부터 소모량을 누적하여, 이벤트 시작 전에 재화를 사용한 유저의 불만을 방지한다.

### 다중 이벤트 동시 운영

* + - 세팅에 따라 서로 다른 재화 이벤트가 동시에 활성화 될 수 있다.

### 채널 랭킹 및 명예 보상

* + - 각 이벤트 별로 해당 재화 누적 소모량 채널 랭킹을 산정해 UI 에 표기한다.
    - 랭킹 구간에 따라 명예 보상 (프로필 스티커 혹은 배경 + 랭킹 코인) 을 지급하여, 참여를 유도한다.

### 확정 보상

* + - 각 이벤트 별로 마일스톤 및 마일스톤 보상 (일반 재화) 설정하여, 참여를 유도한다.

### 영구 보상

* + - 랭킹 구간에 따라 공용 보상 [랭킹 코인]을 지급한다.
    - [랭킹 코인]은 이벤트가 종료된 후에도 사라지지 않고 누적된다.
    - [랭킹 코인]은 언제나 교환상점에서 아이템으로 교환할 수 있다.
      * 무/소과금 유저도 꾸준히 참여할 수 있게 한다.

## 이벤트 타입 정의

### 처리 위치 : EventManagerData@event\_type

### 타입

* + - ResourceRankPreview
      * 재화 소모 랭킹 프리뷰 (사전 예고)
        + 이벤트 시작 전 사전 안내 및 참여 유도를 위한 프리뷰 이벤트
        + 시작 시점부터 지정된 조건 카운트를 누적 집계한다.
        + 이벤트 종료 시점에 카운트 값을 초기화하지 않으며, 동일한 event\_series\_id를 가진 ResourceRank 이벤트로 집계 값이 그대로 이관된다.
        + 프리뷰 이벤트는 단독으로 랭킹을 산정하지 않으며, 같은 event\_series\_id를 사용하는 ResourceRank 이벤트를 기준 이벤트로 삼는다.
        + 프리뷰 기간 동안 집계된 누적 카운트 값은 이후 ResourceRank 이벤트의 랭킹 및 마일스톤 계산에 포함된다.
    - ResourceRank
      * 재화 소모 랭킹 (본 경쟁)
        + 누적 집계 카운트 값을 기준으로 채널 단위 랭킹 경쟁을 진행하는 본편 이벤트
        + 동일한 event\_series\_id를 가진 ResourceRankPreview 이벤트가 존재할 경우, 프리뷰 기간 동안 누적된 카운트 값을 이어받아 연속 집계한다.
        + 이벤트 진행 중에도 카운트 값은 지속적으로 누적되며, 랭킹 및 마일스톤 보상 산정에 사용된다.
        + 이벤트 종료 시점에 누적 카운트 집계는 종료되며, 집계 값은 ResourceRankExchange 이벤트에서 읽기 전용 데이터로 활용된다.
        + 이벤트 종료 후 집계 데이터는 event\_series\_id의 reset\_policy 설정에 따라 초기화된다.
    - ResourceRankExchange
      * 재화 소모 랭킹 교환소 (보상 소비)
        + ResourceRank 이벤트에서 확정된 랭킹 및 마일스톤 결과를 기반으로 한 교환 전용 이벤트
        + 해당 기간 동안 카운트 값은 추가로 집계하지 않는다.
        + 동일한 event\_series\_id의 ResourceRank 이벤트 결과를 참조하여 랭킹 보상, 마일스톤 미수령 보상, 교환 상점 접근을 제공한다.

이벤트 진행도 보상 및 랭킹 결과는 이 기간 동안만 수령 가능하다.

* + - * + 교환 이벤트 종료 시, 이벤트 홈(hub\_id)의 상태에 따라 하위 이벤트는 자동 종료 처리된다.
        + 랭킹 보상으로 지급된 영구 재화(예: 랭킹 코인)는 소멸되지 않는다.
    - ResourceRankQuest
      * 재화 소모 미션
        + 이벤트와 연계된 마일스톤 / 미션형 보상 이벤트
        + 동일한 event\_series\_id를 기준으로 누적 카운트 값을 참조한다.
        + 자체적으로 집계하지 않고, ResourceRankPreview / ResourceRank에서 누적된 집계 값을 사용한다.
        + 스텝 기반 보상 구조를 지원한다.

(예: 1스텝 100 % 달성 보상 / 총 n개 스텝)

* + - * + 퀘스트 진행 및 보상 수령 가능 기간은 이벤트 기간 설정을 따른다.
    - ResourceRankShop
      * 재화 소모 랭킹 상점
        + 이벤트와 연계된 패키지 판매 전용 상점 이벤트
        + 이벤트 참여 유저를 대상으로 한 한정 상품 또는 패키지를 제공한다. (적용되는 게임 재화 판매)
        + 랭킹 및 집계 결과와는 직접적인 연산 관계는 없으며, 접근 조건만 event\_series\_id 또는 이벤트 진행 상태를 참조할 수 있다.
        + 이벤트 종료 시 상점 접근 여부는 이벤트 홈(hub\_id) 또는 별도 상점 정책에 따라 결정된다.
    - 타입 별 역할 정리
      * 집계 주체
        + ResourceRankPreview
        + ResourceRank
      * 참조만 하는 타입
        + ResourceRankQuest
        + ResourceRankExchange
      * 판매/접근 중심
        + ResourceRankShop

## 약속 🤙

### 장비 강화 및 승급 재료는 제외

* + - 시스템적으로 약화 / 강화에 쓰이는 재화를 걸러낼 수 없다.

### 이벤트 재화 [당장 구현할 필요 없음]

* + - 이벤트 종료 시 전환해서 우편 발송
      * 제목 : (이벤트 재화 반환)
      * 내용 : (이벤트가 종료되어, 이벤트 재화를 다른 아이템으로 교환해 드립니다)

### UI 처리

* + - 장비/유닛/다이아 소모 이벤트 (3종의 이벤트 홈)는 모아보는 전용 진입 UI를 제작한다. (EventCenterList]
  + 교환 상점
    - 교환 상점은 이벤트가 종료되어도 진입 아이콘을 남겨둔다.
      * 진행 중인 이벤트가 없을 때에도 이벤트 센터 UI에서 접근할 수 있어야한다.

## 레드닷

### 교환 가능 상품이 있을 때

* + - ResourceRank 일 때
      * 이벤트 센터 진입 아이콘
      * 교환 상점 아이콘
        + 해당 시, ResourceRankExchange 슬롯 내 [교환소 이동] 버튼
    - ResourceRankExchange 일 때
      * 이벤트 센터 진입 아이콘
      * 해당 슬롯 내 교환소 이동 버튼

### 수령 가능 마일스톤 보상 있을 때

* + - 이벤트 센터 진입 아이콘
    - 해당 이벤트 슬롯
    - 퀘스트 탭
    - 보상

### ConsumableData

* + - 데이터 추가
      * 랭킹 코인 ID 추가
    - type 추가
      * EventConsumable
        + 인벤토리 : 아이템 탭
        + 처리 : 해당 이벤트 종료 시 남은 수량은 연결된 재화로 반환해서 우편 발송. (나중에 처리)

### ResourceRankSeriesData

* + - event\_series\_id : 고유 아이디
    - condition\_type
      * 신규 조건
        + 이벤트 소모 - 다이아 누적 소모

Event\_Total\_Currency\_Expense

value1: -1

value2: 수량

* + - * + 이벤트 소모 - 유닛 누적 소환

Event\_Total\_Summon\_Unit

value1 : -1

value2 : 횟수

* + - * + 이벤트 소모 - 장비 누적 소환

Event\_Total\_Summon\_Parts

value1: -1

value2 : 횟수

* + - series\_type
      * OneShot // 1회성
      * Repeatable // 주기 반복 (주간, 월간)
      * Seasonal // 시즌 단위 반복
    - reset\_policy
      * OnEventEnd // 이벤트 종료 시마다 리셋
      * OnSeriesEnd // 시리즈 종료(프리뷰~교환 끝날 때) 시마다 리셋
      * Manual // 수동 처리
    - allow\_overlap // 동일한 event\_series\_id 진행 가능 여부
      * TRUE
      * FALSE
    - aggregation\_scope // 집계 유지 범위
      * None // 안 함
      * Event // 이벤트 단위 집계
      * Series // 프리뷰 > 랭킹까지 연속 집계
        + 교환소에서는 더 추가 않고 읽기만한다.
    - is\_active
      * 운영 차단용
