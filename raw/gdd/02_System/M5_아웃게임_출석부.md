# M5 아웃게임 출석부

> **요약:** 신규/일반 유저 상시 출석부와 이벤트 출석부로 구성된 출석 보상 시스템.
> **현재 상태:** 미적용

M5\_아웃게임\_출석부

**M5\_아웃게임\_출석부**

# 개요

* 출석부는 **상시 출석부**와 **이벤트 출석부**가 존재한다.
  + **상시 출석부**는 유저의 상태에 따라서 2가지로 구분되며, 상시로 진행된다.
    - 신규 유저 출석부 : 계정 생성 시 최초 1회 진행
      * 출석부 ID는 상수값으로 지정한다.
    - 일반 유저 출석부 : 신규 유저 출석부 종료 후 반복 제공
      * 일반 유저 출석부는 신규 유저 출석부를 제외한 나머지 출석부 중 랜덤으로 설정된다.
        + 랜덤 설정 시 동일한 출석부를 연속으로 진행할 순 없다.
    - 상시 출석부는 7일 출석부로 고정한다.
    - 상시 출석부는 동시에 1개만 진행된다.
    - 상시 출석부 출석일수에 따라 누적 접속 보상이 지급된다.
  + **이벤트 출석부**는 특정 기간에 접속한 유저에 한해 별도로 진행된다.
    - 이벤트 출석부는 이벤트 관리 테이블에서 일정과 ID를 관리한다. (추후 작업)
      * 이벤트 출석부는 시작일로부터 N 일 뒤 종료되도록 설정한다.
    - 이벤트 출석부는 동시에 N개 진행될 수 있다.

# 동작

## 콘텐츠 해금

* + Contents\_Type = Attendance

## 콘텐츠 진입

* + 콘텐츠 최초 해금 시
    - 조건 만족 시, 메인 화면에서 1회 자동 팝업
  + 콘텐츠 해금 이후 매일 최초 접속 시
    - 최초 접속 시 자동 팝업
  + 그 외 (기본 위치)
    - 햄버거 메뉴 안에 [출석부] 메뉴 추가
    - 메뉴 선택 시 팝업
    - [이벤트 출석부]는 별도 [이벤트 메뉴]에서 제공한다.

## 출석부 결정

* + 신규 유저의 경우 지정된 출석부 오픈
    - ConstantsData@first\_user\_attendance\_board\_id
      * 반복 유저의 경우, 위 출석부를 제외한 나머지 출석부 중 1개를 랜덤 결정한다.
      * 단, 같은 출석부를 연속으로 진행하지 않도록 규칙으로 정한다.

## 출석

* + UI 오픈 시 자동으로 출석 처리 한다.

## 보상 수령

* + 출석 일자의 보상은 수령 가능 상태가 된다.
    - 수령 가능 상태는 UI에서 표현한다.
  + 수령 가능 보상 선택 시, 수령 가능 상태의 모든 보상을 수령한다.
    - 밀린 보상 한 번에 획득 가능
    - 공용 보상 연출
  + 수령한 보상은 UI 에서 수령 완료 상태로 표현한다.

## 누적 출석 보상

* + 누적 출석일
    - 출석 시 누적 출석일이 1일씩 상승한다.
      * 누적 출석일이 최대 4자리까지 표시되도록 UI 영역을 확보한다.
  + 누적 출석 보상
    - 보상은 테이블(AttendanceProgressRewardData)에서 관리하며, 보상의 해제 순서는 step을 참고한다.
    - 누적 출석 2일 마다 보상을 획득할 수 있다.
      * 2일은 ConstantDatas에서 설정한다.
        + attendance\_progress\_reward\_period\_day

2

* + - * 누적 출석 2,4,6,8,10 … 일에 보상 수령 가능
    - 해제된 보상은 수령 가능 상태가 된다.
      * 수령 가능 상태는 UI에서 표현한다.
    - 수령 가능 보상 선택 시, 수령 가능 상태의 모든 누적 출석 보상을 수령한다
      * 수령한 보상은 수령 완료 상태로 표현한다.
    - 최종 step 보상까지 지급한 경우, 다시 step1 보상부터 지급한다.

## 이벤트 출석부 관리

* + 이벤트에서 출석부 ID 지정
    - 서버에 이벤트 ID 저장
      * 출석부 POOL 복사하지 않고 관리하는 방향
