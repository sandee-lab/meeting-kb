# 아웃게임 광고 제거 상품

> **요약:** 광고 제거 상품 활성화에 따른 보상 분기 처리 설계. 활성화/비활성화 유저별 보상 차이와 광고 슬롯 비활성화 규칙을 정의한다.
> **현재 상태:** 이후 버전으로 대체됨

**M7\_아웃게임\_광고 제거**

# 개요

* 플레이 사이클 내 일부 재화 지급처에 광고 시청 버튼을 배치하여 광고 시청을 유도한다.
* 광고 제거 상품(product id= -) 활성화 시, 각 보상을 즉시 얻을 수 있다.

# 구성 요소

## 광고 시청 버튼 (활성화 유저의 경우 보상 획득 버튼)

* + 광고 제거 상품 활성화 여부로 유저 상태를 구분하여 출력한다.
    - 광고 제거 상품 비활성화 유저 (이하 비활성화 유저)
      * 광고 시청 버튼
        + 시청 기회가 1회인 경우 : 일일 시청 기회 표시 생략
        + 시청 기회가 2회 이상인 경우 : 남은 횟수 / 일일 총 횟수 표시

## 일일 광고 시청 기회 (활성화 유저의 경우 보상 획득 기회)

* + 광고 시청을 연결한 각 콘텐츠 테이블에서 일일 시청 기회를 제한한다.
  + 비활성화 유저의 광고 시청 또는 활성화 유저의 보상 획득 후 기회를 1회씩 차감한다.

# 동작

## 공통

* + 유저가 광고 시청 허용 콘텐츠에 접근한다.

## 비활성화 유저

* + 광고 시청 버튼을 터치한다.
  + 버튼 선택 시 광고 재생
  + 광고 종료 후 보상 지급
  + 일일 광고 시청 기회 1회 차감

## 활성화 유저

* + 받기 버튼을 터치한다.
  + 버튼 선택 시 보상 지급
  + 일일 보상 획득 기회 1회 차감

# 광고 슬롯 지정

* 광고 슬롯을 배치할 각 콘텐츠 테이블에서 관리한다.
  + 현황 (유저가 어떤 광고 시청 많이 하는 지 확인하려면 콘텐츠 별로 관리)
    - **스태미나 충전**
  + **1회**
* **순찰 골드 구매**
  + **1회**
* **일일 상점 리셋**
  + **3회**
* **장비 소환**
  + **1회**
* **빠른 순찰**
  + **2회**
* 택틱 리롤
  + 1회
* 던전 별로 소탕
  + 골드
  + 지휘관
  + 유닛

# 버튼 디자인 구상

* 광고 시청 버튼은 통일하여 사용한다.
* 활성화 유저에게는 ‘받기’ 버튼으로 노출하되, 상품(멤버십) 혜택임을 인지시키는 것이 좋다.
  + (가능 시) 멤버십 만료 n일 전에는 “곧 혜택 만료” 식으로 표현을 추가한다.

# 상품 노출 강화

* 레퍼런스 게임들에서 광고 제거 상품 노출하는 것에서 착안.
* 비활성화 유저에게 광고 제거 상품 노출을 강화한다.
* 레퍼런스 게임의 경우 매 광고 시청 시도 시 노출되나, 피로도를 줄이기 위해 규칙을 적용할 수 있겠다.
  + 비활성화 유저가 퀘스트 광고 시청 요구 횟수(현재 2회) 초과 시청 시 출력한다.
    - 일일 노출 횟수 제한이 필요한 경우 ConstantDatas에서 지정할 수 있다.
