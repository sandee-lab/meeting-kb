# M5 인게임/아웃게임 지휘관 v2

> **요약:** 지휘관의 인게임 포지션/체력/역할 및 아웃게임 성장 구조 통합 설계 v2.
> **현재 상태:** 미적용

M5\_인게임/아웃게임\_지휘관\_v2

**M5\_인게임/아웃게임\_지휘관\_v2**

# 개요

* 지휘관은 SCV 형태로 인게임에 배치된다.
  + 전체적인 게임의 스토리를 포장하는 존재 (자원을 수집하는 역할)
  + 지휘관은 자체 체력을 가지며, 지휘관 사망 시 게임 패배 처리
* 인게임에서의 역할이 주어짐에 따라 아웃게임 구조를 보완한다.

value1 : 지휘관 id

value2 : 승급 단계

해당 승급 단계 지휘관을 장착했는지 검사

TRUE : 지휘관 시작 카드. 풀에서 제외.

FALSE : 대상 아님

## 기능

* + - 지휘관 승급 효과를 정리한다.

## 승급 효과를 스킬에서 택틱 카드로 변경한다.

* + - * 택틱 카드는 전투 시작 시 적용되는 **[기본 카드]**와 택틱 풀에 추가되는 **[추가 카드]**로 구분한다.

## **기본 카드**

시작 유닛 카드

CommanderEvolutionData@start\_unit\_ids

시작 유닛도 UI에선 택틱처럼 포장해서 보여주자.

일반 택틱 카드

CommanderEvolutionData@add\_tactic\_type\_1~4 중 type이 “Default”인 경우 해당

## **추가 카드**

일반 택틱 카드

CommanderEvolutionData@add\_tactic\_type\_1~4 중 type이 “Pool”인 경우 해당

* + - * + 기타 규칙

시작 유닛 및 택틱 카드 정보는 현 승급 단계에 맞는 정보만 불러온다.

승급 추가 스탯은 이전처럼 현 승급 단계 **이하** 수치까지 합산한다. (현행 유지, 현 승급 단계에 입력값만큼 추가해주는 방식)

* + 기본 카드
    - 시작 유닛 카드
      * CommanderEvolutionData@start\_unit\_ids
        + 1개인 경우
        + 2개인 경우 ; 사용하여 구분
        + 최대 3개 입력 가능 (논의 필요)

서로 다른 아이디 사용할 예정인지

* + - 택틱 카드
      * CommanderEvolutionData@add\_tactic\_type\_1~4=Default
        + 해당 택틱은 게임 시작 시 적용
        + 현재 레벨에 입력된 것만 적용
  + 추가 카드
    - CommanderEvolutionData@add\_tactic\_type\_1~4=Pool
      * 택틱 풀에 추가되는 택틱
      * 현재 레벨에 입력된 것만 적용
