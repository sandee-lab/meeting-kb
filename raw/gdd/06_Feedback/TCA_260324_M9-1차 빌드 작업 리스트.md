# M9-1차 빌드 작업 리스트

> **요약:** M9 1차 빌드(목표일: 2026-04-03) 인게임/아웃게임 작업 항목 정리
> **현재 상태:** 미적용

# \# M9-1차 빌드 작업 리스트

# 

# \*\*빌드 전달 목표 일자: 4월 3일\*\*

# 

# \## 작업 방향

# 

# \- 양사가 협의한 아웃게임 항목에 대해 개발을 진행합니다.

# \- 기존 전달된 빌드 및 0323 피드백 내용을 토대로 인게임 개선 작업을 진행합니다.

# 

# \---

# 

# \## 🎮 인게임

# 

# \*\*#1\*\* 유닛 등급 마리수 조정 `\[데이터 작업]`

# \- 전설 유닛 조합 난이도를 고려하여 등급별 마리수를 아래와 같이 조정합니다.

# 

# | 등급 | 변경 전 | 변경 후 |

# |------|:-------:|:-------:|

# | 고급 | 7 | 6 |

# | 희귀 | 6 | 6 |

# | 영웅 | 4 | 5 |

# 

# \- '엔지니어'가 고급 등급에서 영웅 등급으로 변경됩니다.

# &#x20; - 등급 변경에 따라 기본 소환 수 증가 등 스킬 개선이 포함됩니다.

# 

# \*\*#2\*\* 전설 유닛 사용성 개선 \& 신규 전설 추가 `\[개발 필요]`

# \- 엣지 → \*\*고양이 사부\*\*로 대체 (클리버 공속 증폭 + 아군 전체 공속↑)

# &#x20; - 엣지는 추후 전기 속성 전설 유닛으로 활용 예정

# \- 신규 전설 추가 — 우선순위에 따라 2단계로 진행

# &#x20; - \*\*1순위\*\*: 미스테리오 (카오스 속성), 청소부

# &#x20;   - 다음 주 중 리소스 제작 예정. 진행 상황에 따라 이번 빌드 미포함 가능

# &#x20; - \*\*2순위\*\*: 광부(드릴), 슈뢰딩거의 상자 (차기 일정으로 진행)

# 

# \*\*#3\*\* 보스 난이도 동적 밸런싱 적용 `\[데이터 작업]`

# \- \*\*의도\*\*: 덱 구성에 따른 보스 전투 유지 시간 편차를 줄이기 위해 동적 밸런싱을 적용합니다.

# &#x20; - 최대/최소 피해 제한 방식은 특정 유닛에 유불리가 발생할 수 있어, 보스 남은 체력 비율에 따라 피해 저항 수치가 단계적으로 상승하는 방식을 채택합니다.

# \- \*\*적용 방향\*\*: 구현 이슈 없음. 데이터 작업으로 진행합니다.

# &#x20; - 남은 체력 비율에 따라 보스에게 피해 저항 버프를 부여하는 범용 스킬을 제작하여 모든 보스 몬스터에 일괄 적용합니다.

# 

# 

# \*\*#4\*\* 일반\~고급 등급 스킬 카드 비중 조절 `\[데이터 작업]`

# \- 일반·고급 등급 유닛의 스킬 카드 등장 비중을 줄이는 것을 목표로 합니다.

# \- \*\*적용 방안\*\*

# &#x20; 1. \*\*일반 등급 스킬 카드 스펙 아웃\*\*

# &#x20; 2. \*\*지휘관 스킬 카드 추가\*\*

# &#x20;    - 지휘관 승급 시 적용되는 스킬 카드를 인게임 내 배분

# &#x20;    - 지휘관 승급 포지션은 추후 콘텐츠 투입 시 최종 반영 예정

# &#x20;      - 기본 패시브 스킬을 제공하고, 승급에 따라 카드 선택 기회 부여

# &#x20;      - 스테이지 입장 시 카드 선택 가능 (운명의 갈림길 대체)

# &#x20; 3. \*\*스킬 카드 추가 스탯 UI 분리\*\*

# &#x20;    - 스킬 카드에 부가적으로 제공되는 스탯은 UI 디자인상 별도 분리

# 

# \*\*#5\*\* 보스 처치 후 유물 시스템 제거 `\[개발 필요]`

# \- 인게임 보스 처치 후 제공되던 유물 보상을 제거합니다.

# \- 보스 처치 시 보상 기대감 강화를 위해 아래 두 가지 방향으로 보완합니다.

# &#x20; 1. \*\*보스 처치 연출 보완\*\*: 연출 개선과 함께 현재 지급되는 미네랄 획득 표현을 강화합니다.

# &#x20; 2. \*\*대체 콘텐츠\*\*: 후보 — 카드 교환소 (불필요한 카드를 전설\~신화 카드로 교환하는 콘텐츠)

# \- ⚠️ 대체 콘텐츠 확정 필요

#

# \- 승급 단계 도달 시 연출 UI를 추가합니다.

# \- ⚠️ 세부 내역은 별도 문서로 정리 예정
