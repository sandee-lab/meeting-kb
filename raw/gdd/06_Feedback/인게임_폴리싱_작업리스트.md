* 소환수의 hit 처리

      * 어택박스 쪽에 처리

## [완료] 패시브 발동 연출 UI 추가

* 택틱, 유닛 패시브 등이 특정 조건에 발동되는 경우에 대해서 유저에게 정보를 전달할 필요가 있다.
* 각 스킬이나 상태이상이 부여되는 시점에 UI 출력 할 수 있도록 시스템을 추가한다.
  + 구현에 따라서 스킬 발동 시 표시할지, 상태이상 부여시 발동할지 구분이 필요하다.
  + 스킬, 상태이상에서 각각 발동 연출 여부를 지정할 수 있도록 한다.
    - skilldata, skilleffectdata
      * ui\_show\_type 컬럼을 추가
        + None 이면, 표시 안함![](data:image/png;base64...)
        + Default 이면, 데이터 참조해서 아이콘 표시
        + LegendaryUnit 이면, owner의 아이콘 표시
    - skilldata에 local\_name, icon\_path, icon\_name 컬럼을 참조한다.
    - skilleffectdata에 local\_name, icon\_path, icon\_name 컬럼을 참조한다.
  + 전설 유닛의 스킬이 발동되는 경우 유닛의 아이콘을 지정하는 것이 좋아보이는데, 구현적으로 어떻게 할지는 고민이다.
* UI 연출 규칙
  + 연출 리스트에 조건을 만족하는 스킬, 상태이상이 스탯된다.
  + 발동 순서대로 UI 출력한다.
  + 출력 시 0.2초 정도 강조해주고, 위쪽으로 롤링 연출을 적용한다.
  + 발동 시점과 살짝 딜레이 되더라도 순차적으로 다 출력해준다.

### [상]속성 카드 수집 수 만큼 속성 위력 증가

* + Check\_Fire\_Tactic\_Count 인 경우
    - 획득한 택틱의 property\_type 값이 Fire인 것을 카운팅한다.
    - stat\_type에서 지정한 스탯을 다음과 같이 적용한다.
      * 증가 값 = count \* effect\_value
    - 만약 보유한 Fire 속성 택틱이 10개라면,
      * 10 \* 0.01 = 0.1 만큼을 + 시켜준다.
    - 적용할 값은 effect\_value2 값을 넘을 수 없도록 한다.

| 택틱 이름 | effect\_type | stat\_type | effect\_value | effect\_value2 |
| --- | --- | --- | --- | --- |
| 파이어 어셈블 | Check\_Fire\_Tactic\_Cnt | mul\_increase\_fire\_damage | 0.01 | 1 |
| 아이스 어셈블 | Check\_Ice\_Tactic\_Cnt | mul\_increase\_Ice\_damage | 0.01 | 1 |
| 일렉트릭 어셈블 | Check\_Electric\_Tactic\_Cnt | mul\_increase\_electric\_damage | 0.01 | 1 |
| 카오스 어셈블 | Check\_Chaos\_Tactic\_Cnt | mul\_increase\_chaos\_damage | 0.01 | 1 |
| 낫씽 어셈블 | Check\_Nothing\_Tactic\_Cnt | mul\_increase\_nothing\_damage | 0.01 | 1 |
