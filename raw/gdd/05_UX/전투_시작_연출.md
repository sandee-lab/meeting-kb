# 인게임 전투 시작 연출

> **요약:** 전투 시작 시 비행선·지휘관 드롭 연출 시퀀스 설계. 로딩 화면 트랜지션과 택틱 선택 연출을 포함한다.
> **현재 상태:** 이후 버전으로 대체됨

**M7\_인게임\_시작연출**

![](data:image/png;base64...)![](data:image/png;base64...)

# **개요**

* 전투 시작 시, 시작 연출을 통해 지휘관 택틱 효과를 인지시킨다.

# **시퀀스**

* 게임 로비에서 [Play] 버튼 터치
* 로딩 화면 출력

![](data:image/png;base64...)

* + 구성 요소
    - 챕터#
    - 챕터 제목 (StageChapterData@local\_name)
    - 비행선
      * 로딩 화면 중앙에 비행선 ![](data:image/png;base64...) 이미지 배치하여, 로딩-시작 연출 간 연속성을 부여한다.
      * 로딩 종료 시점에는 비행선 형태 활용하여 맵으로 트랜지션 (연출 논의 희망)

![](data:image/gif;base64...)

[연출 예시?]

* + - ‘로딩’ 텍스트
      * 로딩 되는 동안 . / .. / … 반복
* 시작 연출
  + 맵 출력
  + 비행선에서 [지휘관]과 [상자]가 드롭된다.
    - 지휘관 모션 필요.
    - [지휘관]과 [상자] 드롭한 비행선은 화면 밖으로 사라진다.
  + 드롭된 [상자]가 개봉된다. ![](data:image/png;base64...)
    - **게임 시작 (몬스터 스폰 시작)**
    - [상자]에서 현재 지휘관이 보유한 택틱이 드롭된다.![](data:image/png;base64...)
      * 장착한 지휘관의 tactic\_1~ tactic\_5 (CommanderEvolutionData)
        + tactic\_1~tactic\_4 : 택틱 아이콘 출력
        + tactic\_5 : 유닛 아이콘 출력
  + 인게임 상단 메뉴 출력

![](data:image/png;base64...)

* + 상단에서 메뉴가 내려온다.
  + 드롭된 [택틱] 타입 별로 연출을 진행한다.
    - tactic\_1~tactic\_4 (전략 카드) : 개별 아이콘이 일시정지 아이콘 방향으로 플라이
    - tactic\_5 (시작 유닛) : 드롭된 자리에서 버블이 터진다.
      * 상자 터지는 연출 > 버블 터지는 연출로 변경 필요

![](data:image/gif;base64...)

* + ‘Survive’ 문구 출력
* 예외 처리
  + 매 챕터 Wave 1은 기존 웨이브 시작 연출을 생략한다.
* 필요 리소스
  + 로딩 화면
  + 로딩 > 인게임 트랜지션 연출
  + 지휘관 모션
  + 버블 터지는 연출
  + Wave1 시작 텍스트 연출 (Survive!)
