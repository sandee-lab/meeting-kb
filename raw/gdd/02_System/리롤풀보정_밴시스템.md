# 리롤 Pool 보정 (Ban 시스템)

> **요약:** 택틱/유닛 리롤 시 원하는 결과에 확정 수렴하도록 풀 보정(밴) 메커니즘을 설계하는 개발 명세서. 리롤 실행 시 이미 제시된 카드/유닛을 ban 처리하여 재등장을 방지하고, 리롤 횟수가 쌓일수록 원하는 결과 확률이 선형 상승한다. 등급별 독립 관리되며 소환/합성에는 미적용된다.
> **현재 상태:** 미적용

**개발 명세서 — 리롤 Pool 보정**

문서 번호: TCA-SYS-001 | 작성: 윤호민 | 상태: 검토 중

**1. 개요**

| **항목** | **내용** |
| --- | --- |
| **기능명** | 리롤 Pool 보정 (Ban 시스템) |
| **적용 대상** | 택틱 리롤 / 유닛 리롤 |
| **미적용 대상** | 유닛 소환, 유닛 합성(머지) — 리롤에만 적용 |
| **목적** | 리롤 횟수 소모 시 원하는 결과에 확정 수렴하도록 보정 |

**2. 배경 및 문제**

**타 로그라이크 리롤 구조**

타 장르에서 리롤은 '효율적인 자원 관리 → 경제 이득 → 원하는 것을 얻을 때까지 리롤'의 흐름으로 설계되어, 충분히 리롤하면 원하는 결과를 확정적으로 얻을 수 있다.

결과적으로 실패 시 '운이 나빴다'가 아닌 '내가 잘못했다'는 피드백으로 귀결된다.

**현재 TCA 문제**

⚠ 현재 TCA의 택틱/유닛 리롤은 횟수를 모두 소진해도 원하는 결과를 보장받지 못한다.

• 실패 경험이 실력이 아닌 확률의 문제로 귀결됨

• 부정 감정이 커져 이탈 요인으로 작용

• 리롤의 전략적 가치 희석

**3. 택틱 리롤 보정**

**동작 방식**

• 한 번의 카드 선택 기회 내에서 리롤 시, 이미 제시된 카드는 즉시 ban 처리

• ban된 카드는 현재 선택 기회가 끝날 때까지 재등장하지 않음

• 리롤 횟수가 쌓일수록 원하는 카드를 찾을 확률이 선형적으로 상승

**Ban 해제 조건**

*※ 현재 선택 기회 내 등장 가능한 카드가 모두 ban된 경우, ban을 전부 해제하고 풀을 초기화한다.*

**상세 스펙**

| **항목** | **내용** |
| --- | --- |
| **Ban 트리거** | 리롤 실행 시점에 현재 제시된 카드 전체 ban |
| **Ban 범위** | 현재 선택 기회 내에서만 유효 |
| **초기화 시점** | 카드 선택 완료 시 (다음 레벨업 시 자동 초기화) |
| **Ban 해제** | 등장 가능 카드 전체 소진 시 ban 전부 해제 후 풀 초기화 |

**4. 유닛 리롤 보정**

**동작 방식**

• 특정 등급의 유닛을 리롤하면, 해당 아머리 내에서 동일 유닛이 중복 등장하지 않도록 ban 처리

• 등급별 독립 적용 — 고급 리롤 ban은 희귀 리롤에 영향 없음

• 충분한 리롤 횟수 소모 시 해당 등급 내 원하는 유닛 확정 획득 가능

**Ban 해제 조건**

*※ 해당 등급 내 등장 가능한 유닛이 모두 ban된 경우, ban을 전부 해제하고 풀을 초기화한다. 이후 리롤은 다시 전체 풀에서 시작한다.*

**상세 스펙**

| **항목** | **내용** |
| --- | --- |
| **Ban 트리거** | 리롤 실행 시점에 현재 제시된 유닛 ban |
| **Ban 범위** | 현재 아머리 내, 해당 등급에서만 유효 |
| **등급 독립성** | 등급별 ban 풀은 서로 독립적으로 관리 |
| **Ban 해제** | 해당 등급 내 전체 유닛 소진 시 ban 전부 해제 후 풀 초기화 |
| **적용 범위** | 리롤에만 적용 — 소환 및 머지(합성)에는 미적용 |

⚠ 소환 및 머지(합성)는 기존 Pool 로직을 그대로 유지한다. 본 보정은 리롤 액션에만 적용된다.

**5. 기대 효과**

| **항목** | **내용** |
| --- | --- |
| **실패 피드백 개선** | '운이 나빴다' → '리롤을 더 했어야 했다'로 귀결 |
| **리롤 전략성 강화** | 자원 관리와 리롤 타이밍이 실력 요소로 작동 |
| **부정 감정 감소** | 원하는 결과에 수렴한다는 확신이 스트레스 완화 |
| **풀 고갈 방지** | Ban 해제로 무한 리롤 시 빈 풀 오류 없음 |
