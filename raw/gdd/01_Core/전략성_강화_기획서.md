# 전략성 강화

> **요약:** 유닛에 공격 형태/속성/빈도 태그를 부여하는 태그 시스템을 도입하여 전략성을 강화하는 핵심 기획이다. 불꽃/냉각/전기/카오스/무속성 5종 속성 체계와 소환/지속/범위/연사/넉백/지원/멀티타겟 공격 형태를 정의하고, 속성별 상태이상 시너지(화염-폭발-확산, 감전-기절 등)를 설계한다. 택틱을 스택/단일 카드로 재구성하고, 속성 변경/부여/등급별 강화 등 다양한 전략적 선택지를 제공한다.
> **현재 상태:** 미적용

# 전략성 강화

# 목표

* 다양한 유닛을 소환, 머지해서 플레이하는 기본적인 방향성 안에서 전략성을 강화하기 위한 작업들을 추가하도록 한다.

# 방향

* 택틱 선택지에서 **스택되지 않는 개별 택틱**을 혼용한다.
  + 즉각적인 선택과 장기적인 선택지 사이에서 유저가 의미있는 선택을 할 수 있도록 한다.
* 유닛의 공격 형태, 속성 등의 계열을 추가하여 계열에 다른 성장 선택이 의미있게 이어질 수 있도록 한다.
* 상태이상을 중심으로 속성 별 플레이 시너지가 극대화 될 수 있도록 고려한다.

# 개요

* 태그 시스템
  + 개별 유닛에게 공격 형태, 속성 등의 태그를 부여한다.
  + 각 유닛은 이러한 태그를 통해서 특정 계열에 속하게 된다.
  + 유닛이 변하더라도 계열에 대한 성장 선택지가 유지되도록 하여 유저의 선택이 의미있게 이어지도록 한다.
* 유닛 조합의 직관성 강화
  + 유닛 > 전설 유닛으로 넘어가면서 같은 계열, 비슷한 공격 형태가 이어지도록 구성하여 중간 과정이 의미있어지도록 구성한다.
  + 예)
    - 방패병(방패) + 제다이(전기) = 토르(전기 + 방패)
    - 새퍼(소환) + 블레이즈(화염) = 라이더(화염지뢰 소환)
* 택틱 구성
  + 스택 카드
    - 최대 레벨 5 > 3으로 압축
    - 최대 레벨 달성 시 스페셜 카드 선택
      * 기존 : 특정 전설 유닛과 조합되는 카드
      * 변경 : 선택한 택틱 카드에 해당하는 능력치를 크게 올려주는 즉각적인 강화 카드로 구성
      * 몇 가지 플레이스타일에 따른 선택지 제공 가능
        + 예) 공격 택틱 카드

A : 유닛 공격력 100% 증가

B : 이동 중에 공격력 150% 증가

C : 매 초 마다 공격력 증가 (최대 200%) (웨이브 종료 시 초기화)

* + - 스택 카드의 구성
      * 범용 스탯 : 공격력, 공격 속도, 쿨타임 가속, 치명 확률 등
      * 속성 강화 : 특정 속성 강화
  + 단일 카드
    - 누적 되지 않으며, 1장만 획득할 수 있는 카드
    - 등급은 전설로 고정.
    - 카드의 종류
      * 지휘관에게 스킬 부여 (Add\_Skill)
      * 공격형태 강화
        + 유닛이 보유한 태그에 따른 스킬 강화

소환, 지속, 범위, 연사 등등

* + - * 속성 부여
        + 특정 계열의 유닛에게 속성 부여

예)무속성 유닛에게 불속성 부여

* + - * 속성 변경
        + 특정 계열의 유닛 속성을 변경

예) 불속성 유닛을 물속성으로 변경

# 유닛 태그 시스템

* 각 유닛에게 태그를 부여할 수 있도록 시스템을 추가한다.
* 하나의 유닛은 여러 개의 공격 형태 태그와 하나의 속성 태그를 가질 수 있다.

## 공격 형태

* + - 기본 스킬
      * 하나의 유닛은 하나의 기본 스킬을 갖는다.
      * 기본 스킬은 별도의 언락 레벨 조건 없이 기본으로 등록한다.
      * UI 상에서 기**본 스킬을 기준으로 유닛의 공격 형태 태그를 표시해준다.**
        + UnitData@linked\_base\_skill\_id
    - 서브 스킬
      * 기본 스킬 외에 여러 개의 보조 스킬을 가질 수 있다.
      * 각 보조 스킬 별로 언락 레벨 조건을 설정해야 하고, 해당 레벨 달성 시 잠금 해제된다.
        + UnitData@linked\_sub\_skil\_unlock\_level
        + UnitData@linked\_sub\_skil\_ids
      * 보조 스킬은 태그를 쓰지 않는다.
    - 공격 형태
      * 소환, 범위, 연사, 지원, 넉백, 멀티타겟 등
      * 스킬 시스템 구현에 따라서 공격 형태는 추가될 수 있다.

## 소환

### [소환] 태그 지정

* + - SkillData@is\_tag\_spawn
      * 값이 True 이면, 소환 태그 부여

### 기본 소환 수

* + - 소환 스킬에 기본 소환 수를 지정하도록 컬럼을 추가한다.
      * SkillData@spawn\_count

### 필요 스탯

* + - 소환 수 증가(add\_spawn\_count)

### 계산 방식

* + - 최종 소환 수 = 기본 소환 수 + 소환 수 증가 능력치 값

### 소환 위치

* + - 내 주변 랜덤한 위치에 스폰하는 것을 기본 규칙으로 한다.
    - 몬스터의 패턴 다양화를 고려하면, 어택박스를 활용해서 소환 마리 수와 위치를 컨트롤 하고 싶다. (이 부분에 대한 확장 가능성에 대해서는 구현 논의 필요)

## 지속

### [지속] 태그 지정

* + - SkillData@is\_tag\_duration
      * 값이 True 이면, [지속] 태그 부여

### 필요 스탯

* + - 지속 시간 증가(mul\_attackbox\_duration)

### 계산 방식

* + - 어택박스의 기본 Duration \* ( 1 + mul\_attackbox\_duration)

### 예외 옵션

* + - 어택박스의 duration 값이 -1이면 적용 하지 않는다.
    - 각 어택박스 마다 스탯 영향을 받을지 On / Off 할 수 있어야 한다. 값이 Off 이면, 적용 하지 않는다.
      * check\_mul\_attackbox\_duration

## 범위

### [범위] 태그 지정

* + - SkillData@is\_tag\_scale
      * 값이 True 이면, [범위] 태그 부여

### 필요 스탯

* + - 범위 증가(mul\_attackbox\_scale)

### 계산 방식

* + - 어택박스의 기본 사이즈 \* ( 1 + mul\_attackbox\_scale)
      * 어택박스에 종속된 이펙트에도 적용되어야 한다.
      * 프리팹 링크로 연결된 이펙트도 적용되어야 한다.

### 예외 옵션

* + - 어택박스에 종속된 이펙트의 스케일도 적용될 수 있어야 한다.
    - 각 어택박스 마다 스탯 영향을 받을지 On / Off 할 수 있어야 한다. 값이 Off 이면, 적용 하지 않는다.
      * check\_mul\_attackbox\_scale

## 연사

### [연사] 태그 지정

* + - SkillData@is\_tag\_repeatable
      * 값이 True 이면, [연사] 태그 부여

### 기본 연사 확률

* + - 스킬 데이터에 기본 연사 확률 컬럼을 추가한다.
      * SkillData@repeatable\_rate

### 필요 스탯

* + - 연속 공격 확률(add\_repeatable\_rate)

### 계산 방식

* + - 최종 연사 확률 = 기본 연사 확률 + add\_double\_attack\_rate

### 처리 규칙

* + - 스킬 종료(Skipable) 시점에 확률을 판단한다.
    - 확률 성공 시 쿨타임으로 넘어가지 않고, 스킬을 다시 오퍼레이팅한다.
    - 연속 공격은 1번 발동하면, 쿨타임 종료 후에 횟수가 회복된다. 즉, 연속 공격이 중복 발동되지는 않는다.

  + 효과 레벨 상승

    - 예)

      * 값이 True 이면, [멀티타겟] 태그 부여

* + - SkillData@passive\_target\_count

### 필요 스탯

* + - 타겟 수 증가(mul\_passive\_target\_count)

### 계산 방식

# 공격 빈도

## 공격 빈도 태그 지정

* + SkillData@tag\_frequency\_type
    - 값이 Speed 이면, [공격 속도] 태그 부여
    - 값이 Cooltime 이면, [쿨타임] 태그 부여
    - 값이 Channeling 이면, [채널링] 태그 부여

## 공격 속도 태그 규칙

### 필요 스탯

* + - mul\_atk\_speed

### 계산 방식

* + - [공격 속도] 태그를 가진 경우의 계산 방식
      * 공격 속도 = add\_atk\_speed \* (1 + mul\_atk\_speed)
    - [공격 속도] 태그가 아닌 경우의 계산 방식
      * 공격 속도 = add\_atk\_speed

## 쿨타임 태그 규칙

### 필요 스탯

* + - mul\_skill\_recharge\_speed

### 계산 방식

* + - [쿨타임] 태그를 가진 경우의 계산 방식
      * 쿨타임 충전 속도 = add\_skill\_recharge\_speed \* ( 1 + mul\_skill\_recharge\_speed
    - [쿨타임] 태그가 아닌 경우의 계산 방식
      * 쿨타임 충전 속도 = add\_skill\_recharge\_speed

## 채널링 태그 규칙

### 필요 스탯

* + - mul\_channeling\_duration

### 계산 방식

* + - [채널링] 태그를 가진 경우의 계산 방식
      * 채널링 시간 = loop\_duration \* ( 1 + mul\_channeling\_duration)
    - [채널링] 태그가 아닌 경우의 계산 방식
      * **채널링 스킬로 동작하지 않는다.**

### 채널링 스킬의 동작 방식

* + - 스킬 발동 시 채널링 스킬이면, [채널링 시간] 동안 스킬 오퍼레이팅 상태를 유지한다.
    - 스킬 오퍼레이팅 상태일 때, 공격 모션을 Loop 한다.
      * 모션이 1초 이고, 채널링 시간이 5초면, 1초 모션을 5번 반복한다.
      * 모션의 0.5초 시점에 이벤트가 등록되었다면, 0.5초 - 1.5초 - 2.5초 - 3.5초 - 4.5초 시점에 이벤트가 발동된다.
    - 채널링 시간이 모두 소진되면, 스킬을 종료 처리한다.
    - 스킬 종료 시 쿨타임 상태로 넘어간다.
    - 쿨타임 시간 종료 후 준비 상태가 되어 조건을 만족하면, 다시 스킬이 발동된다.
  + 채널링 스킬의 모션 제작 규칙
    - 영삼님 체크 필요

# 속성

## 불꽃

### 속성 타입

* + - Unit\_Data@Property\_Type
      * Fire

### 관련 스탯

* + - 불 속성 주는 피해 증가(mul\_increase\_fire\_damage)
      * 불속성 유닛의 스킬일 대, 주는 피해량에 곱연산으로 적용
      * = 주는 피해량 \* ( 1 + mul\_increase\_fire\_damage)
    - 불 속성 받는 피해 증가(mul\_increase\_fire\_damage\_taken)
      * 불속성 유닛에게 피해 받을 때, 받은 피해량에 곱연산으로 적용
      * = 받는 피해량 \* ( 1 + mul\_increase\_fire\_damage\_taken)

### 관련 상태이상

* + - 화염
      * 지속시간동안 매 초마다 아군 공격력의 n% 피해
    - 폭발
      * 화염 상태이상 최대 스택 달성 시 좁은 범위의 어택박스 생성
    - 확산
      * 화염에 걸린 적 사망 시 주변 적에게 화염 효과 부여

## 전기

### 속성 타입

* + - Unit\_Data@Property\_Type
      * Electric

### 관련 스탯

* + - 전기 속성 공격 강화(mul\_increase\_electric\_damage)
      * 전기 속성 유닛의 스킬일 대, 주는 피해량에 곱연산으로 적용
      * = 주는 피해량 \* ( 1 + mul\_increase\_electric\_damage)
    - 전기 속성 받는 피해 증가(mul\_increase\_electric\_damage\_taken)
      * 전기 속성 유닛에게 피해 받을 때, 받은 피해량에 곱연산으로 적용
      * = 받는 피해량 \* ( 1 + mul\_increase\_electric\_damage\_taken)

### 관련 상태이상

* + - 감전

| Upgrade\_SkillEffect\_All | -1 (모든 대상) | - 상태이상을 부여하는 스킬이 is\_tag\_support = TRUE 일 때만 적용한다.  id 무관하게 부여할 level 에 +1 해준다. |
| Upgrade\_SkillEffect | 5 | - 부여할 상태이상 id 가 5이면 부여할 level 에 +1 해준다. |
| 8 | - 부여할 상태이상 id 가 8이면 부여할 level 에 +1 해준다. |
| 11 | - 부여할 상태이상 id 가 11이면 부여할 level 에 +1 해준다. |

* + [지원] 태그 전용 택틱
    - Upgrade\_SkillEffect\_All 택틱은 [is\_tag\_support] 값이 TRUE인 스킬이 상태이상을 부여할 때만 +1 처리한다.
  + 개별 상태이상 택틱
    - Upgrade\_SkillEffect 택틱은 id를 지정하므로 부여하려는 상태이상 id와 동일한 경우에만 +1 처리한다.
* 상태이상 부여 시 같은 id의 낮은 level 상태이상이 부여되어 있다면, 상위 level의 상태이상을 부여하면서 기존의 level 값을 덮어쓰기한다.
* 예외 상황
  + 스킬에서 상태이상을 부여하는 경우에만 레벨을 적용한다.
  + 단, 오라를 통해서 부여하는 상태이상은 1레벨로 고정한다. 성장하는 상태이상과 분리해서 사용한다.
