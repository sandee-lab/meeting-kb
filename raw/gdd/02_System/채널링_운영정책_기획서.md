# 채널링

> **요약:** 채널 단위 운영 정책으로, 서비스 확장성과 유저 유입을 관리한다. 채널 정원 기준 자동 증설, 채널 그룹 관리, 채널별 계정 생성(복수 계정 허용), 채널 변경 시퀀스와 UI, 게스트 로그인 제한, 동시 접속 불허 정책을 포함한다. 이벤트 관리 테이블을 채널링 구조에 맞게 변형하며, 콘텐츠 언락 시스템에도 채널/계정 기반 조건을 반영한다.
> **현재 상태:** 미적용

**M8\_채널링**

# 개요

* 확장성과 유저 유입 관리를 고려해 운영 정책을 채널 단위로 수정한다.
* 채널 운영에 따라 이벤트 및 콘텐츠 운영 테이블을 수정한다.

# 채널 구조 및 규칙

## 구조

### 최초 채널

* + - 서비스 개시 시점에 최초 채널을 1개 개설한다.
    - 해당 채널의 상태는 ‘신규’로 정한다.

### 채널 정원

* + - 채널 정원은 설정하지 않는다.
    - 채널 증설 기준 인원을 설정한다. (직전 채널이 기준 달성 시)
      * ConstantDatas@channel\_creation\_capacity
      * 서버 데이터 시트로 패치 없이 서버에서 바꿀 수 있게
    - 예 :
      * channel\_creation\_capacity= 1,000
      * 최초 채널 정원이 1,000명 도달 ▶ 신규 채널 증설. 증설 전 신규 채널 상태는 보통으로 변경

### 채널 증설

* + - 마지막 신규 채널의 정원이 채널 증설 기준 인원에 도달하면, 채널을 증설한다.
  + 채널 그룹
    - 채널은 그룹으로 묶어 관리한다. (리스트 표시 목적)
      * 1개 채널 그룹에 속하는 채널 갯수
        + ConstantDatas@channel\_group\_capacity
    - 채널 그룹이 최대 채널 갯수에 도달하면 신규 그룹을 개설한다.
  + 채널 상태
    - 신규 : 신규 채널. 반드시 1개 있어야 하고, 동시에 1개만 존재할 수 있다.
    - 보통 : 신규 채널이 아닌 채널.
    - 점검 : 현재 점검 중인 채널. 관리 목적으로 구분.

| **조건** | **결과** | **운영 기준** |
| --- | --- | --- |
| 최초 채널 | 채널 상태 = 신규  (신규 유저 자동 배치 대상) | 유입을 통한 신규 채널 활성화 유도. |
| 최초 채널 정원 = ConstantDatas@channel\_creation\_capacity | 즉시 신규 채널을 자동 개설,  개설된 채널 상태를 '신규'로 설정 | 서버 확장성 확보 및 유저 유입 차단 방지. |
| 신규가 아닌 채널 | 채널 상태 = 보통  (선택 가능) | 일반적인 플레이 상태 유지. |
| 채널 점검 | 채널 상태 = 점검 | 점검은 아니지만,  관리자가 필요에 의해 특정 채널을 ‘점검’으로 해둘 수 있음  (접속 불가) |

## 규칙

### 계정

* + - 신규 유저는 최초 접속 시점 신규 채널에 배치한다.
      * 별도 채널 선택 화면은 제공하지 않는다.
        + 진입 장벽 최소화
    - 유저가 원할 경우, 계정을 보유하고 있지 않은 채널에 새 계정을 만들 수 있다.
      * 단, 해당 시점에 포화 또는 점검 상태인 채널에는 새 계정을 만들 수 없다.

### 기존 운영 정책 변경

| **항목** | **현재** | **변경** |
| --- | --- | --- |
| 구조 | 1서버 내 채널 분리 | - |
| 콘텐츠 운영 기준 | 채널 | 계정 |
| 계정 생성 | 1서버 당 1계정 | 1채널 당 1계정 |
| 복수 계정 보유 | 불가 (탈퇴 후 재가입) | 가능 (채널 당 1계정 허용) |
| 동일 계정으로 채널 이전 | 불가 | - |

### 저장 요소 규칙

| **구분** | **저장 요소** |
| --- | --- |
| 계정 저장 요소 | 모든 성장 및 경제 관련 데이터 (재화, 인벤토리, 레벨, 덱 설정, 친구 목록)는 채널 계정에 귀속되어 서버에 저장 |
| 클라이언트 저장 요소(로컬) | 단순 환경 설정 데이터 (그래픽 옵션, BGM 볼륨, 언어)는 클라이언트(기기)에 귀속됨. 채널 계정 변경 시에도 설정 유지. |
| ★ 튜토리얼 | 튜토리얼 완료는 각 채널 계정 별로 관리 |

### 중복 검사

* + - 기존 중복 검사 (닉네임) 기준도 채널로 변경한다.

### 동시 접속 (2025-12-15 추가)

* + - 동시 접속은 불허한다.
    - 동시 접속 시, 최근 접속한 기기를 유지하고 이전 접속한 기기를 접속 종료 처리한다.
      * 팝업 필요

# 이벤트 관리 변경

## 개요

* + 시즌 패스 개발 시 진행한 이벤트 관리 테이블을 채널링 구조에 맞게 변형한다.
  + 추후 활용을 위해 콘텐츠 언락 시스템으로 변경 내용을 반영한다.

## 변경

### EventManagerData

#### event\_scope\_type (active\_type 에서 변경)

* + - * 이벤트 타입 결정
        + 타입

Global : 시작일시~종료일시 진행

event\_scope\_type=Global 일 때만, start\_date & end\_date 컬럼 사용

Personal : 계정생성 n일차 기준 운영

Channel : 채널오픈 n일차 기준 운영

#### event\_duration\_type

* + - * 이벤트 진행 방식 결정
        + 타입

fixed\_range : 고정 기간

access\_triggered : 기간 중 접속 시 [period]일 동안

[서버 확인 결과] 이벤트 개시 시 접속 허용일과 기간이 고정되어, 추후 수정하여도 이미 개시된 이벤트 변경은 불가

#### start\_day\_offset & end\_day\_offset

* + - * event\_scope\_type=Personal OR Channel 일 때, 고정 기간 혹은 접속 인정 기간 설정
    - unlock\_condition\_type & unlock\_condition\_value1~2
      * 이벤트 접근 조건 추가
        + **Stage\_Clear**
        + 계정 레벨 X 달성
        + (추가될 수 있음)
  + 예

| **event\_scope\_type** | **event\_duration\_type** | **start\_day\_offset** | **end\_day\_offset** | **period** | **이벤트** |
| --- | --- | --- | --- | --- | --- |
| Channel | access\_triggered | 3 | 7 | 5 | 채널 생성 3~7일차 사이에 접속하면  접속한 날부터 5일간 이벤트 가능 |
| Channel | fixed\_range | 3 | 14 | 0 | 채널 생성 3~14일차 적용 |
