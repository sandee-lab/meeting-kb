# 스폰-몬스터-시스템

> 출처: [[raw/meetings/[0109] 스폰 시스템]], [[raw/meetings/[0915] 플레이 시나리오]], [[raw/gdd/02_System/스페셜_스폰_패턴_기획서]]

## 개요

몬스터 스폰 시스템은 게임의 난이도와 재미를 조율하는 핵심 엔진이다. [0109] 스폰 시스템 문서에서 Vampire Survivors의 시간 결정론적 호드 시스템과 Hades의 상태 의존적 인카운터 시스템 두 패러다임을 기술적으로 분석하고, 이를 통합한 하이브리드 아키텍처를 제안하였다. [0915] 플레이 시나리오에서는 챕터별 웨이브/보스 구성이 정의되었다.

## 상세 내용

### 패러다임 I: 시간 결정론적 호드 시스템 (Vampire Survivors 모델)

글로벌 타이머에 종속되어 작동하는 스폰 방식이다. "시간이 던전 마스터"인 구조.

**웨이브 데이터(WaveData) 구성 요소:**
- WaveID, TimeIndex(시작/종료 초), GlobalMultipliers(HP/Speed 계수)
- SpawnRules: MinActiveCount, MaxActiveCount, SpawnInterval, BatchSize
- EnemyPool: 가중치 기반 무작위 선택(Weighted Random Selection)
- Events: 특수 패턴 스폰 정의

**핵심 메커니즘:**
- MinActiveCount 미만 시 "패닉 모드" 전환 — SpawnInterval을 무시하고 즉시 스폰하여 긴장감 유지
- MaxActiveCount는 물리 엔진 과부하 방지용 상한
- EnemyPool은 1분마다 교체되어 약한 몬스터 누적 방지("물갈이" 메커니즘)
- BatchSize를 활용한 군집(Flock) 느낌 구현 — "0.1초마다 5마리씩" 등

**뷰포트 상대 스폰:**
- 맵의 절대 좌표가 아닌 플레이어 카메라 뷰포트 기준 생성
- 16:9 비율에 맞춘 직사각형 고리(Rectangular Annulus) 알고리즘 사용
- 카메라 경계 + Padding으로 스폰 영역 정의
- 4변(Top/Bottom/Left/Right) 중 무작위 선택 후 해당 변 위 무작위 위치 결정

**디스폰 및 풀링:**
- 카메라 뷰포트에서 1.5배 이상 멀어진 몬스터는 비활성화 후 오브젝트 풀 반환
- 300~1,000마리 활성 개체 유지를 위한 필수 최적화

### 특수 패턴 스폰

**꽃의 벽(Flower Wall):**
- 극좌표계를 사용한 원형 포위 패턴
- 높은 넉백 저항(100%), 시간에 따른 반지름 수축으로 플레이어 압박
- 높은 HP 또는 일정 시간 무적으로 "벽" 기능 유지

**떼거지(Swarm) 및 돌진 패턴:**
- 화면 한쪽 모서리에서 플레이어 방향으로 높은 초기 속도 부여
- "피할 틈"을 찾아 움직이도록 유도

### 패러다임 II: 상태 의존적 인카운터 시스템 (Hades 모델)

플레이어 상태와 전투 진행에 따라 유동적으로 반응하는 시스템이다.

**계층 구조:** Biome(지역) → RoomData(방 데이터) → EncounterData(인카운터 데이터)

**증원(Reinforcement) 트리거:**
- Kill Count Trigger: 현재 웨이브 적의 80% 이상 처치 시 다음 웨이브 스폰
- 스폰 전조(Telegraphing): 스폰 위치에 1~2초간 시각적 효과 표시

**AI 감독관(Director) 시스템:**
- 반복 방지: 이전 방에서 많이 나온 적의 등장 확률을 다음 방에서 하향
- 동적 난이도 조절(DDA): 플레이어 체력/클리어 시간에 따라 난이도 미세 조정
- 엘리트 적의 배분: 일반 몬스터 사이 아머 엘리트 비율 제어

### 난이도 밸런싱 수학적 모델

**난이도 계수 공식:** D = 1 + (Time_elapsed / ScalingRate + StageFactor) * PlayerFactor
- MonsterHP = BaseHP * D^1.5 (체력 지수적 증가)
- SpawnRate = BaseRate / D (난이도 오를수록 스폰 간격 감소)

**고통 예산(Pain Budget) 시스템:**
- 감독관 AI에게 시간 비례 크레딧 지급 (예: 10분차 = 1000 크레딧)
- 각 몬스터에 비용 매김 (해골 5, 박쥐 3, 골렘 50 등)
- "원거리 유닛은 예산의 30%까지만" 등 제약 조건으로 불합리한 조합 방지

**톱니바퀴형 페이싱(Sawtooth Pacing):**
- 상승기(Ramp-up) → 절정(Peak) → 휴식기(Valley) 반복
- 절정 직후 10~20초 스폰량 급감으로 정비 시간 제공

### 챕터별 웨이브 구성 (2025년 9월)

[0915] 플레이 시나리오에서 정의된 구조:

- **1챕터 [5분]:** 10웨이브, 1보스 — 소환 언락
- **2챕터 [10분]:** 10웨이브, 1보스 — 클리어 보상 언락, 영웅 강화, 가이드 완료, 영웅 뽑기
- **3챕터 [20분]:** 15웨이브, 2보스 — 골드 강화 언락, 7일 미션, 인벤토리, 상점
- **4챕터 [30분]:** 20웨이브, 3보스

### 구현 아키텍처 제안

[0109] 스폰 시스템에서 제안된 하이브리드 구조:

- 기본적으로 VS의 시간 기반 웨이브를 따르되, 특정 조건(킬 카운트/피격 여부)에 따라 Hades식 감독관 AI가 개입
- 단순 추적 패턴 사이에 정교한 엘리트 패턴을 섞는 방식

**구현 레이어:**
1. 데이터 레이어: 몬스터 데이터(ID, HP, 속도, 행동 패턴, XP 가치) + 웨이브 데이터(시간, 몬스터 풀, 스폰 주기 함수)
2. 글로벌 매니저: TimeManager(초 단위 OnSecondTick), SpawnManager(웨이브 로드/스폰), PoolManager(오브젝트 풀링)
3. 몬스터 AI: 보이드 알고리즘/컨텍스트 스티어링으로 자연스러운 군집 이동 구현

**최적화 기법:**
- ECS/DOTS 아키텍처로 수천 개 오브젝트 병렬 처리
- 물리 엔진 대신 거리 제곱 비교 기반 자체 충돌 감지
- 경험치 보석 병합 (500개 이상 시 높은 가치의 보석으로 합치기)

### 스페셜 스폰 패턴 시스템 (GDD v1.0, 2026.03)

> 출처: [[raw/gdd/02_System/스페셜_스폰_패턴_기획서]]

스페셜 스폰 시 사용할 스폰 위치 패턴을 지정하는 시스템. 프리팹 기반 좌표 리스트와 스킬 부여 기능을 포함한다.

**현재 상태:** 미적용 (기획 완성)

#### 스폰 타입 3종 (StageWaveData@special_spawn_type)

| 타입 | 설명 |
|------|------|
| None | 기본 규칙대로 범위 내 랜덤 스폰 |
| PrefabPivot | 지정 프리팹의 상대 좌표 활용 (`special_spawn_prefab_path`, `special_spawn_prefab_name`) |
| MapPivot | 각 맵에 지정해둔 고정 좌표 활용 (`map_pivot_id` 지정 필요) |

- 프리팹에서 스폰 몬스터들에게 `add_skill` 해줄 스킬 id 지정 가능 — 해당 스킬은 스폰 시 최우선 발동
- 동시 스폰 또는 min~max 간격으로 n마리씩 랜덤 스폰 선택 가능

#### 완성된 특별 패턴 6종

**[완] 스폰 스킬 패턴**
- `passive_spawn` 조건으로 발동하는 스킬
- 스폰 시 자기 자신을 hide 처리 → 스킬 완료 후 unhide
- 예) 하늘에서 메테오 낙하 연출 후 해당 위치에 몬스터 스폰

**[완] 고정 방향 이동 패턴**
- 스폰 시 가로/세로/지휘관 방향 중 정해진 방향으로 이동
- `UnitData@is_out_bound = TRUE` 설정 시: 맵 바운드 밖에서 태어나고 맵 바운드 밖으로 이동 가능
- `SelfDestructActionSkillObject`로 고정 방향 이동 + 충돌 시 자폭 처리
- 맵 바운드 밖으로 이동하면 자동 디스폰
- 이동 중에도 `passive_move` / `passive_damaged` / `passive_dead` 등 패시브 스킬 정상 발동

**[완] 돌진 스킬**
- `Dash_Start → Dash_Loop` 2단계 모션 구성
- 스킬 발동 시 돌진 방향으로 워닝 마크 표시
- Dash_Start 종료 지점에 AttackBox01 트리거 삽입 → 어택박스 생성(duration -1, 스킬 종료 시 제거)
- Dash_Loop: 스킬에서 지정한 목표 거리 이동 완료 시 LoopEnd 처리
- 스킬 오브젝트에서 목표 거리·이동 속도 지정

**[완] 텔레포트 스킬**
- start-loop-end 시전 모션 포함
- 시전 중 목표 좌표 기준 워닝 마크 표시
- 시전 모션 중 특정 타이밍에 텔레포트 실행
- 이동 후 나머지 모션 + 어택박스 소환으로 피해 부여

**[완] 이동 스킬**
- 이동 모션 그대로 사용
- 스킬 발동 시 이동 목표 지정 (예: 타겟 반대 방향으로 n 거리 도망)
- 목표 도달까지 이동 스킬 상태 유지
- 이동 중에도 패시브 스킬 발동

**[완] 소환 위치·타이밍 어택박스 제어 소환 스킬**
- `attackbox spawn listener`를 통해 어택박스가 아닌 소환수 스폰 이벤트 발동
- 원하는 타이밍·위치·수량으로 소환수 스폰 제어
- 예) 슬라임이 독 구슬을 파라볼라로 발사 → despawn 타이밍에 소환수 스폰

#### 추가 이슈: 한 웨이브 내 복수 스페셜 스폰

`special_spawn_type` 옵션으로 once(1회) / interval(간격 반복) 구분 방식 고려 중 — 한 웨이브에서 스페셜 스폰을 여러 번 발동할 수 있는 구조 검토.
