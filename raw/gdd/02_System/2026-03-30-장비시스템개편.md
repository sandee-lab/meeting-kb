# S급 장비 시스템 개편

> **요약:** 스페셜 소환에 S급 장비를 추가하기 위한 CSV 구조 변경(PartsData, ShopDrawData, ShopDrawPoolData) 및 클라이언트 변경 사항 정리
> **현재 상태:** 미적용

# S급 장비 시스템 개편

> 작성일: 2026-03-30

---

## 개요

스페셜 소환에 S급 장비를 추가하기 위한 데이터 구조 및 클라이언트 변경 사항 정리.
확률 계산은 서버에서 수행. 클라이언트 CSV는 UI 표시용 + 서버 동기화 기준.

### 설계 방침
- `ShopDrawData`에 `epic_s_rate` 컬럼 추가 (S급 Epic 확률 분리)
- `ShopDrawPoolData`에 일반/S급 장비 ID를 모두 등록
- 서버가 풀에서 아이템 선택 시 `PartsData.is_special`로 일반/S급 분기하여 해당 확률 적용

---

## 1. CSV 변경

### 1-1. PartsData.csv
- `is_special` bool 컬럼 추가
- S급 장비 행에 `TRUE` 설정

### 1-2. ShopDrawData.csv
- 스페셜 소환 행 추가 (draw_type=`Special`, product_id=50002)
- `epic_s_rate` 컬럼 추가 (S급 Epic 확률, 만분율)
- 기존 `epic_rate` = 일반 Epic 확률 유지
- 일반 소환(50001): `epic_s_rate=0` (S급 없음)
- 스페셜 소환(50002): `epic_rate=340`, `epic_s_rate=60`
- `fail_count_max` = 일반 영웅 천장 (10회)
- `fail_count_max_s` 컬럼 추가 = S급 영웅 천장 (60회)
- `success_grade_s` 컬럼 추가 = S급 천장 보장 등급

### 1-3. ShopDrawPoolData.csv
- product_id=50002 풀에 일반 장비 + S급 장비 ID 모두 등록
- **풀 자체에는 일반/S급 구분 컬럼 불필요** — `PartsData.is_special`로 런타임 판별

---

## 2. C# 데이터 클래스 변경

| 파일 | 변경 |
|------|------|
| `PartsDatas.cs` | `public bool is_special;` 추가 |
| `ShopDrawDatas.cs` | `public int epic_s_rate;` 추가 |
| | `public int fail_count_max_s;` 추가 |
| | `public GradeType success_grade_s;` 추가 |
| `ShopDrawPoolDatas.cs` | 변경 없음 |

---

## 3. 소환 UI 변경

### 3-1. SummonPopUpUI.cs
- `ShopDrawType.Special` (이미 enum에 존재) 활용
- Parts 내부 일반/스페셜 서브탭 추가
- 천장 UI: 일반 천장 + S급 천장 **2줄** 표시
  - `fail_count_max` → 일반 영웅 잔여 횟수
  - `fail_count_max_s` → S급 영웅 잔여 횟수

### 3-2. PartsSlot.cs / PartsSummonResultSlot.cs
- `PartsData.is_special` 기준 S급 뱃지/테두리 표시

### 3-3. PartsUI.cs / PartsPopupUI.cs
- 인벤토리에서 S급 필터/정렬
- 상세 팝업에 S급 표시

---

## 4. UserShopDatas.cs — 이중 천장 카운터

현재: `_userShopDrawFailCountInfo` = `Dictionary<int, UserItemData>` (product_id → count 1개)

변경 방안:
- S급 천장 카운터를 **별도 키**로 관리 (예: product_id + offset)
- 또는 `Dictionary<int, (int normal, int special)>` 구조로 확장
- 서버 API 응답에서 두 카운터 모두 반환 필요

---

## 5. 서버 동기화 (WAS)

- `ShopBuySend()`에서 product_id=50002로 스페셜 소환 요청
- 서버 응답에 이중 천장 카운터(일반/S급) 포함 필요
- 서버 측 확률 로직: 풀에서 아이템 선택 → `PartsData.is_special` 확인 → `epic_rate` or `epic_s_rate` 적용

---

## 6. 소환 연출

- `PartsSummonResultSlot.cs`에서 `is_special` 기준 분기
