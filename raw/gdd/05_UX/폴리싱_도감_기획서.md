# 폴리싱 - 도감

> **요약:** 도감의 정보 과부하와 클릭 피로를 해결하기 위한 폴리싱으로, 택틱(등급별)/몬스터(타입별) 카테고리 분류, 일괄 수령 버튼 배치, 보상 미수령 항목 자동 앞정렬 기능을 포함한다. 레드닷은 햄버거 메뉴부터 개별 항목까지 계층적으로 표시하며, 유닛 택틱 카드 조건 표기 등 편의성 개선도 포함한다. CollectionTacticGroupData는 삭제 대상이다.
> **현재 상태:** 미적용

## 0.현재 도감의 문제점

* 찾기 어려움
* 클릭하려면 오래 걸림
  + **불편함을 넘어 불쾌하다**

##

## 1.해결 방식

**"한 번에 노출되는 정보량 감소 + 클릭 수 최소화"**

* **도감 별 카테고리 구성 :** 한 번에 노출되는 수집 요소 수량을 줄인다
* **일괄 수령 버튼 배치 :** 한 번에 보상을 수령하게 한다.
* **보상 받을 카드는 자동으로 맨 앞 정렬 :** 카테고리 별로 보상 받을 슬롯을 가장 앞에 노출해준다 (계속 1번 슬롯을 눌러 보상 받을 항목 팝업 열어 볼 수 있음)

## 2-1. [완료] 도감 별 카테고리

* 처리
  + 기존 도감 구조 변경
    - CollectionGroupData 삭제
  + 각 도감에 카테고리 반영
    - 택틱
      * 기준 : 등급 (전체, 일반, 고급, 희귀, 영웅, 신화 - 총 6개)
        + StageModeTacticPoolData@grade

전설, 고대 등급은 도감 카테고리 및 필터에서 사용하지 않으나, 데이터 구조상 확장 가능성은 유지한다.

* + - 몬스터
      * 기준 : 몬스터 등급 (일반, 보스)
        + UnitData@monster\_grade

## 2-2. [완료] 일괄 수령 버튼 배치 [수정 필요]

* 처리
  + 배치 기준
    - 택틱 도감 통합 (카테고리 구분 X)
    - 몬스터 도감 통합 (카테고리 구분 X)
      * **[추가 요청] 도감 별로 통합이 필요합니다.**
  + 동작
    - [일괄 수령] 버튼을 배치한다.
      * 활성화 :
        + 조건 : 수령 가능한 보상이 있을 때
        + 처리 : 해당 도감의 수령 가능한 보상 모두 수령
      * 비활성화 :
        + 조건 : 수령 가능한 보상이 없을 때
  + 연출
    - 보상 수령 시, 공용 보상창을 띄운다.
      * 개별 보상은 상세 팝업에서 보상 수량을 출력하고, 플라이 연출만 진행한다.
      * 전체 받기 시, 전체 보상량 확인이 필요하므로 공용 보상창을 추가한다.
        + **[추가 요청] 공용 보상 창에서 동일한 아이템 수량은 합산해서 보여주고자 합니다.**

**현황: 다이아 보상이 개별 슬롯으로 찍힘 (5개 항목의 보상 수령 시, 다이아 슬롯이 5개 찍힘)**

**개선: 다이아 수량 합쳐서 1개 슬롯으로 표현**

## 2-3. [완료] 자동 정렬

## 처리

* + **[확인 필요]** 보상 미수령 항목이 있는 경우, 일시적으로 '미수령' 카테고리를 편성할 수 있는지 점검
  + 위 방식으로 처리가 어렵다면, 각 카테고리 (택틱의 경우 등급, 몬스터의 경우 타입) 안에서 아래 순서로 배치합니다.
    - 순서 : 보상 수령 가능 > 수집 완료 > 미획득 수집 항목 > 미발견 수집 항목
      * 동일 상태에는 ID순 정렬
      * 모두 획득했다면 ID순 (기본 정렬)
    - 정렬 : 도감 정렬은 자동 정렬만 제공하며, 유저가 임의로 변경할 수 없다.
