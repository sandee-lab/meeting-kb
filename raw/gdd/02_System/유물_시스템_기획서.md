# 유물 시스템

> **요약:** 인게임 보상으로 유물 조각을 수집하고 강화하여 계정 전체에 영구 능력치를 적용하는 아웃게임 성장 시스템. 등급 없이 강화만 존재하며 유물 분해/삭제 불가. 향후 초월 유물로 확장 가능한 구조로 장기 수집 목표를 제공한다.
> **현재 상태:** 미적용

## 2. 달성 목표

### 플레이어 경험 목표
- 인게임 클리어 보상으로 유물 조각을 획득하며 **꾸준한 성장 체감** 제공
- 유물을 수집하고 각각 강화하여 키워가는 **장기 수집 목표** 부여
- 균등 확률 드롭으로 **핵심 유물을 빠르게 해금**하고, 강화로 장기 성장
- 중복 획득이 강화 재료가 되어 **어떤 드롭이든 가치 있음**을 느끼게 함

### BM 목표
- 기본은 **게임 플레이를 통한 획득** (인게임 보상)
- **이벤트 패키지, 핫딜** 등을 통해 핵심 유물 조각을 판매하는 것을 고려
- 유물 조각 1개 = **3,000원** 가치 기준
- 인게임 플레이 리텐션 강화를 통한 장기 LTV 향상

### 운영/개발 목표
- CSV 데이터 드리븐 구조로 유물 추가/밸런스 조정을 코드 변경 없이 가능하게 설계
- 기존 StatElement + StatSource 시스템에 자연스럽게 통합
- `artifact_type` 필드로 일반/초월 유물을 단일 테이블에서 관리 (향후 확장 대비)

## 3. 수행 방향

### 핵심 설계 원칙
1. **등급 없음, 강화만 존재**: 모든 유물은 등급 구분 없이 동일하게 획득되며, 중복 조각으로 강화 레벨만 올린다.
2. **균등 확률 드롭**: 전 유물이 균등 확률로 드롭되어, 핵심 효과를 얻는 난이도가 후반으로 밀리지 않도록 한다.
3. **보유 효과 즉시 적용**: 유물 획득 즉시 능력치가 계정 전체에 적용. 강화할수록 효과 증폭.
4. **잉여 조각 누적**: 만렙 이후 잉여 조각은 재고에 쌓인다. 향후 초월 유물 업데이트 시 변환 시스템으로 활용.

### 제약 사항 / 금지 사항
- 유물에 등급 구분 없음 (강화 레벨로만 성장)
- 강화 시 새로운 능력치가 추가되지 않음 (기존 능력치의 수치만 상승)
- 유물 분해/삭제 불가 (획득하면 영구 보유)
- 세트 보너스 없음 (개별 유물 강화에만 집중)
- 클라이언트 단독 강화 불가 (서버 검증 필수)
- 강화 비용은 골드만 소모 (다이아 등 추가 재화 없음)
- 런칭 시 조각 교환 시스템 미포함 (잉여 조각은 재고 누적)

### 우선순위

| 우선순위 | 항목 | 이유 |
|---------|------|------|
| P0 (필수) | 유물 해금 / 보유 효과 적용 | 시스템의 핵심 가치 — 획득 즉시 스탯 적용 |
| P0 (필수) | 강화 시스템 | 장기 성장의 핵심 루프 |
| P0 (필수) | 유물 조각 드롭 | 공급 없이 시스템 동작 불가 |
| P1 (중요) | 강화 가능 레드닷 | 유저 편의 및 전환율 |
| P1 (중요) | 유물 정보 UI 팝업 | 유물 상세 확인 및 강화 진입 |
| P2 (선택) | 강화 연출 VFX | 기존 이펙트 재활용으로 대체 가능 |
| P2 (선택) | 이벤트 패키지 판매 | 운영 시점에 별도 기획 |
| 향후 업데이트 | 초월 유물 시스템 | 일반 유물 만렙 후 잉여 조각 활용 경로 |
| 향후 업데이트 | 잉여 조각 변환 시스템 | 초월 유물 업데이트와 함께 오픈 |

## 4. 개요

### 기본 구성
- **유물 종수**: 일반 유물 27종 (런칭)
- **등급 없음**: 모든 유물이 동일한 방식으로 획득
- **강화 시스템**: 중복 조각 + 골드로 강화 (최대 강화 Lv.10)
- **보유 효과**: 유물 보유 즉시 계정 전체에 능력치 적용 (장비/장착 개념 없음)

### 핵심 규칙
- 유물 최초 획득 시 **Lv.1**로 해금 + **최대 효과의 50%** 즉시 적용
- 중복 획득 시 해당 유물 조각 +1
- 조각 + 골드 소모 → 강화 레벨 상승 → 기존 능력치의 **수치만 증가** (새 능력치 추가 없음)
- 능력치 수치는 **GrowArtifactEnchantData에 Lv.0~10 레벨별로 직접 기입** (공식 기반이 아님)
- 강화 재화는 Consumable 타입으로 관리 (fee_type/fee_id/fee_amount/fee_gold_amount)
- 최대 강화(Lv.10) 이후 잉여 조각은 재고에 누적 (향후 초월 유물 변환에 활용)

### 진입 조건 / 해금 조건
- **해금 조건**: 25챕터 클리어
- ContentsUnlockData에 신규 콘텐츠 타입 `Artifact` 추가
- 해금 전에는 장비 메뉴 내 유물 탭이 잠금 상태로 표시
- 유물 조각 드롭 시작 챕터는 StageChapterData의 `reward_item_type_2` 컬럼에 유물 상자를 배치하여 데이터로 관리

### 보상 구조 요약

| 획득 경로 | 보상 | 수량 | 비고 |
|----------|------|------|------|
| 챕터 클리어 / 순찰 보상 | 유물 상자 | 약 1.5개/일 | StageChapterData `reward_item_type_2`로 드롭 관리. 상자 개봉 시 균등 확률로 유물 조각 1개 획득 |
| 보스 모드 | 유물 상자 | 10개/주 | 주간 보상 |
| 이벤트 패키지/핫딜 | 유물 조각 지정 판매 | - | P2 — 운영 시점 별도 기획 |

## 5. 시스템 동작 흐름

### 전체 흐름 다이어그램

```
[25챕터 클리어] → ContentsUnlockData 해금
  → 장비 메뉴 내 유물 탭 활성화
  → StageChapterData reward_item_type_2에 유물 상자 배치

[챕터 클리어 / 순찰 / 보스 모드 보상] → 유물 상자 획득
  → 상자 개봉 → 균등 확률로 유물 조각 1개 결정
    ├─ [미보유 유물] → 유물 해금 (Lv.1, 50% 효과) + 스탯 재계산
    └─ [보유 유물] → 해당 유물 조각 +1

[유물 UI] → 유물 선택 → 정보 팝업
  └─ [강화 가능] → 강화 버튼 활성화
       → 조각 + 골드 소모 → 서버 검증 (artifact_enhance API)
       → 성공 → 레벨업 + 스탯 재계산 + 강화 연출
```

### 단계별 상세 흐름

#### 1단계: 콘텐츠 해금
1. 유저가 25챕터를 클리어한다.
2. ContentsUnlockData에서 `Artifact` 콘텐츠 타입이 해금된다.
3. 장비 메뉴 내 유물 탭이 활성화된다.
4. StageChapterData `reward_item_type_2`에 유물 상자가 배치된 챕터부터 유물 상자가 드롭된다.

#### 2단계: 유물 상자 획득 → 조각 획득
1. 챕터 클리어 / 순찰 / 보스 모드 시 보상으로 유물 상자를 획득한다.
2. 유물 상자 개봉 시, 전체 유물 중 균등 확률로 유물 1종을 결정한다.
3. 해당 유물이 미보유 상태이면 **Lv.1**로 해금 + 보유 효과(최대치의 50%) 즉시 적용한다.
4. 이미 보유 중이면 해당 유물의 조각 수가 +1 증가한다.
5. 서버에서 유저 데이터를 갱신하고 클라이언트에 결과를 반환한다.

#### 3단계: 유물 강화
1. 유물 정보 팝업에서 강화 버튼을 터치한다.
2. 클라이언트에서 조각 수/골드 충분 여부를 사전 검증한다.
3. `artifact_enhance` API를 서버에 요청한다.
4. 서버에서 조건 검증 → 조각/골드 차감 → 강화 레벨 +1 → 스탯 재계산한다.
5. 성공 응답 수신 → 클라이언트 UI 갱신 + 강화 성공 연출 재생한다.

### 예외 처리 / 엣지 케이스

| 상황 | 처리 방식 |
|------|----------|
| 강화 중 네트워크 실패 | WAS 기본 재시도 3회. 실패 시 에러 팝업 표시, 재화 미차감 보장 (서버 트랜잭션) |
| 동일 유물 강화 동시 2회 요청 | 서버에서 중복 요청 차단 (WASPacket index 기반 순차 처리) |
| 조각/골드 부족 상태에서 강화 시도 | 클라이언트 사전 검증으로 버튼 비활성화. 서버에서도 이중 검증 |
| 서버-클라 데이터 불일치 | 서버 데이터를 Single Source of Truth로 사용. 불일치 감지 시 서버 데이터로 동기화 |
| 해금 전 유물 조각 획득 시도 | 25챕터 미클리어 시 유물 조각이 드롭 풀에 포함되지 않으므로 불가 |
| 만렙 유물 조각 계속 드롭 | 조각은 그대로 재고에 쌓임. 향후 초월 유물 업데이트 시 변환 시스템으로 활용 |
| 모든 유물 만렙 달성 후 조각 드롭 | 조각은 계속 쌓임. 초월 업데이트 때까지 재고 누적 |

### 프로그래머 협의 사항

| # | 항목 | 논의 내용 |
|---|------|----------|
| 1 | 스탯 재계산 시점 | 유물 해금/강화 시 StatElement 재계산 트리거 방식 (MessageSender 이벤트?) |
| 2 | 유물 조각 드롭 풀 연동 | 기존 StageChapterData 보상 시스템에 유물 조각 RewardType 추가 방법 |
| 3 | 순찰/보스 모드 보상 연동 | 유물 조각을 기존 보상 시스템에 통합하는 방법 |
| 4 | 신규 스탯 구현 | mul_increase_damage_to_curse 스탯 신규 구현 (확정) |
| 5 | 보상 슬롯 확장 | 기존 보상 테이블(RewardType + reward_id + reward_amount) 구조 재활용. 보상 슬롯 부족 시 컬럼 추가 필요 |

## 6. 데이터시트 구조

### 테이블 목록

| 테이블명 | 역할 | 신규/기존 |
|---------|------|----------|
| GrowArtifactData | 유물 마스터 데이터 (효과 타입/능력치 타입) | **신규** |
| GrowArtifactEnchantData | 강화 레벨별 필요 조각/비용/능력치 증가량 정의 | **신규** |
| ContentsUnlockData | 유물 콘텐츠 해금 조건 추가 (1행) | **기존 확장** |

### `GrowArtifactData` 테이블 (**신규**)

| 필드명 | 타입 | 설명 | 예시 |
|--------|------|------|------|
| id | int | 유물 고유 ID (PK) | 1 |
| artifact_type | ArtifactType | 유물 분류 (Normal / Transcend) | Normal |
| artifact_effect_type | ArtifactEffectType | 유물 효과 처리 방식 | Stat |
| local_name | string | 이름 로컬라이징 키 | artifact_name_1 |
| local_desc | string | 설명 로컬라이징 키 | artifact_desc_1 |
| icon_path | string | 아이콘 리소스 경로 | Unit |
| icon_name | string | 아이콘 파일명 | Unit_Item_1 |
| stat_type | StatType | 적용 스탯 타입 (artifact_effect_type이 Stat일 때 사용) | add_atk |

> `artifact_type`은 런칭 시 모두 `Normal`. 향후 초월 유물 추가 시 `Transcend` 사용.
> `artifact_effect_type`이 `Stat`이면 `stat_type` 필드의 스탯을 직접 적용. 특수 효과가 필요한 경우 새로운 enum 값을 추가하여 개별 로직으로 처리.

### `GrowArtifactEnchantData` 테이블 (**신규**)

> UnitEnchantData와 동일한 구조 패턴: `id` = 마스터 ID, `enchant_level`로 레벨 구분 (복합키), fee 슬롯 방식

#### 필드 정의

| 필드명 | 타입 | 설명 | 예시 |
|--------|------|------|------|
| id | int | 유물 ID (GrowArtifactData.id 참조) | 1 |
| enchant_level | int | 현재 강화 레벨 (0~10) | 0 |
| fee_type_1 | string | 강화 재화 타입 1 | Consumable |
| fee_id_1 | int | 강화 재화 ID 1 | 5001 |
| fee_amount_1 | int | 강화 재화 수량 1 | 1 |
| fee_type_2 | string | 강화 재화 타입 2 | Gold |
| fee_id_2 | int | 강화 재화 ID 2 | 1000000 |
| fee_amount_2 | int | 강화 재화 수량 2 | 2000 |
| stat_value | float | 현재 레벨의 스탯 수치 (유물별 직접 기입) | 0 |

> Lv.0 = 미획득 상태. 현재 레벨 행을 읽어서 `stat_value`는 현재 효과, `fee`는 다음 레벨 진입 비용으로 처리.
> Lv.10(만렙)은 fee 없음 — 최종 stat_value만 존재.
> 유물 조각은 Consumable 타입으로 등록하여 `fee_id_1`로 참조. 골드는 `fee_type_2 = Gold`로 처리.

### 화면별 상세

#### 유물 메인 화면
- **레이아웃:** 스크롤 리스트, 4개씩 1줄 배치 (27개 기준 7줄)
- **주요 요소:**
  - 유물 아이콘 (27종 개별 아이콘)
  - 해금 여부 표시 (미해금 시 실루엣/잠금 표시)
  - 해금 시 현재 강화 레벨 표시
  - 보유 조각 수 / 다음 강화 필요 조각 수
  - 만렙(Lv.10) 시 MAX 뱃지 표시
  - 강화 가능 시 레드닷 표시
- **인터랙션:** 유물 아이콘 터치 → 유물 정보 팝업 오픈

#### 유물 정보 팝업
- **레이아웃:** 중앙 팝업 (OnlyPopup)
- **주요 요소:**
  - 유물 아이콘 (확대) + 이름 + 설명
  - 현재 효과 수치 / 다음 레벨 효과 수치 (비교 표시)
  - 보유 조각 수 / 필요 조각 수
  - 필요 골드 표시
  - 강화 버튼 (조건 충족 시 활성화)
- **인터랙션:**
  - 강화 버튼 터치 → 서버 요청 → 성공 시 레벨업 연출 + 수치 갱신
  - 배경 터치 / X 버튼 → 팝업 닫기

### UX 주의사항
- 강화 성공 연출은 기존 강화 이펙트를 재활용한다 (반짝임 + 수치 변경 애니메이션)
- 레드닷은 강화 가능한 유물이 1개 이상일 때 장비 메뉴 → 유물 탭에 계단식 표시
- 미해금 유물도 아이콘 실루엣으로 보여주어 수집 목표를 시각적으로 제공
- 만렙 유물은 MAX 뱃지 + 골드 테두리 등으로 시각적 성취감 제공

## 향후 업데이트 방향: 초월 유물

> 런칭 시 미구현. 일반 유물 만렙 이후 잉여 조각 활용 경로로 향후 업데이트 예정.

### 초월 시스템 개요
- **초월 유물 (Transcend)**: 일반 유물보다 특수한 효과를 가진 상위 유물
- 효과: 특수 스탯, 택틱 연동, 게임 메커니즘 영향 등
- 출시 시 **3~4종**, 이후 CSV 추가로 확장
- 단일 테이블(`GrowArtifactData`)에서 `artifact_type = Transcend`로 구분

### 초월 조각 획득 경로
- **일반 유물 만렙 → 잉여 조각 1:1 초월 조각 교환** (변환 UI 오픈 시)
- 특정 콘텐츠 / 이벤트 / 과금으로 초월 조각 직접 획득

### 진입 조건
- 아무 일반 유물 1종이라도 만렙 달성 시 초월 조각 변환 가능
- 잉여 조각은 런칭 시부터 재고에 누적되므로, 초월 업데이트 시점에 소급 활용 가능
