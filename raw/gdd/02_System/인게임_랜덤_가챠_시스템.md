# 인게임 랜덤 가챠 시스템

> **요약:** RandomGacha 오브젝트 타입 추가. 확률 그룹에서 랜덤 유닛/택틱/꽝 결과를 뽑는 인게임 시스템.
> **현재 상태:** 미적용

# 목표

* 인게임에서 랜덤 요소를 통해서 유닛을 획득하는 비중을 높이기 위해서 시스템을 추가한다.
* 연출적인 요소를 고려해서 시스템을 설계한다.

# 개요

* 상호작용 오브젝트에 RandomGacha 타입을 추가한다.
* RandomGacha 타입일 때 RandomGachaPoolData에서 가챠 결과를 확률에 따라서 뽑는다.
* 결과는 가챠 연출을 통해서 유저에게 프리젠테이션 한다.

# 시스템 구조

## 기능 연결

* + 상호작용 오브젝트에 신규 결과 타입을 추가한다.
    - Type = Random\_Gacha

## 결과 선택

* + 확률에 따라서 가챠 결과 그룹을 하나 선택한다.

## 그룹 구성

* + 하나의 그룹에는 3개의 결과가 포함된다.
  + 3개의 결과 배치 순서는 각각 랜덤하게 결정한다.

## 결과의 종류

* + 랜덤 유닛 소환(Random\_Spawn\_Unit)
    - UnitData에서 상대 확률에 따라서 하나의 유닛을 선택한다.
    - 선택한 유닛을 소환한다.
    - 선택된 유닛의 아이콘을 참조
  + 꽝(Fail)
    - 아무런 효과가 없다.
    - UI 연출 상으로 꽝의 느낌을 표현해준다.
    - 전용 아이콘을 지정한다.
      * Ingame 아틀라스에 in\_ic\_gacha\_fail 추가 필요
  + 택틱 (Tactic\_Draw)
    - pool\_id를 지정한다.
    - 지정한 pool 에서 하나의 택틱을 랜덤하게 선택한다.
    - 선택된 택틱의 아이콘을 참조
  + 아이템 드랍 (Drop\_Item)
    - 아이템 하나를 지정해서 드랍한다.
    - 선택된 아이템의 아이콘을 참조

## 중복 결과 확률 보정

* + 동일한 결과 타입이 중복으로 선택되었을 때, 세부 id 값을 결정하는 확률을 보정하여, 잭팟 결과의 빈도를 컨트롤할 수 있도록 구현한다.
    - 선택된 결과 그룹의 결과 타입이 다음과 같은 경우

| result\_type\_1 | Random\_Spawn\_Unit |
| --- | --- |
| result\_type\_2 | Random\_Spawn\_Unit |
| result\_type\_3 | Random\_Spawn\_Unit |

* + - result\_type\_1 의 세부 값을 확률에 따라서 선택한다.
    - result\_type\_2 의 세부 값을 constantsdata@stagemode\_random\_gacha\_result\_rate\_by\_second 값에 따라서 result\_type\_1 과 같은 값으로 적용한다.
    - result\_type\_2 확률 성공 판단 시
      * result\_type\_3 의 세부 값을 constantsdata@stagemode\_random\_gacha\_result\_rate\_by\_second 값에 따라서 result\_type\_1 과 같은 값으로 적용한다.

| 두 번째 결과의 보정 확률 | constantsdata@stagemode\_random\_gacha\_result\_rate\_by\_second | 0.5 |
| --- | --- | --- |
| 세 번째 결과의 보정 확률 | constantsdata@stagemode\_random\_gacha\_result\_rate\_by\_third | 0.3 |

## 연출의 종류

* + 3개의 슬롯에 선택된 Random\_Spawn\_Unit 이벤트의 갯수에 따라서 다음과 같이 연출 단계를 구분한다.

| 연출 단계 | 조건 | 예시 |
| --- | --- | --- |
| 0 | Random\_Spawn\_Unit x 0 | Bad |
| 1 | Random\_Spawn\_Unit x 1 | Good |
| 2 | Random\_Spawn\_Unit x 2 | Great |
| 3 | Random\_Spawn\_Unit x 3 | Perfect! |
| 4 | Random\_Spawn\_Unit x 3  세부 값이 모두 같을 때, | JackPot! |

## 보너스 이벤트

* + 조건
    - 3개의 슬롯에 결과가 모두 동일하게 선택된 경우라면, 보너스 이벤트가 발동된다.
    - 동일한 타입안에서도 id 값 까지 동일하게 선택되어야 한다.
  + 결과
    - 보너스 이벤트 결과는 추후 기획 검토한다.

# 필요 데이터시트

## ConstantsData

* + stagemode\_random\_gacha\_result\_rate\_by\_second
    - 두 번째 슬롯이 1번 슬롯과 같은 결과일 확률
  + stagemode\_random\_gacha\_result\_rate\_by\_third
    - 세 번째 슬롯이 1번 슬롯과 같은 결과일 확률

  + select\_rate
    - 결과 그룹의 선택 확률
    - 상대 확률로 판단한다.
  + result\_type\_1
    - 1번 슬롯의 결과 타입
  + result\_vaule\_1
    - 1번 슬롯의 세부 결과 값
  + result\_type\_2
    - 2번 슬롯의 결과 타입
  + result\_vaule\_2
    - 2번 슬롯의 세부 결과 값
  + result\_type\_3
    - 3번 슬롯의 결과 타입
  + result\_vaule\_3
    - 3번 슬롯의 세부 결과 값
