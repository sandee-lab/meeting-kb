# 폴리싱 - 프로필 시스템 기획

> **요약:** 유저 프로필 네임카드 및 커스터마이징 시스템 UX 설계. 스티커/배경 수집과 랭킹 연동 프로필 표시를 정의한다.
> **현재 상태:** 검토 중

## **프로필**

![](data:image/png;base64...)

## **1. 기획 의도 및 방향성**

### **1.1. 기획 목표**

* **비주얼 임팩트:** 텍스트 정보를 최소화하고 아트워크(배경+스티커)를 강조하여 유저의 개성을 시각적으로 즉시 전달.
* **정보의 계층화:** 보여주기 위한 정보(네임카드)와 확인하기 위한 정보(상세 진행도)를 명확히 분리하여 UI 혼잡도 해결.
* **수집과 창작:** 유저가 자유롭게 배치하는 커스터마이징 요소를 제공.

### **1.2. 목표 달성을 위한 방향**

* **아트 중심 레이아웃:** 네임카드 내에는 식별에 필수적인 정보(닉네임, 길드 이름)만 남기고 나머지 텍스트는 모두 제외.
* **최적화된 노출 구조:** 리스트에서는 가벼운 썸네일로, 랭킹 및 상세 팝업에서는 화려한 풀 렌더링으로 노출하여 클라이언트 성능 부하 방지.

## **2. 프로필 구조**

### **2.1. 네임카드 영역**

* *유저가 꾸미는 그래픽 영역입니다.*
* **필수 텍스트:** 닉네임, 길드 이름(자리 확보).
* **시각 요소:** 명함 배경, 스티커

### **2.2. 상세 진행도**

* *프로필 팝업에 표시되는 상세 진행도 정보입니다. (네임카드에는 표시되지 않음)*
* **공용**
  + **계정 정보:** 레벨, 경험치, 전투력
  + **성장 정보:**
    - 장비 : 착용 중인 장비
    - (아웃게임이 늘어나면 추가될 수 있음)
  + **기록 정보:**
    - 선착순 랭킹 (마지막 챕터 기준)
      * 랭킹 등재 시 덱 정보
      * 등수
    - 무한 모드 랭킹
      * 랭킹 등재 시 덱 정보
      * 등수
* **선택**
  + **내 프로필 보기**
    - 꾸미기
      * 전체 스티커 목록 : 내가 보유하지 않은 것도 표시
      * 전체 배경 목록 : 내가 보유하지 않은 것도 표시

## **3. 커스터마이징 기능**

### **3.1. 커스터마이징 요소**

| **요소** | **설명** | **주요 획득처 (예상)** |
| --- | --- | --- |
| **배경** | 네임카드의 캔버스. 전체 분위기 결정. | 배틀패스, 상점, 이벤트 |
| **스티커** | 자유 배치 가능한 스티커형 트로피. | 업적, 랭킹, 수집 보상 |

### **3.2. 커스터마이징 규칙**

* **공통**
  + 목록
    - 내가 보유한 요소만 목록에 나열한다.
  + 획득
    - 기존 조건들을 사용할 수 있다.
      * Get\_Consumable 조건 추가 필요
        + 특정 아이템 획득 시 해제 되는 조건

커마 요소를 보상으로 배치하기 위함

보상으로 배치되는 요소는 아이콘 제작이 필요

잠금 조건 표시 시에는 “[아이템] 획득” 대신 개별 로컬로 연결해서 표시한다. (예: 이벤트 참여 보상)

* + - * + 예외 처리

중복 획득 시 별도 처리 안 함

* + 적용
    - 보유한 요소만 프로필에 사용할 수 있다.
* **배경**
  + **고정 배치:** 배경은 캔버스 내 지정된 좌표에 배치.
  + **변형 불가**
  + **레이어 우선순위:** 배경 < 스티커 < 텍스트(닉네임, 길드 이름).
  + **제한:** 반드시 1개의 배경이 지정되어야 한다.
* **스티커**
* **배치:**
  + 목록에서 배치할 슬롯을 터치해서 상단 프로필에 반영합니다.
* **선택 상태:**
  + 상단 프로필에서 마지막에 배치(선택)된 스티커는 선택 상태로 처리됩니다.
  + 선택 상태의 스티커를 이동 및 변형할 수 있습니다.
* **변형할 스티커 선택:**
  + 목록에서 배치된 스티커 슬롯을 다시 터치하면, 상단 프로필에서 해당 스티커가 선택 상태로 변경됩니다.
* **삭제:** 선택 상태의 스티커가 있는 상태에서 ‘삭제’버튼을 누르면 해당 스티커가 삭제됩니다.
* **자유 배치:** 획득한 스티커를 캔버스 내 원하는 좌표에 배치.
  + *기본 위치 정 중앙*
* **변형 가능:** 이동, 회전, 확대/축소 지원
* **레이어 우선순위:** 배경 < 스티커 < 텍스트(닉네임, 길드 이름).
  + *텍스트 가림 방지:* 닉네임 등 중요 정보가 뱃지에 가려지지 않도록 최상단 고정.
* **스티커 레이어 우선순위:**
  + 마지막으로 터치한 스티커 = 최상단
  + 선택한 스티커에 1~10 번 부여하며, 마지막으로 터치한 스티커가 1번을 가지도록 한다.
* **제한:** 최대 10개의 스티커 부착 가능
  + 11번째 스티커 선택 시 토스트 메시지 출력
    - “스티커는 10개까지 배치할 수 있습니다.”
* **되돌리기**
  + 확인 팝업 출력
  + 변경된 내역을 취소하고, 마지막 저장 상태로 돌아간다.
* **저장하지 않고 나가기**
  + 확인 팝업 출력
  + 나가기 시 모든 변경 사항 원상복귀하고 팝업 종료

## **4. 프로필 노출 규칙**

### **4.1. 소형 썸네일**

메인 화면 상단, 그리고 친구 목록, 길드원 목록과 같이 다수의 유저가 나열되는 UI입니다.

* **표시 형태:** 소형 육각형
* **구성:** 네임카드 중앙 마스크
* **액션:** 아이콘 터치 시 [프로필 팝업] 호출.

### **4.2. 대형 썸네일**

랭킹과 같이 유저의 프로필을 강조하는 영역에 사용되는 UI입니다.

* **표시 형태:** 대형 사각형 (660\*170)
* **구성:** 네임카드 전체 영역을 커버하는 사이즈의 마스크
* **액션:** 아이콘 터치 시 [프로필 팝업] 호출.

### **4.3. 프로필 팝업**

썸네일을 터치하여 진입하는 팝업 UI입니다.

* **프로필 표시 형태:** 마스크 없이 풀렌더링
* **구성:**
  1. 상단: 완성된 네임카드 (배경+스티커+닉네임+길드 이름)
  2. 하단: 플레이어 상세 진행도.
* **기능 제한:**
  1. 내 프로필을 열었을 때 [꾸미기/수정] 버튼 노출
  2. 타인의 정보를 열람할 때는 [꾸미기/수정] 버튼은 노출되지 않음.

## **5.레드닷(N 마크)**

  + 메인 화면 상단 프로필 아이콘
  + 꾸미기 탭
