## 2025년 1월

> 한줄 요약: Alpha 1~2 빌드의 폴리싱과 플레이 테스트를 거쳐 마일스톤 2를 마감하고, Alpha Test v2 방향 및 마일스톤 3 계획을 수립한 달.

### 주요 논의
- (01/03) Alpha 1 폴리싱 리스트 — 버그 수정, 머지 연출 개선, 튜토리얼 구현, 광고 카드 수량 보정 등 — [[raw/meetings/[0103] 폴리싱 리스트 Alpha 1]]
- (01/09) 마일스톤 2 플레이 테스트 공지 — 전사 대상 안드로이드 APK 테스트, 플레이/결제 지표 확인 목적 — [[raw/meetings/[0109] 마일스톤 2 플레이 테스트]]
- (01/09) Alpha 1 빌드 피드백 — 아트 스타일 긍정 평가, 코어에 유저 운 개입 요소 필요, 전장 가시성 개선 필요, 밸런스 초기 단계 — [[raw/meetings/[0109] 빌드 피드백 Alpha 1]]
- (01/10) Alpha 2 폴리싱 리스트 — 웨이브별 콜로니 배치 변경, 밸런스 대폭 조정(생산속도, 승급, 광폭화), 광폭화 게이지 UI, 인트로 단축, 사운드 추가 — [[raw/meetings/[0110] 폴리싱 리스트 Alpha 2]]
- (01/13) 추가 컨텐츠 메모 — 랜덤 카드(스킬카드) 시스템, 4성 머지 유닛(지휘관), 연구 포인트, 이벤트(서버 협동, 라이벌 랭킹) 기획 — [[raw/meetings/[0113] 추가 컨텐츠 메모]]
- (01/17) 마일스톤 3 제작회의 — 기간 2/3~2/28(4주), 핵심 목표: 코어플레이 재미 검증, 캐릭터 소환 방식 변경(랜덤 소환 + 보너스 룰렛), 로그라이크 요소(전략 카드) 도입, 신규 유닛/맵 리소스 — [[raw/meetings/[0117] 마일스톤 3 제작회의]]
- (01/20) Alpha Test v2 방향 — CTR/리텐션 개선 목표, 밸런스 기준 전투 템포 정립, 유닛 조합(탱/딜/힐), 스킬 기능 추가, 카드 순서 시스템, 영상 테스트 계획 — [[raw/meetings/[0120] alpha test v2]]

### 핵심 변화
- 마일스톤 2 종료 및 Alpha 빌드 플레이 테스트 완료
- 코어플레이에 유저의 운 개입 요소 추가 필요성 확인 (스킬카드 시스템 기획)
- 유닛 생산 쿨타임 1초 이하 통일, 비행기-유닛 위치 조정으로 전장 가시성 개선
- 광폭화 게이지 UI 도입
- 밸런스 방향: 승급 시 부스팅 효과 강화, 기본 성급 파워 하향으로 전투 리듬 부여
- 마일스톤 3 계획 확정 (2/3~2/28, 코어플레이 재미 검증)
- 장기 컨텐츠 로드맵 수립: 랜덤 카드, 4성 유닛, 연구 포인트, 이벤트 시스템

## 2025년 2월

> 한줄 요약: 운빨 기반 코어 디자인 철학을 정립하고, 지원사업 문서를 통해 게임의 경제 모델과 컨텐츠 구조를 체계화한 달.

### 주요 논의
- (02/08) 운빨 메모 — 신화 유닛 획득 사이클이 핵심 경험, 리스크와 운의 비율 조합(POE 아이템 제작 참고), 소과금(3,000~10,000원) BM 구조, 협동 플레이 부재 시 대안(상대경쟁), 선술집 시스템 — [[raw/meetings/[0208] 운빨 메모]]
- (02/25) 메모 — 보스 넉백 시스템 검토, 유닛 대쉬(버프 시 전열 진입), 템포 개선 고민 — [[raw/meetings/[0225] 메모]]
- (02/28) 지원사업 문서_아웃게임 — 경제 모델 시뮬레이션(Machinations), 행동력/골드/다이아 자원 구조, 랭킹/이벤트 보상 체계 — [[raw/meetings/[0228] 지원사업 문서_아웃게임]]
- (02/28) 지원사업 문서_컨텐츠 — 게임 특징 정의(서먼 배틀 + 로그라이크 + 럭가챠), 인구수(업킵) 시스템, 확률 구조(기본50%/희귀35%/전설12%/신화3%), 전략 카드, 선술집, 종족 시너지, 경쟁형 스테이지 모드 — [[raw/meetings/[0228] 지원사업 문서_컨텐츠]]

### 핵심 변화
- 게임 코어 디자인 철학 정립: "운빨"을 중심으로 하되 빌드업 과정에서 재미를 창출
- 소과금 BM 모델 방향 확정 (3,000~10,000원대)
- 난이도 평탄화 + 변수 로테이션으로 스테이지 기반 경험 다양화 방침
- 지원사업 문서를 통해 경제 모델과 전체 컨텐츠 구조 공식화
- 인구수(업킵) 시스템, 럭가챠 확률 구조, 전략 카드 시스템 상세 설계 완료
- 경쟁형 스테이지 모드(5인 매칭, 24시간 블럭 클리어) 기획

## 2025년 3월

> 한줄 요약: 기존 서먼 배틀에서 캐릭터 직접 조작 방식(오토 슈터)으로 코어플레이를 전환하며 신규 마일스톤 1을 시작한 달.

### 주요 논의
- (03/05) 신규 마일스톤 M1 — 기간 3/5~3/21(2주), 캐릭터 조작형으로 전환(모바일 조작계), 웨이브 100단계 클리어, 택틱 드로우(레벨업 시 선택), 히든 택틱(궁극기/시너지 스킬) — [[raw/meetings/[0305] 신규 마일스톤[m1]]]
- (03/05) 신규 마일스톤 M3 계획 — 기간 5/7~5/30(3주+), 유닛 속성/시너지 택틱, 특수 소환 테스트, 인게임 UI 개선, 아웃게임(스토리/챕터 구조/골드 성장), 6월 이후 로드맵(6~8월 신규개발, 9월 폴리싱, 10월 QA, 11월 검수) — [[raw/meetings/[0305] 신규 마일스톤[m3]]]
- (03월) M1 로그라이트 버전 기획 — 단일 영웅 캐릭터 직접 조작(탕탕특공대/외계인배고파 스타일), 유닛은 소환 형태로 유지, 미션 기믹(벙커 지키기/해처리 파괴/수송선 호위), 히든 택틱(계열 궁극기), 맵 오브젝트(브레이커블/건물) — [[raw/meetings/[2503]M1_미니크래프트_로그라이트버전]]
- (03월) 캐릭터 변신 시스템 상세 기획 — Random_Change_Unit(랜덤 변신), Random_Select_Unit(등급 이상 선택), 게임 시작 시 랜덤 유닛 결정, 히든 변신 하이라이트, 무빙샷 처리, 아이템 시스템 — [[raw/meetings/[2503]M1_캐릭터 중심으로 플레이 완성하기]]

### 핵심 변화
- 코어플레이 대전환: 서먼 배틀 → 캐릭터 직접 조작 + 오토 슈터(Vampire Survivors 라이크) 방향
- 택틱 드로우 시스템 도입 (레벨업 시 평타 강화/스킬 획득/유닛 소환/버프 선택)
- 히든 택틱 개념 도입 (계열 궁극기, 부대 전투 강화, 단일 유닛 중복 강화)
- **캐릭터 변신 시스템** 도입: 운에 의해 캐릭터가 변신하며 플레이 스타일 전환
- 장기 출시 로드맵 수립: 6~8월 신규개발 → 9월 폴리싱 → 10월 QA → 11월 검수

## 2025년 4월 (M2 마일스톤)

> 한줄 요약: 멀티 유닛 컨트롤과 유닛 각성 시스템을 도입하여 기존 단일 캐릭터 방식에서 부대 단위 전투로 설계를 발전시킨 달.

### 주요 논의
- (04월) M2 방향성 전환 — 부대 단위 멀티 유닛 컨트롤, 유닛 각성(3종 선택지), 속성 시너지(불/전기/얼음), 기본 유닛 4종 확정(파이어뱃·칼냥이·총냥이·메딕) — [[raw/meetings/M2_4월 마일스톤]]

### 핵심 변화
- **멀티 유닛 컨트롤** 도입: 단일 유닛 → 부대 단위 조작으로 전환
- **유닛 각성 시스템**: 최종 등급 도달 시 3종 선택지 중 하나로 각성(완전히 다른 유닛으로 변신)
- 등급 성장: 일반→고급→희귀→영웅 (외형 변화 없음) → 각성 시 외형 변경
- 랜덤 소환과 머지를 통한 덱빌딩 난이도 조절 메커니즘 정립
- 기존 단일 캐릭터 변신 방식 대비 차별화 확인 (선택 강제 → 항상 전투력 상승)
- 속성 시너지용 추가 피해/받는 피해 증가 스탯 체계 도입

## 2025년 5월 (M3 마일스톤)

> 한줄 요약: 덱빌딩 로그라이크 플레이 루프를 구체화하고 6월 이후 mass 단계 전환을 준비한 달.

### 주요 논의
- (05월) M3 목표 설정 — 인게임 R&D(덱빌딩 로그라이크 플레이 루프 구체화), 6월 mass 단계 준비(아웃게임 큰그림·컨텐츠·BM·리소스 계획), 인게임 전투(캐릭터 전투 기획 구현, 스탯/스킬 시스템 보완), 상호작용 오브젝트 추가, 기본 아웃게임 성장 구현(스테이지·골드) — [[raw/meetings/M3_5월 마일스톤]]

### 핵심 변화
- 전체 출시 로드맵 확정: 6·7·8월 mass 개발 → 9월 폴리싱 → 10월 QA → 11월 출시
- 개별 캐릭터 전투 컨셉 R&D 시작 (조합 컨셉 샘플링 목표)
- **상호작용 오브젝트** 신규 시스템 도입: 택틱 드로우·유닛 소환·아이템 획득 등
- 기본 아웃게임 성장 구조 구현 착수 (스테이지 모드 + 골드 성장)

## 2025년 6월 (M4 마일스톤)

> 한줄 요약: M4 인게임 플레이 검증 목표 하에 유닛 스킬·택틱·세트 효과를 정리하고, 지휘관/장비 아웃게임 시스템 개발을 착수한 달. 오토 슈터 빌드의 핵심 문제점도 확인.

### 주요 논의
- (06/25) 메모 — 유닛 승급 정보 표시 문제(택틱에서 선택의 의미 부족), 유닛 조합 방식 재검토(직접 컨트롤과 조합식의 불편함), 기본 유닛 공격 방식 다양화 필요, 몬스터 패턴/간격 개선, 연출 추가(승급/초월/보스/아이템), 스타크래프트 감성 강화, 인게임 목표 시스템 필요 — [[raw/meetings/[0625] 메모]]
- (06월) M4 마일스톤 — 인게임 플레이 검증(유닛 스킬·택틱·세트 효과·오브젝트 정리, 인게임 UI), 5월 작업 폴리싱(스테이지·행동력·골드 성장), 지휘관·장비 아웃게임 개발 착수 — [[raw/meetings/M4_6월 마일스톤]]

### 핵심 변화
- M4 목표: 6월에 "게임의 재미를 판단할 수 있는 수준"의 인게임 완성
- 유닛 승급 시 전략적 선택의 의미가 불명확한 문제 식별
- 기본 유닛 간 공격 방식 차별화 요구 (방향성 있는 공격: 포탄→장판, 칼냥이→좌우 강타 등)
- 몬스터 패턴의 단조로움 지적 (무리 패턴 변화, 템포 변화 필요)
- **지휘관·장비 아웃게임 시스템** 개발 준비 시작
- 인게임 목표 시스템 제안 (부대 수 확장 조건, 중간 마일스톤, 지휘관 승급 등)

## 2025년 7월 (M5 마일스톤)

> 한줄 요약: M5 아웃게임 성장 시스템 테스트와 함께 프로젝트 방향성을 오토 슈터 + 부대 시너지로 재정립하고, 플레이 테스트 설문을 통해 핵심 피드백을 수집한 달.

### 주요 논의

### 핵심 변화
- 프로젝트 최종 방향 확정: 오토 슈터(Vampire Survivors-like) + 부대 시너지 + 웨이브/기지 정비 반복 구조
- 핵심 디자인 철학 3가지 정립: 무작위성+의미 있는 선택, 조합 시너지(스타크래프트 감성), 밀도 높은 긴장감
- 부대 시너지 시스템 상세 설계 (바이오닉/메카닉 종족 시너지 + 개별 유닛 시너지)
- 기지 테크트리 도입 (일반→레어→하이브, 리스크&리턴 전략)
- M5 플레이 테스트 설문(11명) 통해 초월유닛 조합의 재미 확인, 동시에 유닛 합성 피로도와 복잡함이 주요 문제로 부각
- **택틱 상점** 구현 완료 (인게임 재화로 택틱 구매)
- **유닛 재뽑기(리롤)** 기능 추가
- Add_Skill 택틱 시스템 완성: 지휘관·랜덤 유닛·특정 유닛에게 스킬 부여 가능
- 재화 체계 정리: 레벨업 삭제, 미네랄(유닛 소환)/가스(택틱 구매)/오브젝트(슬롯머신 대체)
- 웨이브 클리어 조건 변경: 버티기 → 특정 마릿수 처치 + 시간 제한
- UI 비주얼 방향 확정: Retro Futurism (CRT, 기계식 사운드, 낮은 채도 + 선명한 포인트 컬러)
