# 무한 모드 개발 명세서 (공허의 균열)

> **요약:** 공허의 균열 무한 모드의 상세 개발 명세서로, 던전 로비 UI, 도전 탭, 난이도 시스템(레벨 1~10), 달성 보상 정책, 시즌 버프, 자동 클리어(스킵) 시스템, 리그 랭킹(일간/시즌 이원화), 시즌 결산, 교환소 등 아웃게임 전반의 구현 스펙을 정의한다. 입장 티켓 충전 규칙, 전투력 기반 소탕 권유 팝업, 보상 수령 UX 등을 상세히 기술한다.
> **현재 상태:** 미적용

# **공허의 균열 (무한 모드) 상세 기획서**

# **1. 콘텐츠 개요**

## **1.1. 콘텐츠 정의**

* **콘텐츠 명:** 공허의 균열
* **장르:** 고밀도 무한 서바이벌
* **한 줄 요약:** 끊임없이 몰려오는 적들을 물리치고, 전장의 오브젝트를 전략적으로 활용해 시간을 구매하며 한계까지 생존하는 타임 어택형 경쟁 콘텐츠.

## **1.2. 기획 의도**

**① 끊김 없는 전투 경험**

* 기존 스테이지 모드의 웨이브 대기 시간, 페이즈 전환 로딩 등 흐름을 끊는 요소를 모두 제거했습니다.
* 오직 '전투'와 '이동'에만 집중할 수 있는 긴 호흡의 전장을 제공하여, 유저가 플레이 자체에 온전히 몰입하도록 유도합니다.

**② 극한의 성능 검증 및 경쟁**

* 유저가 아웃게임에서 쌓아온 성장 결과물(유닛, 장비, 컨트롤)을 가감 없이 쏟아부을 수 있는 **'최고의 성능 전시장'** 역할을 수행합니다.
* 단순 클리어가 아닌 "누가 더 오래 버티는가"라는 원초적인 경쟁 심리를 자극하여 최상위권 유저들의 성장 욕구를 지속적으로 고취합니다.

**③ 전략적 선택과 시간의 농도**

* 제자리에서 적만 잡는 단조로움을 탈피하기 위해, '이동'을 강제하는 전략 오브젝트(노드)를 배치했습니다.
* **Risk & Return:** 안전한 구역에서 방어할 것인가, 아니면 위험을 무릅쓰고 적진 한복판의 오브젝트를 점령하여 강력한 버프를 얻을 것인가? 매 순간 실시간 판단을 요구합니다.

**④ 장기적인 도전 가치**

* 1주일 단위의 **시즌제 랭킹**과 **리그 승급 시스템**을 도입하여 엔드 콘텐츠로서의 수명을 확보합니다.
* 명예 보상(전체 랭킹)과 실속 보상(리그/달성 보상)을 이원화하여, 코어 유저와 라이트 유저 모두에게 확실한 플레이 동기를 부여합니다.

유저가 도전할 난이도를 선택하고, 시즌 정보를 확인하여 전략을 수립하는 메인 화면입니다.

![](data:image/png;base64...)

### **3.1.1. 콘텐츠 명과 정보**

* 균열 이름 - “공허의 균열” 텍스트 키: Local\_Contents\_Unlock\_InfiniteMode
* 3분 보상에 공허의 균열 티켓 보상으로 지정

### **3.1.2 시즌 남은 시간**

* 시간 정보 - 시즌끝나기 까지 남은 시간 출력

### **3.1.3 시즌 버프**

매주 변경되는 전장 규칙을 통해 특정 유닛/속성의 활용을 유도합니다.

* **데이터 관리:** InfiniteModeSeasonBuffData 테이블 참조.
* **적용 규칙:** 시즌(주) 단위로 buff\_type에 따라 아군 유닛 전체에 버프 적용.
  + 예시: 이번 시즌 [화염] 속성 유닛 공격력 +20%.
* **UI 표시:** [도전] 탭 상단 배너 영역에 아이콘과 설명 텍스트 노출.
* 다른 균열 콘텐츠에서는 사용 안할 수 있습니다.

### **3.1.4 권장 전투력**

* **전투력 표기 및 색상 코딩:**
  + 유저 전투력과 권장 전투력을 비교하여 난이도를 직관적으로 표시합니다.
  + **안전 (Green):** 내 전투력 ≥ 권장 전투력
  + **위험 (Red):** 내 전투력 < 권장 전투력 (경고 아이콘 표시)
  + 빨강 > 주황 > 하양 > 연두 > 초록
  + [전투력 단계 별 난이도 보정 시스템](https://docs.google.com/document/d/1suECm21kZVPT_l1IOeFCgEcxS2PiojiAP7I8H70WTXY/edit?tab=t.0)

  + 던전 쪽 전투력 기준으로 변경하는 것
    - 여기에서도 난이도 보정 적용하는 것

### **3.1.5 덱 정보**

* 기존 던전과 같지만, 스파인으로 표현 (상위 콘텐츠 느낌)

  + 달성 보상 수령 가능할 때,

### **3.1.7 난이도 선택 리스트**

* **단계 구성:** **Level 1 ~ Level 10** (총 10단계).
  + InfiniteModeStageData@stage\_step 참조
  + 스테이지가 더 추가되면 레벨 10에서 더 확장될 수 있는 구조
  + 유저 성장도에 따라 몬스터 스펙을 더 잘게 나눌 필요 있음
* **해금 조건:** 이전 단계 **3분 이상 생존**시 다음 단계 잠금 해제.
  + *시즌 초기화:* 새 시즌 시작 시 모두 잠김 상태로 초기화 (Lv 1부터 다시 도전).

### **1. 달성 보상 정책**

스테이지 내 생존 시간(1분/2분/3분) 달성 시 획득 권한이 주어지는 보상입니다.

* **초기화:** 매주 시즌 시작 시(월요일 00:00 UTC) 초기화되어 다시 획득 가능.
* **지급 방식:** 수동 수령
  + 게임 종료 즉시 인벤토리에 꽂아주지 않고, [도전 대기 화면]에서 유저가 보상 아이콘을 터치해야 지급됩니다.
  + *이유:* 로비로 돌아와서 성취감을 다시 한번 느끼게 하고, 재방문율을 높이기 위함.

### **2. 게임 종료 후 동선**

게임 플레이가 끝나면 메인 로비가 아닌, **[공허의 균열 도전 대기 화면]**으로 복귀하여 연속적인 플레이 흐름을 유지합니다.

1. **게임 오버/클리어:** 결과 화면 출력.
2. **확인 버튼 터치:** 결과 화면 닫힘.
3. **화면 이동:** 도전 대기 화면으로 이동.
   * ***수정 전:*** 메인 로비로 이동.
   * ***수정 후:*** 공허의 균열 UI로 즉시 복귀

### **3. UI 진입 시 자동 포커스**

화면에 진입했을 때, 어떤 난이도(Level X) 페이지를 먼저 보여줄 것인가에 대한 규칙입니다.

#### **A. 우선순위 판정 로직**

시스템은 아래 순서대로 체크하여 포커스를 맞출 페이지를 결정합니다.

* **1순위 (최우선): 보상 수령 대기 상태**
  + **보상을 받을 스테이지가 있는가?**
  + 그렇다면, 그중 가장 높은 단계의 페이지를 보여줌.
    - *(예시: 3단계와 4단계에 안 받은 보상이 있다면 → 4단계 페이지 포커스)*
  + *기획 의도:* 방금 플레이한 곳이 4단계라면, 5단계가 열렸더라도 4단계 보상을 먼저 받도록 유도.
* **2순위 (차선): 모든 보상을 수령한 경우**
  + 수령할 보상이 하나도 없는가?
  + 그렇다면, 현재 도전 가능한 가장 높은 단계의 페이지를 보여줌.
  + *(예시: 4단계까지 3분 클리어하고 보상도 다 받음 → 5단계 페이지 포커스)*

#### **B. 예외 케이스 시나리오**

상황: 유저가 Lv.4를 플레이하여 3분 생존(Lv.5 해금)에 성공했으나, 아직 Lv.4 보상을 터치해서 받지 않음.

1. 게임 종료 후 UI 복귀.
2. 시스템 체크: "Lv.4에 받을 보상이 있는가?" → Yes.
3. 결과: Lv.5(다음 단계)가 아닌, Lv.4(방금 깬 단계) 페이지가 표시됨.
4. 유저가 보상을 모두 터치해서 수령 완료.
5. UI를 닫았다가 다시 들어오면? → 받을 보상이 없으므로 Lv.5(최고 단계)가 표시됨

### **4. 보상 수령 인터랙션**

#### A. 보상 아이콘 상태값 (State)

각 시간(1m/2m/3m) 별 상자 아이콘은 3가지 상태를 가집니다.

| 상태 | 조건 | UI 표현 | 인터랙션 |
| --- | --- | --- | --- |
| 잠김 (Locked) | 생존 시간 미달성 | 🔒 자물쇠 아이콘 + 흑백 처리 | 없음 |
| 수령 가능 (Claimable) | 달성 완료 AND 미수령 | 🎁 상자 흔들림(Ani) + ✨ 반짝임(Effect) | 터치 시 보상 지급 (수령 처리) |
| 수령 완료 (Claimed) | 달성 완료 AND 수령 완료 | ✅ 체크 표시 + 어둡게 처리(Dimmed) | 터치 시 반응 없음 |

#### **B. 수령 시 연출**

1. 유저가 [수령 가능] 상태의 상자를 터치.
2. 획득 팝업 UI 출력.
   * 받을 수 있는 모든 달성 보상을 받음 (일괄 처리)
   * 아이템 아이콘, 이름, 수량 표시.
   * 배경 터치 시 팝업 닫힘.
3. 해당 상자 아이콘이 [수령 완료] 상태로 변경됨.
4. [레드 닷] 갱신: 해당 페이지의 모든 보상을 받았다면, 탭이나 버튼에 붙어있던 레드닷 제거.
5. 달성 보상을 받았다면, 현재 입장 가능한 스테이지로 페이지를 갱신함

## **4.5. 정산 및 예외 처리**

### **4.5.1. 정산 스케줄**

* **시즌 기간:** 월요일 00:00 ~ 일요일 23:30 (7일). (GMT)
* **초기화:** 매주 월요일 00:00.
  + 모든 유저의 '단계별 달성 보상' 기록 초기화.
  + 랭킹 점수 초기화 및 티어 재배치

### **4.5.2. 신규 서버 보정**

오픈 초기 유저 형평성을 위해 첫 시즌 기간을 유동적으로 운영합니다.

* **규칙:** 서버 오픈일로부터 첫 정산일(월요일)까지의 기간이 **7일 미만**일 경우, 시즌을 연장합니다.
* **예시:**
  + *목요일 오픈:* 목~일(4일) + 다음주 월~일(7일) = **총 11일간 첫 시즌 진행**.
  + *이후:* 정상적인 1주일 단위 시즌 합류.

### **4.5.3. 동점자 처리**

* **규칙:** 동일 점수 발생 시, 해당 점수를 **먼저 달성한(Timestamp가 빠른)** 유저가 상위 순위를 차지합니다.

### **🚨 [공허의 균열] 레드닷(Red Dot) 통합 로직**

**대원칙: 하위 요소에 레드닷이 하나라도 켜지면, 상위(부모) UI에도 무조건 켜집니다.**

1. 뎁스 구조도

* [최상위] 탐사 입장 버튼 🔴
  + ㄴ [하단 탭] 랭킹 🔴
    - ㄴ [버튼] 축하하기 (시즌 보상) 🔴
  + ㄴ [하단 탭] 교환소 🔴
    - ㄴ [상품] 무료 상품 🔴
  + ㄴ [하단 탭] 도전 (메인) 🔴
    - ㄴ [아이콘] 달성 보상 상자 🔴
    - ㄴ [버튼] 입장/시작 🔴

2. 상세 출력 조건

A. 랭킹 탭

* 대상: '축하하기' 버튼 & 하단 랭킹 탭
* **ON 조건:**
  + 지난 시즌 결산이 완료되어 수령 가능한 시즌 보상이 있을 때. (시즌당 1회)
* **OFF 조건:**
  + 보상을 수령했을 때.

B. 교환소 탭

* 대상: 무료 상품 슬롯 & 하단 교환소 탭
* **ON 조건:**
  + 무료 상품의 구매 가능 횟수가 남아있을 때. (Stock > 0)
* **OFF 조건:**
  + 무료 상품을 구매(수령)하여 품절되었을 때.

C. 도전 탭

* 대상: 하단 도전 탭
* **ON 조건 (OR 조건):**
  + 달성 보상을 안 받은 게 있거나 (Unclaimed\_Reward > 0)
  + 입장 티켓이 있을 때 (Ticket\_Count >= 1)
  + *(상세)* 입장 버튼 레드닷: 티켓이 1장 이상일 때 켜짐.
  + *(상세)* 보상 상자 레드닷: 해당 상자의 목표를 달성했고, 아직 안 받았을 때 켜짐.

D. 최상위 입장 버튼 (메인 로비의 '탐사' 아이콘)

* **ON 조건:** 위 3개 탭(랭킹, 교환소, 도전) 중 하나라도 레드닷이 켜져 있으면 켜짐**.**

### **1. 인게임 종료 후 처리 프로세스**

* 1단계: 결과 화면
  + 게임 오버 또는 클리어 시 결과 팝업 출력.
  + 획득한 점수 및 보상 표시.
* 2단계: 확인 터치
  + 유저가 결과창의 [확인] 버튼 터치.
* 3단계: 씬 이동
  + [변경 전]: 메인 로비로 이동.
  + [변경 후]: 공허의 균열 도전 대기 화면으로 즉시 복귀.
* 4단계: 랭킹 갱신 연출
  + 방금 플레이로 최고 기록을 경신했다면, 랭킹 슬라이딩 연출 출력.
  + *내 프로필 바가 순위 목록을 밀어내며 올라가는 애니메이션.*
* 5단계: 페이지 포커스 (Page Focus)
  + 아래 '2. 디폴트 페이지 로직'에 따라 적절한 난이도(Lv) 페이지를 자동으로 펼쳐줌.

### **2. 디폴트 페이지 및 자동 포커스 로직**

공허의 균열 내부 진입 시 (도전 대기 화면)

화면에 진입하자마자 [도전] 탭의 [난이도 선택 리스트]가 펼쳐집니다. 이때 어떤 난이도(Lv 1~10)를 보여줄지 결정하는 우선순위 로직입니다.

* **1순위 (최우선): 보상 수령이 필요한 경우**
  + 조건: 전체 난이도 중, **달성 보상을 안 받은 스테이지**가 있는가?
  + 동작: **미수령 보상이 있는 스테이지 중 가장 높은 단계(Level) 페이지**를 보여줌.
  + *의도: 방금 깬 단계의 보상을 까먹지 않고 받도록 유도 (성취감 강화).*
* **2순위 (차선): 모든 보상을 받은 경우**
  + 조건: 받을 보상이 하나도 없는가?
  + 동작: 현재 **도전 가능한 가장 높은 단계** 페이지를 보여줌.
  + *의도: 유저가 바로 다음 목표에 도전할 수 있도록 편의성 제공.*

[예시 시나리오]

1. 유저가 Lv.4에 도전하여 3분 생존(Lv.5 해금) 성공.
2. 게임 종료 후 대기 화면 복귀.
3. Lv.5가 해금되었지만, 아직 Lv.4 달성 보상을 안 받은 상태.
4. 시스템은 Lv.4 페이지를 먼저 보여줌 (보상 상자가 반짝임).
5. 유저가 보상을 수령함.
6. 받을 게 없으므로 Lv.5 페이지가 보임.
