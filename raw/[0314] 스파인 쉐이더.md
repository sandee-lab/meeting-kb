    ## Spine 전용 구현 방식 (중요)
    
    Spine은 유니티 기본 `SpriteRenderer`가 아닌 자체 `SkeletonAnimation` 컴포넌트를 쓰기 때문에, 셰이더를 직접 두 번 그리는 방식이 안 됩니다. 대신 **자식 오브젝트에 두 번째 SkeletonAnimation을 추가**하는 방식을 사용합니다.
    
    **유닛 프리팹 구조:**
    
    ```
    UnitPrefab
    ├── SkeletonAnimation (본체)   ← Shader A 적용 (스텐실 기록용)
    └── HighlightLayer (자식)
        └── SkeletonAnimation      ← Shader B 적용 (겹침 표시용)
                                      (비활성화 상태로 두다가, 하이라이트 시 활성화)
    ```
    
    자식 SkeletonAnimation은 부모와 동일한 `SkeletonDataAsset`을 공유하고, 매 프레임 스크립트로 애니메이션 상태를 동기화합니다.
    
    ---
    
    ## Shader A — 모든 유닛에 적용 (스텐실 기록)
    
    Spine의 기존 `Spine/Skeleton` 셰이더를 열어서 `SubShader` 블록 안에 아래를 추가합니다.
    
    ```
    // Spine/Skeleton 셰이더에 이 블록 추가
    SubShader {
        Tags { ... }
    
        // ← 이 부분을 Pass 위에 추가
        Stencil {
            Ref 1
            Comp Always      // 항상 통과
            Pass Replace     // 통과한 픽셀에 Ref값(1) 기록
        }
    
        Pass {
            // 기존 Spine 셰이더 내용 유지
        }
    }
    ```
    
    ---
    
    ## Shader B-1 — 방법 A: 단색 실루엣
    
    겹치는 영역을 완전 불투명한 단색으로 채웁니다. 새 셰이더 파일로 만드세요.
    
    ```
    Shader "Custom/SpineStencilSilhouette" {
        Properties {
            _Color ("Silhouette Color", Color) = (1, 0.2, 0.2, 1)
        }
        SubShader {
            Tags { "Queue"="Transparent+1" "RenderType"="Transparent" }
    
            Stencil {
                Ref 1
                Comp Equal       // 스텐실 값이 1인 곳(다른 유닛이 있는 곳)에서만 통과
                Pass Keep
            }
    
            Blend SrcAlpha OneMinusSrcAlpha
            ZWrite Off
    
            Pass {
                CGPROGRAM
                #pragma vertex vert
                #pragma fragment frag
                #include "UnityCG.cginc"
    
                fixed4 _Color;
    
                struct appdata { float4 vertex : POSITION; float2 uv : TEXCOORD0; };
                struct v2f { float4 pos : SV_POSITION; float2 uv : TEXCOORD0; };
    
                v2f vert(appdata v) {
                    v2f o;
                    o.pos = UnityObjectToClipPos(v.vertex);
                    o.uv = v.uv;
                    return o;
                }
    
                // Spine 텍스처의 알파를 마스크로만 사용, 색은 _Color로 고정
                sampler2D _MainTex;
                fixed4 frag(v2f i) : SV_Target {
                    fixed4 tex = tex2D(_MainTex, i.uv);
                    return fixed4(_Color.rgb, tex.a * _Color.a);
                }
                ENDCG
            }
        }
    }
    ```
    
    ---
    
    ## Shader B-2 — 방법 B: 반투명 오버레이
    
    겹치는 영역에 반투명한 색을 입힙니다. B-1에서 딱 한 줄만 다릅니다.
    
    ```
    Shader "Custom/SpineStencilOverlay" {
        Properties {
            _Color ("Overlay Color", Color) = (1, 0.2, 0.2, 0.5)  // ← Alpha 0.5로 반투명
        }
        SubShader {
            Tags { "Queue"="Transparent+1" "RenderType"="Transparent" }
    
            Stencil {
                Ref 1
                Comp Equal
                Pass Keep
            }
    
            Blend SrcAlpha OneMinusSrcAlpha
            ZWrite Off
    
            Pass {
                CGPROGRAM
                #pragma vertex vert
                #pragma fragment frag
                #include "UnityCG.cginc"
    
                fixed4 _Color;
                sampler2D _MainTex;
    
                struct appdata { float4 vertex : POSITION; float2 uv : TEXCOORD0; };
                struct v2f { float4 pos : SV_POSITION; float2 uv : TEXCOORD0; };
    
                v2f vert(appdata v) {
                    v2f o;
                    o.pos = UnityObjectToClipPos(v.vertex);
                    o.uv = v.uv;
                    return o;
                }
    
                fixed4 frag(v2f i) : SV_Target {
                    fixed4 tex = tex2D(_MainTex, i.uv);
                    // 원본 색상 위에 _Color를 반투명하게 덮어씌움
                    return fixed4(_Color.rgb, tex.a * _Color.a);
                }
                ENDCG
            }
        }
    }
    ```
    
    실루엣과 오버레이의 차이는 `_Color`의 Alpha 값 하나입니다. `(r, g, b, 1.0)`이면 단색 실루엣, `(r, g, b, 0.5)`이면 반투명 오버레이입니다.
    
    ---
    
    ## Sorting Layer 설정 (필수)
    
    렌더 순서가 틀리면 효과가 완전히 깨집니다.
    
    | 오브젝트 | Sorting Layer | Order in Layer |
    | --- | --- | --- |
    | 모든 유닛 본체 (Shader A) | Units | 0 |
    | 하이라이트 레이어 (Shader B) | Units | 1 |
    
    하이라이트 레이어가 반드시 본체보다 나중에 그려져야 합니다.

    ## URP 사용 중이라면
    
    위 셰이더는 Built-in 렌더 파이프라인 기준입니다. 프로젝트가 **URP(Universal Render Pipeline)**를 쓰고 있다면 셰이더 문법이 달라지므로, 개발팀에 "URP 쓰냐 Built-in 쓰냐"를 먼저 확인하고 전달하시는 게 좋습니다. Spine URP 패키지도 별도로 임포트되어 있어야 합니다.
    
- Grok
    
    **네, 가능합니다.** Spine으로 만든 Unity 프로젝트에서 같은 유닛이 겹칠 때 **하나의 유닛만 하이라이트**하고, **겹친 영역만 스텐실 효과처럼 특정 색으로 비치게** 하는 건 스텐실 버퍼 + Spine 커스텀 셰이더 조합으로 충분히 구현할 수 있어요.
    
    Spine은 기본적으로 투명도(Alpha Blending) 기반 2D 렌더링을 쓰고 ZWrite가 Off라서 겹침 처리가 까다롭지만, 공식 Outline 셰이더 + 스텐실 + MaterialPropertyBlock을 쓰면 딱 맞아요. 아래에 가장 실용적인 방법 두 가지를 정리했어요 (URP 기준, Built-in도 거의 동일).
    
    ### 1. 가장 간단하고 추천하는 방법: Spine 내장 Outline + Stencil + 선택 로직
    
    Spine 런타임에 이미 **Spine/Outline/OutlineOnly-ZWrite** 셰이더가 있어서 overlapping attachments(같은 스켈레톤 안 부분 겹침)도 잘 처리해줍니다. 다른 유닛 간 겹침은 이걸 확장하면 됩니다.
    
    **구현 단계**
    
    1. **하나의 유닛만 선택하는 로직 (C#)**
        
        ```csharp
        // 예: Update나 LateUpdate에서 호출
        void UpdateHighlight()
        {
            var colliders = Physics2D.OverlapBoxAll(myUnit.transform.position, size, 0); // 또는 Bounds intersect
            SkeletonRenderer selected = null;
            float maxY = float.MinValue;
        
            foreach (var col in colliders)
            {
                var renderer = col.GetComponent<SkeletonRenderer>();
                if (renderer && renderer.transform.position.y > maxY)
                {
                    maxY = renderer.transform.position.y;
                    selected = renderer;
                }
            }
        
            // 모든 유닛 outline 끄기
            foreach (var unit in allUnits) unit.SetOutlineColor(Color.clear); // 또는 property block
        
            if (selected != null)
                selected.SetOutlineColor(highlightColor); // 아래 PropertyBlock 사용
        }
        ```
        
    2. **MaterialPropertyBlock로 하나만 하이라이트 적용** (배칭 깨지지 않음)
        
        ```csharp
        private MaterialPropertyBlock _block;
        public void SetOutlineColor(Color color)
        {
            if (_block == null) _block = new MaterialPropertyBlock();
            _block.SetColor("_OutlineColor", color);           // Spine Outline 셰이더 프로퍼티
            _block.SetFloat("_OutlineWidth", 5f);             // 원하는 두께
            GetComponent<SkeletonRenderer>().SetPropertyBlock(_block);
        }
        ```
        
    3. **겹친 영역만 특정 색으로 비치게 (Stencil 효과)**
        - Spine/Outline 셰이더를 복사해서 커스텀 셰이더 만들기 (Spine 패키지 안에 있음).
        - Pass에 Stencil 블록 추가:
            
            ```
            Stencil {
                Ref 1                     // 겹침 감지용 Ref
                Comp Always
                Pass Replace              // 이 유닛이 그린 픽셀은 Ref=1로 표시
                ZFail Keep
            }
            ```
            
        - 두 번째 Pass (또는 별도 Material)에서:
            
            ```
            Stencil {
                Ref 1
                Comp NotEqual             // 이미 다른 유닛이 그린 영역(겹침)만 통과
                Pass Keep
            }
            // 여기서 특정 색으로 렌더 (예: _OverlapColor = Color.red)
            half4 frag() : SV_Target {
                return _OverlapColor;     // 겹친 부분만 이 색으로 비침
            }
            ```
            
        - **렌더 순서 중요**: 비하이라이트 유닛 먼저 렌더링 → 하이라이트 유닛 나중에 (Sorting Group이나 Order in Layer로 제어).
    
    이렇게 하면 **선택된 유닛만 풀 하이라이트** 되고, **겹친 픽셀만 스텐실로 특정 색** (예: 빨강/노랑 테두리나 반투명 tint)으로 비치면서 자연스럽게 보입니다.
    
    ### 2. 더 고급: Overlapping Highlight 방지 + 단일 색 처리 (hyunkell 스타일)
    
    여러 유닛이 동시에 하이라이트될 때 색이 additive로 쌓이는 걸 완전히 막고 싶다면 (또는 “하나만” 규칙을 셰이더 레벨에서 강제), **Stencil + ID Texture** 기법을 쓰세요.
    
    - Render Texture 하나 만들어서 모든 유닛의 “ID”를 미리 기록.
    - Stencil Pass 0: Ref 1, Always Replace → 픽셀 마킹.
    - Pass 1: Ref 1, Equal + IncrWrap → 같은 픽셀은 두 번 안 그리게 함.
    - 결과: 겹친 영역은 **무조건 하나의 특정 색**만 나옴 (additive 안 됨).
    
    Spine에 적용할 때는 SkeletonRendererCustomMaterials나 RenderCombinedMesh 컴포넌트로 전체 스켈레톤을 하나의 메쉬처럼 취급하면 더 깔끔합니다.
    
    ### 추가 팁 (실제 프로젝트에서 자주 쓰는 것들)
    
    - **URP 추천**: Built-in보다 Renderer Feature로 CommandBuffer 넣기 쉽습니다. (Post-process처럼 전체 하이라이트 패스 하나로 처리 가능)
    - **ZWrite 문제**: OutlineOnly-ZWrite 셰이더 쓰면 같은 유닛 내부 겹침(팔/다리 등)도 제대로 occlusion 됩니다.
    - **성능**: MaterialPropertyBlock 쓰면 인스턴싱 유지돼요. 100마리 유닛도 문제없음.
    - **테스트**: Spine Examples → Outline Shaders 예제 씬 열어보고 시작하세요. 거기서 ZWrite 옵션 보고 바로 이해할 수 있어요.
    
    이 방법으로 안 되면 (예: URP 버전이나 Spine 버전 알려주시면) 더 정확한 셰이더 코드 스니펫 드릴게요.
