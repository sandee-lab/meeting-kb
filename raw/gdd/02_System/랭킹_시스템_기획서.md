# 스테이지 랭킹

> **요약:** 주간 전체 랭킹(상위 100명)과 일간 조별 랭킹(50명 단위 그룹)으로 구성된 스테이지 랭킹 시스템이다. 다이아~브론즈 5개 리그로 편성하며, 7일 단위로 조를 재편성한다. 실시간 기록 집계 후 30분 전 스냅샷 기준으로 공개하고, 신기록 달성 시 실시간 순위 변동 연출을 제공한다. 랭킹 보상은 우편함으로 지급되며, 미접속 시 최근 회차 보상만 제공된다.
> **현재 상태:** 미적용

**스테이지 랭킹**

# 개요

* 랭킹 시스템을 구성해 유저 간 경쟁심을 유발하고 플레이 지속 동기를 강화한다.
* 소프트 런칭에는 [스테이지 랭킹]만 도입되며, 이는 아래 두 종류로 구성된다
  + 주간 전체 랭킹
    - 7일마다 갱신되며, 전체 채널 유저에서 상위 100위가 집계된다.
  + 일간 조별 랭킹
    - 매일 전날 성적 기준으로 갱신되며, 7일간 유지되는 조의 유저가 포함된다.
* 랭킹은 서버에서 실시간으로 집계되며, 유저가 신기록을 달성할 경우 실시간 순위 변동에 따른 연출이 출력된다.
* 공개 랭킹은 주간 및 일간 별로 기준 시점의 데이터를 기반으로 고정되어 표시된다.
* 각 랭킹에는 별도의 보상이 주어진다.

# 랭킹 운영 정책

## 랭킹 집계

* + 채널 내 모든 유저의 기록은 서버에서 실시간으로 저장 및 정렬된다.
  + 유저가 콘텐츠에서 신기록을 달성할 경우, 해당 기록은 즉시 서버에 반영된다.

## 공개 기준

* + 공개 랭킹은 고정된 시점의 스냅샷 데이터를 기준으로 구성된다.
  + 집계 시점은 공개 시간 기준 30분 전으로 고정되며, 해당 시점 이후의 기록은 반영되지 않는다.

## 공개 시간

* + 주간 전체 랭킹
    - 매주 서버 갱신일 오전 9시 공개 → 오전 8:30 기준으로 집계
  + 일간 조별 랭킹
    - 매일 오전 9시 공개 → 오전 8:30 기준으로 집계
  + 실제 운영 시 시간은 서버 상태에 따라 조정될 수 있다.

## 실시간 피드백

* + 유저가 콘텐츠에서 신기록을 세우면, 현재 조 내에서의 순위 변동 여부를 실시간 연출로 피드백한다.
  + 단, 실시간 순위 변동은 공개 랭킹에 즉시 반영되지 않는다.

## 등재 및 삭제 조건

* + 등재 조건
    - 각 랭킹 콘텐츠별로 최소 참여 기준을 설정한다.
      * 예: 챕터 랭킹은 챕터 2 클리어 시 등재
      * ConstantDatas에서 관리한다.
        + rank\_enroll\_required\_chapter
  + 삭제 조건
    - 제재 대상 유저는 랭킹에서 제외 및 삭제한다.

# 그룹 시스템

## 개요

* + 7일 단위로 주기적 개편되는 채널 내 경쟁 단위

## 목적

* + 중상위 유저에게도 도전 가능한 경쟁 환경 제공
  + 전체 랭킹 독점 구조 방지, 상대적 목표 제시
  + 소규모 그룹 경쟁을 통해 랭킹 참여율 제고
  + 그룹 주기적 개편을 통한 콘텐츠 수명 확보

## 리그 구성

### 편성안

* + - 상/하위 구간 명확하게 구분할 수 있도록 함
    - 실제 게임 인원에 따라 기준은 변경될 수 있음
      * 소수점 버림

| **리그명 (예시)** | **편성 기준** | **비고** |
| --- | --- | --- |
| 다이아 리그 | 1- 50 | 최상위 실력자 그룹 |
| 플래티넘 리그 | 51-100 | 중상위권 경쟁 그룹 |
| 골드 리그 | 101~ 상위 30~60% 유저 | 중간권 그룹 |
| 실버 리그 | 상위 60~90% 유저 | 중하위권 그룹 |
| 브론즈 리그 | 하위 10% 유저 | 초보자 / 복귀 유저 포함 |

### 조 인원 제한

* + - 조 당 인원 : 50명
    - 각 리그 별로 조 개수는 달라지게 된다.
    - 조는 7일 단위 유지, 7일 후 재편성을 기본으로 한다.
      * 주간 전체 랭킹 갱신 시점에 재편성
      * 편성된 조에서 제재 유저, 미접속 유저 발생 시 인원 미달인 채로 유지한다.

## 유저 편성 기준

* + 서비스 오픈 후 초기 편성 (신규 유저 기준)
    - 계정 생성 순서대로 50명 묶어 그룹 편성
    - 정기 편성 이전까지 유지
  + 정기 편성 (매주)
    - 전체 유저의 게임 진행도 기준으로 채널 전체를 5개 리그로 구분
    - 각 리그 내부에서 접속 순으로 그룹 생성
  + 신규/복귀 유저\* 예외 처리
    - 주기 중간 유입은 대기 그룹 편성
      * 유입 시점 따라 7일보다 짧게 유지될 수 있음

| \*복귀 유저 : 미접속 7일. 최종 배치된 그룹 경쟁 진행하지 않은 유저 (해당 그룹에서 기록 없음)  월요일기준으로 다음주 월요일 까지 접속 하지 않은 사람 기준  최소 일주일 이상 미접속 유저가 대상 |
| --- |

## 설정

* + 리그 및 조 명칭은 SF 콘셉트에 따라 '함대', '연합', '섹터' 등의 테마를 가져갈 수 있다.

## 참고

* + 유저는 본인이 속한 그룹과 리그 정보를 UI에서 확인할 수 있다. (일간 조별 랭킹)
  + 그룹 내 랭킹은 매일 1회, 전날 기록 기준으로 갱신된다.
  + 그룹 구성은 서버에서 처리하며, 유저 간 변경/이동 불가

# 랭킹 종류

## 채널 랭킹 (로우데이터)

* + 개요: 채널 내 유저 데이터를 실시간 집계하여 랭킹 산정의 기반이 되는 데이터
  + 용도: 실시간 순위 피드백, 조 편성, 실력 측정 등에 활용됨
  + 갱신: 실시간

## 일간 조별 랭킹 (前 라이벌 랭킹)

* + 구성: 동일 리그의 유저 중 50명 단위로 편성된 그룹(조) 기준
  + 갱신: 매일 오전 9시 (전일 기준)
  + 공개: 각 조별 상위 50명(전체) 게시
  + 목적: 소규모 경쟁으로 현실적인 목표 제시 및 유저 동기 부여
  + 주기: 조는 7일 간 유지 후 재편성됨

## 주간 전체 랭킹 (前 명예의 전당)

* + 대상: 채널 전체 유저 중 상위권
  + 갱신: 매주 서버 갱신일 기준 (예: 월요일 9시)
  + 공개: 상위 100명 표시
  + 목적: 전반적인 실력 랭킹 제공 및 명예성 부여

## 도전(경쟁)모드 랭킹 (추후 업데이트 예정)

* + 구조: 소규모 조 단위 경쟁 → 승급/유지/강등 시스템 도입
  + 특징: 랭킹 결과에 따라 보상 및 리그 이동 가능
  + 구성: 조 인원 수는 일반 조보다 적을 수 있음
  + 갱신 및 순위 기준: 추후 정책 정의 예정

# 순위 표기 방식

## 최대 순위 이내

* + 랭킹 등재 조건을 만족하고, 최대 순위(예: 상위 100위, 50위 등) 내에 포함된 유저
  + 표기 방식: 1위부터 최대 순위까지 개별 순위 숫자 표시 (예: 1위, 2위, ..., 50위)

## 미진입

* + 랭킹 등재 조건 자체를 만족하지 못한 유저 (예: 챕터 2 미클리어, 기록 없음 등)
  + 표기 방식: "미진입" Local\_Rank\_Not\_Applicable

## 순위 밖

* + 랭킹 등재 조건은 만족했으나, 최대 표시 순위를 초과한 유저
  + 표기 방식: "순위 밖" Local\_Rank\_Not\_Listed

# 랭킹 보상 정책

## 지급 방식

* + 랭킹 갱신 시점 이후, 유저가 접속하면 우편함으로 보상 지급
  + 유저는 직접 우편에서 보상 수령

## 미수령 보상 처리

* + 유저가 접속하지 않고 보상을 수령하지 않은 경우, 해당 회차 보상은 소멸
  + 항상 가장 최근 갱신 회차의 보상만 제공됨 (=접속 보상)
    - 예시
      * 25일에 일간 조별 랭킹 1위 기록
      * 26일 미접속, 일간 조별 랭킹 3위 기록
      * 27일 접속 시, [조별 랭킹 3위] 보상만 제공
      * 일간 랭킹은 다음 날에 수령해야 함.

## 보상 기준 및 예시

* + 콘텐츠마다 보상 지급 여부를 설정할 수 있다.
  + 기본 구조

## 보상 예시

| **구간** | **보상 예시** |
| --- | --- |
| 1위 | 상징 보상 + 희귀 재화 |
| 2~10위 | 중간 재화 |
| 11~100위 | 소량 재화 |

## 지급 시점

* + 우편 지급 시점: 각 랭킹 갱신 직후
    - 제목 텍스트 : {0:랭킹 이름} 보상 지급
    - 내용 텍스트 : 보상을 확인해 주세요.

## 보상 가치 결정 기준

* + 일간 랭킹 등은 매일 지급되므로, 보상 가치를 과도하게 높이지 않도록 조정 필요하다.

## 실시간 순위 상승 연출

* + 목적
    - 콘텐츠 플레이 결과에 따라 유저의 실시간 순위 상승 여부를 직관적으로 피드백

  + 적용 콘텐츠 및 타이밍

**리그 예상 문제점**

1. 조 편성 단위 불일치

* 리그별 인원이 50명으로 딱 나누어지지 않는 경우 (마지막 조가 1~49명)

2. 전체 유저 수 감소 시 불균형

* 특정 리그에서 조 편성 불가 상황 발생하는 경우.
* 고정 퍼센트 기준으로 리그 분포 → % 작은 리그 인구가 지나치게 작아질 수 있음

**해결 방안**

1. 인접 인원 흡수 [추가 규칙으로 작업 필요해보임]

* 남은 인원이 25명 미만 → 하위 리그로 이동
* 남은 인원이 25명 이상 → 상위 리그로 이동
* 예시

| 리그 | 인원 | 조 편성 결과 | 조 편성 보정 후 |
| --- | --- | --- | --- |
| 다이아 | 101명 | 50/50/1 | 1명 → 플래티넘 리그로 이동 |
| 플래티넘 | 501명 | 50×10 + 1명 | 다이아에서 1명 받아 50×10 + 2명 → 마지막 2명 → 골드로 이동 |

2. 리그 경계 조정 [추가 규칙으로 작업 필요해보임]

* 고정 퍼센트 대신 조 단위로 나누어떨어지게 경계 조정
  + 전체 유저 N, 조 크기 M(50) → 목표 비율에 따라 예상 인원 계산 → 조 단위로 반올림 → 합계 불일치 시 중앙 리그(골드)에서 조정
  + 예시 (N=9800):
    - 다이아 10% → 980 → 1000
    - 플래티넘 20% → 1960 → 1950
    - 골드 30% → 2940 → 2950
    - 실버 30% → 2940 → 2950
    - 브론즈 10% → 980 → 950
  + 합계 조정 예시 (N=9800):
    - 합계 = 9802명 → +2명이 초과
      * → 골드 리그에 2명 빼서 조정 → 합계 = 9800명 맞춤
    - 합계 = 9798명 → -2명 부족
      * → 골드 리그에 2명 추가 → 합계 = 9800명 맞춤

3. 리그 인원 최소 기준 + 병합 [서비스 후 선택 가능]

* 최소 인원 = 100명(2조 이상) [고정]
* 특정 리그 인원이 최소 이하 → 상하위 리그 병합
* UI 상 리그명 유지, 내부 데이터만 통합
* 예시

| 리그 | 기존 인원 | 병합 조건 충족 여부 | 결과 |
| --- | --- | --- | --- |
| 다이아 | 43명 | 100명 미만 | 플래티넘과 병합 |
| 플래티넘 | 310명 | 병합 대상 포함 | 다이아 + 플래티넘 → 상위 리그 그룹으로 통합 |
| 골드 이하 | 유지 | - | 유지 |
