## 개요

미니크래프트의 UI/UX 디자인 방향은 레트로 퓨처리즘을 기반으로, 밝은 톤과 캐주얼한 버튼 스타일을 유지하면서 시각적 폴리싱을 지속적으로 진행해왔다. 초기 UI 폴리싱(2025.07)부터 소프트런칭 폴리싱(2026.03)까지, 유저 피드백을 반영하여 시인성 개선, 연출 최적화, 전환 딜레이 축소 등이 주요 과제로 반복 등장하였다. 특히 유닛 가독성 문제는 3개 빌드 연속 최다 지적 사항으로, 게임의 코어 병목으로 인식되었다.

## 상세 내용

### 디자인 언어 및 아트 디렉션

#### 레트로 퓨처리즘 컨셉 (2025.07)
- CRT 모니터의 화면 왜곡, 스캔라인, 노이즈 효과 적용
- 전체적으로 채도가 낮은 색상을 기본으로 하되, 레인보우 컬러 등으로 색상 대비를 포인트로 활용
- 투박한 기계식 또는 초기 디지털 폰트 사용으로 시대적 느낌 전달
- 버튼과 메뉴를 물리적인 스위치나 다이얼 모양으로 제작
- 버튼 클릭 시 기계식 사운드, 노이즈 등 아날로그 효과음 활용
- 키 컬러: 종이 베이스에 블랙 계열 포인트, 선명한 컬러 포인트
- 아웃라인 컬러 가급적 통일

#### 디자인 기본 원칙 (2025.08)
- 기존 디자인 언어 유지: 밝은 톤 베이스, 버튼 및 팝업 창에서 라운드와 볼록한 형태
- 캐주얼 방향: 큰 버튼, 밝은 톤과 선명한 컬러
- 게임에 어울리는 디자인 코드: 종이 박스, 스타크래프트(SF), 레트로
- 출시 전 최소 2회의 디자인 폴리싱이 필요하다고 판단

#### 폰트 규칙
- 아웃라인이 있는 경우 자간 값을 적게, 아웃라인이 없는 경우 값을 크게
- 한국어, 영어 폰트에 따라 다르게 적용할 수 있도록 시스템 지원
- 강조 폰트 컬러 룰 정리 필요 (고유명사, 강화 스탯 등)

### 제작 프로세스 변경 (2025.08)

- **제작툴 피그마로 변경**: XD는 향후 관리 이슈가 크기 때문에 피그마로 전환 결정
  - XD에서 작업된 이미지들을 점진적으로 피그마로 전환 예정
  - 당분간 Free 버전 사용 (Ingame / Outgame / Shop 영역 구분)
- **이미지 최적화 방식 변경**: 버튼 및 등급 배경 등을 만들어진 이미지로 모두 변경, 컬러를 지정해서 조립하는 방식 지양
  - 디자인이 픽스되지 않은 상태에서 조립 형태 사용 시 디자인 변경에 너무 많은 시간이 소요되는 리스크
- **담당자 배분**:
  - 현재 제작된 중요 페이지 폴리싱 및 디자인 방향성 정리 — 정해룡
  - 신규 콘텐츠 레이아웃 (상점, 던전 등) — 김수현
  - 드로잉 이미지 및 아이콘 — 김수현, 정승언

### UI 폴리싱 대상 화면 (2025.07)

골드 성장, 메인 화면, 프로필, 순찰, 공용 보상창, 기록, 7일 이벤트, 퀘스트, 출석부, 파츠, 지휘관, 종료창 등 전체 UI 화면에 대한 폴리싱이 진행되었다.

### 시인성 개선 — 핵심 반복 과제

시인성 문제는 M9(2025.11)부터 소프트런칭(2026.03)까지 3개 빌드 연속 최다 지적 사항이었다.

#### M9 폴리싱 (2025.11)
- SCV 피격 시 스크린 이펙트 추가, 낮은 HP 상태 표기
- 대미지 폰트 속도 조정 (더 빠르게 사라지도록)
- 골드 및 미네랄 획득 연출 간소화
- 적군 프로젝타일 컬러 통일, 장판 통일
- 강조 폰트 컬러 룰 정리 필요
- 아이콘 전면 정비 (종합 파워, 탐사, 시즌패스, 7일 미션, 설정, 하단 아이콘 전체)
- 상점 패키지 로드 시 진입 렉 해결

#### 2월 폴리싱 리스트 (2026.01)
- 지휘관 강조: 아웃라인 오브젝트 추가로 최상위 렌더링
- 지휘관 주변 콜라이더로 여유 공간 확보
- 지휘관 피격 시 카메라 쉐이크 약화 + 30% 이하 스크린 이펙트
- 전체 유닛 범위 동적 바닥 표시
- 이펙트 끄기 / 유닛 반투명도 옵션 제공
- 설치형 스킬 아군/적군 컬러 구분
- 몬스터 사망 이펙트 과다 제거
- 맵 내 불필요 오브젝트 제거
- 경험치 보석 획득 리듬감 개선 (10/30 단위 묶음 처리, 이징 적용)
- 재화 획득 연출 변경 (화면 날아가는 연출 삭제, 위치 표기로 변경)

#### 소프트런칭 폴리싱 리스트 (2026.03)
- **[P1]** 지휘관 아웃라인 오브젝트 추가 — 근거 5/5 (B2에서 "1웨이브 끝날 때까지 중심 캐릭터인지 몰랐다"는 피드백 대응)
- **[P0]** 지휘관 콜라이더 확대 — 근거 4/5
- **[P0]** 지휘관 피격 연출 개선 — 근거 4/5
- **[P0]** 전체 유닛 범위 동적 표현 — 근거 4/5
- **[P1]** 이펙트 끄기 / 유닛 반투명도 옵션 — 근거 4/5
- **[P1]** 몬스터 표현 약화로 아군 유닛 시각 강조 — 근거 3/5
- **[P0]** 몬스터 사망 이펙트 과다 제거 — 근거 3/5
- **[P1]** 맵 내 불필요 오브젝트 제거 — 근거 3/5 (B2에서 "노란 꽃이 재화로 인식된다"는 피드백)

### 연출 및 전환 딜레이 최적화

연출 딜레이 축소는 Alpha 2(2025.01)부터 소프트런칭(2026.03)까지 지속적으로 요구된 사항이다.

#### Alpha 1 폴리싱 (2025.01)
- 동전 올라가는 애니메이션 속도 전체적으로 20~30% 빠르게
- 머지 연출에 이펙트 및 연출 추가 (유닛 아이콘 등록, 유닛마다 대사 차별화)
- 유닛 피격 시 플래시 효과 추가
- 튜토리얼 앞 부분 구현

#### Alpha 2 폴리싱 (2025.01)
- 인트로 단축 (스킵 기능 추가, 카메라 이동 시간 축소)
- 인게임 플로우 디테일 정리 (UI 등장 후 0.5초 대기, 대사 타이밍 조정)
- 광폭화 게이지 및 워닝 연출 추가
- 카메라 쉐이크 (기지 파괴, 레이지 모드 시작 시)
- 사운드 채우기, 내 기지 폭파 연출 이펙트 추가

#### 소프트런칭 폴리싱 (2026.03)
- **[P0]** 레벨업 카드 선택 시 렉 현상 수정 — 근거 4/5
- **[P1]** 시작 전 연출 길이 짧게
- **[P1]** 레벨업 슬로우 최소화 (이펙트는 정상속도, 진입 직전에만 짧게 슬로우)
- **[P1]** 웨이브 시작 연출 바로 나오게
- **[P1]** 미네랄 채집 속도 지수형으로 변경
- **[P1]** 게임 종료 후 보상 받기까지 시간 줄이기

### 스파인 쉐이더 — 유닛 겹침 시각 처리 (2026.03)

유닛 시인성 문제 해결을 위해 Spine 전용 스텐실 쉐이더 구현 방안이 정리되었다.

- **구조**: 유닛 프리팹에 본체 SkeletonAnimation(Shader A, 스텐실 기록용) + 자식 HighlightLayer(Shader B, 겹침 표시용)
- **Shader A**: 모든 유닛에 적용하여 스텐실 값 기록 (Ref 1, Always, Replace)
- **Shader B-1**: 단색 실루엣 방식 — 겹치는 영역을 불투명한 단색으로 표시
- **Shader B-2**: 반투명 오버레이 방식 — 겹치는 영역에 반투명 색상 적용
- Sorting Layer 설정 필수 (하이라이트 레이어가 본체보다 나중에 그려져야 함)
- URP 사용 시 셰이더 문법이 달라지므로 렌더 파이프라인 확인 필요
- MaterialPropertyBlock을 사용하면 인스턴싱 유지 가능 (100마리 유닛도 성능 문제 없음)

### 편의성 관련 UI 개선

- 도감 보상 일괄 받기
- 일괄 강화 및 추천 장착
- 일괄 머지 기능 (논의 필요 — 전략적 재미와 편의성의 밸런스)
- UI 렉 개선 (상점 등 리스트가 많은 화면 사전 로드)

### 필수 체크 리스트 (2026.01)

2월 폴리싱 시점에서 설정된 5가지 핵심 평가 기준:
1. 게임 진입부터 3분 내에 게임의 매력이 느껴지는가
2. 4챕터 전후로 가치 있는 재화나 유닛 등이 명확히 인지되는가
3. 해당 재화 혹은 유닛이 가지고 싶다는 생각이 드는가
4. 첫 1시간 동안 플레이의 흥미가 유지되는가
5. 신경 쓰일 정도로 불편하거나 불만족스러운 부분은 없는가
