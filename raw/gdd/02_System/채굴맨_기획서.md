## 축약본

> **목적:** 자원 이득을 보기 위한 중간 단계 전설 유닛으로, 초반 경제 확보 → 후반 DPS 폭발의 2단계 전환 구조를 통해 매 판마다 "언제 변신할 것인가"의 전략적 판단을 제공한다.
> **핵심 제한사항:** 변신은 영구적(되돌릴 수 없음), 미네랄 10 소모 필요, 변신 전/후 택틱 카드 공유를 위한 스킬 전환 시스템 필요
> **비전 연결:** 전설 유닛 시리즈 확장 — 기존 6종 10점 전설(전투형)과 차별화된 "경제 루트" 추가. 덱 전환 흐름(Phase 1~2 경제 구간)의 핵심 유닛.

## 2. 달성 목표

### 플레이어 경험 목표
- 미네랄을 쌓아가며 "언제 변신할 것인가"를 판단하는 **전략적 타이밍 재미**
- 변신 후 축적된 [채굴] 스택이 폭발적 공격력으로 전환되는 **보상감과 카타르시스**
- 경제 구간과 전투 구간의 리듬 전환으로 매 판 다른 플레이 경험

### BM 목표
- 무속성 10점 전설의 새로운 포지션(경제 루트)으로 덱 다양성 확대
- 엔지니어 2개 합성 필요 → 엔지니어 확보 욕구 자극

### 운영 목표
- GDD의 광부→드릴 머신 컨셉을 전설 등급 유닛으로 통합, 유닛 관리 간소화
- 향후 미네랄 시스템 확장의 기반 유닛

### 개발 목표
- 변신 시스템(유닛 내 모드 전환 + 스킬 교체)의 프로토타입 확립
- 스킬 전환 시스템을 범용적으로 설계하여 향후 변신 유닛에 재사용

## 3. 수행 방향

### 핵심 설계 원칙
1. **2단계 전환의 전략성** — 변신 타이밍이 매 판 달라야 하며, 빠른 변신 vs 늦은 변신 모두 유효한 전략이어야 함
2. **스택 보상의 명확성** — [채굴] 스택 1개당 공격력 증가가 체감 가능해야 하며, 스택 축적 과정의 피드백이 시각적으로 명확해야 함
3. **기존 덱 전환 흐름과 정합** — GDD의 Phase 1~2(경제 구간) → Phase 3~6(DPS 전환) 흐름을 유닛 1개에 압축

### 제약 사항 / 금지 사항
- 변신은 되돌릴 수 없음 (영구 전환) — 리스크 있는 의사결정으로 설계
- 변신 전 채굴맨의 DPS는 최소화 — 경제 유닛 정체성 유지
- [채굴] 스택은 변신 전에만 적립 가능 — 변신 후 추가 스택 불가

### 우선순위
| 우선순위 | 항목 | 이유 |
|---------|------|------|
| P0 (필수) | [채굴] — Base Skill (채굴 모드) | 경제 핵심 기능 |
| P0 (필수) | [드릴 다운] — 변신 메커니즘 | 유닛의 핵심 전환 기능 |
| P0 (필수) | [드릴 브레이커] — Base Skill (드릴 모드) | 변신 후 DPS |
| P0 (필수) | [채굴 역량] — 스택 비례 공격력 | 채굴 보상 |
| P1 (중요) | 스킬 전환 시스템 (택틱 카드 호환) | 변신 전/후 카드 공유 |
| P1 (중요) | Lv2 레벨업 효과 | 성장 동기 |
| P2 (선택) | 변신 연출 VFX/SFX | 변신 만족감 |

## 4. 개요

### 유닛 기본 정보

| 항목 | 값 |
|------|-----|
| 등급 | Legendary |
| 속성 | Nothing (무속성) |
| 점수 | 10점 |
| 계열 | 쿨타임 |
| 합성 재료 | 엔지니어 + 엔지니어 |
| 외형 컨셉 | 채굴 로봇 (기계/로봇 컨셉, 드릴 장비 장착) |
| 기능적 역할 | 2단계 전환형 경제 유닛 (경제 → DPS) |
| 메타 포지션 | 10점 전설의 "경제 루트" — 기존 4종 기본 루트(전투형) + 2종 소환 옵션과 차별화 |

### 10점 전설 원칙과의 관계

| 원칙 | 채굴맨 적용 |
|------|-----------|
| 기본 성능 50% | 채굴 모드에서 추가 미네랄 확보 = 경제적 기여 (단독 효능감) |
| 시너지 성능 50% | 변신 후 [채굴] 스택 기반 DPS = 축적 보상 (자체 시너지) |
| 하위 유닛 시너지 | **없음** (10점 원칙의 예외) |
| 범용 버퍼 | **없음** (10점 원칙의 예외) |

> 채굴맨은 하위 유닛 시너지/범용 버퍼 대신, **2단계 변신 메커니즘 자체가 핵심 가치**인 특수 포지션의 10점 전설이다.

### 컨셉 점검

| 체크 항목 | 판정 |
|----------|------|
| 외형-메커니즘 연결성 | 채굴 로봇 → 드릴로 자원 채굴 + 드릴로 공격 = 자연스러움 |
| 속성 내 어울림 | 무속성: 엔지니어(건설), 캐논/로켓(화력), 채굴맨(채굴) = 기계/산업 테마 일관 |

### 스킬 구성 요약

```
채굴맨 (채굴 모드)
├─ [채굴] — Base Skill (Passive)
│   ├─ Lv1: 미네랄 획득 시 30% 확률 추가 미네랄 + [채굴] 스택 1 적립
│   └─ Lv2: 스택 2개씩 적립 + 추가 미네랄 확률 50%로 상승
└─ [드릴 다운] — Sub Skill (Active)
    └─ Lv1: 미네랄 10 소모 → 영구 변신 (드릴 모드로 전환)

채굴맨 (드릴 모드 = 변신 후)
├─ [드릴 브레이커] — Base Skill (Action_Skill)
│   ├─ Lv1: 3초 쿨타임마다 드릴을 땅속으로 발사 → 적 위치에서 솟아올라 범위 피해
│   └─ Lv2: 드릴 관통 — 솟아오른 후 추가 적 1체에 관통 피해
└─ [채굴 역량] — Sub Skill (Passive)
    ├─ Lv1: 보유 [채굴] 스택 × 5% 공격력 증가 (최대 20스택 = +100%)
    └─ Lv2: 스택당 7% 증가 (최대 20스택 = +140%)
```

### 핵심 전략 루프

```
[전투 시작] → 채굴 모드로 배치
  → 미네랄 드롭 수집 시마다 30% 확률로 추가 미네랄 + [채굴] 스택
  → 미네랄 축적 + 스택 축적 (경제 구간)
  → 플레이어 판단: "지금 변신할 것인가?"
  → [드릴 다운] 발동 (미네랄 10 소모) → 영구 변신
  → [채굴 역량] 즉시 적용: 축적된 스택 × 5% = 공격력 대폭 증가
  → [드릴 브레이커] 발동: 강화된 공격력으로 범위 DPS
```

### 전략적 딜레마

| 선택 | 이득 | 리스크 |
|------|------|--------|
| 빠른 변신 (스택 5~8) | 빠른 DPS 투입, 위기 대응 | 낮은 스택 = 공격력 증가 미미 (+25~40%) |
| 늦은 변신 (스택 15~20) | 높은 스택 = 폭발적 DPS (+75~100%) | 경제 구간 DPS 공백 길어짐 |
| 변신 안 함 | 미네랄 최대 확보 | DPS 기여 없음, 1슬롯 낭비 |

## 5. 시스템 동작 흐름

### 전체 흐름 다이어그램

```
[전투 시작]
  → [채굴 모드 배치]
  → [미네랄 드롭 수집]
  → [채굴 스킬 발동?] ─ 30% 확률 ─→ [추가 미네랄 + [채굴] 스택 +1]
  → [미네랄 ≥ 10?] ─ Yes ─→ [드릴 다운 발동 가능]
  → [플레이어 변신 선택?]
      ─ Yes ─→ [미네랄 10 소모] → [영구 변신] → [드릴 모드]
      ─ No ─→ [채굴 계속]

[드릴 모드]
  → [채굴 역량] 즉시 적용 (스택 × 5% 공격력)
  → [드릴 브레이커] 3초마다 발동
      → 드릴 땅속 발사 → 적 위치에서 솟아오름 → 범위 피해
  → [전투 종료까지 반복]
```

### 스킬별 상세 흐름

#### [채굴] — Base Skill (채굴 모드)

1. **트리거**: 미네랄 자원 획득 이벤트 발생 시
2. **판정**: 30% (Lv2: 50%) 확률 체크
3. **성공 시**:
   - 추가 미네랄 1개 획득
   - 자신에게 [채굴] 스택 +1 (Lv2: +2) 적립
   - [채굴] 스택 최대 20개, 지속시간 무제한 (전투 종료 시 초기화)
4. **시각 피드백**: 스택 적립 시 유닛 위에 채굴 아이콘 + 스택 수 표시

#### [드릴 다운] — Sub Skill (채굴 모드)

1. **발동 조건**: 보유 미네랄 ≥ 10
2. **발동 시**:
   - 미네랄 10 소모
   - 변신 연출 재생 (VFX + SFX)
   - 채굴맨 → 드릴 모드로 영구 전환
   - 스킬셋 교체: [채굴] + [드릴 다운] → [드릴 브레이커] + [채굴 역량]
3. **되돌림 불가**: 한 번 변신하면 전투 종료까지 드릴 모드 유지

#### [드릴 브레이커] — Base Skill (드릴 모드)

1. **발동**: 3초 쿨타임마다 자동 발동
2. **동작**:
   - 드릴을 땅속으로 발사
   - 타겟 적의 위치로 이동
   - 적 위치 아래에서 솟아올라 범위 피해
3. **Lv2 추가**: 솟아오른 후 인접 적 1체에 관통 피해 (damage_weight 0.5)

#### [채굴 역량] — Sub Skill (드릴 모드)

1. **타입**: 패시브 (변신 즉시 적용)
2. **효과**: 보유 [채굴] 스택 × 5% (Lv2: 7%) 만큼 공격력 비율 증가 (mul_atk)
3. **갱신**: 변신 시점의 스택 수로 고정 (변신 후 스택 변동 없음)

### 예외 처리 / 엣지 케이스

| 상황 | 처리 방식 |
|------|----------|
| 미네랄이 10 미만일 때 변신 시도 | 발동 불가 — 미네랄 부족 피드백 |
| [채굴] 스택 0인 상태에서 변신 | 가능 — 단, [채굴 역량] 공격력 증가 0% (순수 드릴 DPS만) |
| 변신 중 적 공격 | 변신 연출 중 무적 or 짧은 연출(0.5초) |
| 웨이브 전환 시 스택 유지 | [채굴] 스택은 전투 종료까지 유지, 웨이브 전환에 영향 없음 |
| 택틱 카드로 추가된 스킬의 변신 시 처리 | 같은 unit_group_id 공유 → 유닛 그룹 수준 택틱은 자동 유지 |
| 변신 후 새로운 택틱 카드 획득 | 드릴 모드 유닛에 정상 적용 (같은 유닛 그룹) |

## 7. 밸런스 계획

> ⚠️ 세부 수치는 확정값이 아니며, 테스트 후 조정될 수 있습니다.

### 설계 기준
- 변신 후 DPS는 같은 10점 전설(고양이 사부, 시즈 등)과 유사 수준이어야 함
- 단, [채굴] 스택이 높을수록 더 강하고, 낮으면 약한 구조
- 스택 10개(중간) 기준으로 다른 10점 전설과 동등 DPS

### 주요 수치 테이블

| 항목 | Lv1 | Lv2 | 비고 |

## [유닛 전용] 설계 고려사항: 스킬 전환 시스템

### 문제

인게임에서 택틱 카드로 획득한 스킬이 변신 전/후 유닛 모두에 적용되어야 하며, 동시에 변신 전/후의 고유 스킬은 달라야 한다.

### 현재 시스템

- 택틱 카드는 Commander 수준에서 관리, **유닛 그룹(unit_group_id)**에 바인딩
- 같은 유닛 그룹이면 유닛 그룹 스킬이 자동 재적용됨
- 개별 인스턴스 버프는 유닛 교체 시 소실 (Eliminate()로 삭제)

### 설계 방향 후보

**A. 조건 분기 스킬**: 같은 스킬 ID이지만 유닛의 변신 상태(플래그)에 따라 효과가 달라지는 구조
- 장점: 택틱 카드 1장 = 1 스킬 ID 유지, 기존 그룹 스킬 시스템과 자연 호환
- 고려: 스킬 내부에 변신 상태 분기 로직 필요

**B. Pair Skill 시스템**: 변신 시 연결된 pair 스킬로 자동 교체
- 장점: 스킬별 독립 설계 가능, 레벨업도 개별 관리
- 고려: pair 관계 데이터 구조 + 변신 시 교체 로직 필요

→ **구현 시 선택** (양쪽 모두 기술적으로 가능, 향후 변신 유닛 추가 시 확장성 고려하여 결정)
