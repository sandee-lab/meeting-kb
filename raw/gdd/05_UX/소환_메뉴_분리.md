# 소환 메뉴 분리

> **요약:** 소환을 상점에서 분리하여 별도 메뉴로 독립시키는 UX 개편. 유닛 소환과 장비 소환의 메뉴 구조와 레드닷 처리 규칙을 정의한다.
> **현재 상태:** 이후 버전으로 대체됨

**M6\_소환 메뉴 분리**

# 개요

* 플레이 사이클 (전투 > 전투 보상 획득 > 소환 > 성장 > 전투) 완성을 위해 소환을 분리한다.
* 소환을 상점에서 분리한다.
* 메인 화면에 소환 진입 아이콘을 배치한다.
* 소환 가능 시 소환 진입 아이콘에 레드닷을 찍어주고, 선택 시 해당 소환창을 열어 경로를 단축한다.

# UI 구조

* 소환 (신규)
  + 유닛 소환 탭
    - 1회 소환
    - 10회 소환
  + 장비 소환 탭 (기존의 고급 장비 소환)
    - 광고 소환
    - 1 회 소환
    - 10 회 소환

# 동작

* 진입 아이콘 선택
* 소환 UI 출력
  + 기본 탭은 [유닛 소환]으로 한다.
    - 이용 빈도가 높은 소환을 기본 탭으로 설정한다.
      * 장비의 경우 뽑기 외에도 획득처가 많지만, 유닛은 뽑기를 통해 주로 수급이 이루어진다.
* 소환 시도
  + 비용 표기
    - 현재 각 소환은 2개의 비용이 설정돼 있다.
      * 1번을 각 소환 별 소환권, 2번을 다이아로 한다.
        + 2개 비용 중 1개만 만족한 경우

만족한 해당 재화를 표기한다.

* + - * + 2개 비용 중 2개 모두 부족한 경우

1번 비용을 표기한다.

* + - * + 2개 비용 중 2개 모두 만족한 경우

1번 비용을 표기한다.

* 결과 결정
  + 소환 버튼 선택 시 비용을 차감하고, 해당 횟수만큼 결과를 정한다.
    - 비용 차감과 동시에 결과가 정해진다.
    - 연출 중 게임 종료하는 경우, 재접속 시 비용이 차감돼 있고 결과물을 얻은 상태이다.
* 연출
  + 소환 시도 후 각 소환 연출을 재생한다.
  + 연출은 스킵 가능하다.
    - 스킵 시 결과창으로 넘겨준다.
* 이어서 소환하기
  + 소환 연출 결과창에 직전 시도한 소환 버튼을 배치한다.
  + 비용을 만족한 경우, 이어서 소환을 시도할 수 있다.
  + 소환 종류에 따라 연출이 생략될 수 있다.
    - 유닛 : 연속으로 진행하는 경우 초반부 연출 생략 (현재 적용 상태)
    - 장비 : 기본 연출 동일 진행

# 레드닷

* 장비 소환 가능 시
  + 진입 아이콘 레드닷
    - 진입 시 장비 소환 탭을 열어준다.
  + 장비 소환 탭에 레드닷
    - 다른 탭 선택 시
  + 가능한 소환 버튼에 레드닷
* 유닛 소환 가능 시
  + 진입 아이콘 레드닷
    - 진입 시 유닛 소환 탭을 열어준다.
  + 유닛 소환 탭에 레드닷
    - 다른 탭 선택 시
  + 가능한 소환 버튼에 레드닷
* 예외 처리
  + 장비, 유닛 소환 모두 가능한 경우 바로가기 처리
    - [유닛 소환] 탭 (기본 상태)

# 논의 사항

* 아이콘
  + 장비 / 유닛 소환 아이콘을 구분하고, 해당 소환 가능 시에 띄워주는 방식 적용
    - 모두 가능한 경우 캐러셀 처리 (1~2초 간격으로 스위칭)
    - 모두 불가한 경우 [유닛 소환] 아이콘 노출
