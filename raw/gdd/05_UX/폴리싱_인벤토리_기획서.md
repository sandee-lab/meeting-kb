# 폴리싱 - 인벤토리

> **요약:** 인벤토리 기능 보완을 위한 폴리싱으로, Show_type 반영(비표시 항목 숨김), 골드/보석 표시 추가, 순찰 재화 시간 표기, 수량 및 소지 한도 표기(90%/100% 경고 알림), 상자 확률/구성품 표기 등을 포함한다. 소지 한도 초과분은 우편 발송으로 처리하며, 레드닷은 상자 개봉/한도 근접/한도 도달 3단계로 계층 표시한다. 16개 테스트 항목 중 12개 정상, 1개 오류 확인 상태이다.
> **현재 상태:** 미적용

# 폴리싱

## **[완료] [시안 불필요]** Show\_type 반영

* + 처리
    - ConsumableData에서 Show\_type=false 라면, 인벤토리에 표시하지 않는다.

## **[완료] [시안 불필요]** 인벤토리 표시 항목 추가

* + 개선
    - 골드 추가
    - 보석 추가
      * 게임 전반에서 사용되는 핵심 재화이므로, 요약 정보는 상단 UI, 상세 확인은 인벤토리로 역할 분리
  + 처리
    - [아이템] 탭에 타입 추가
      * 대상
        + ConsumableData@type= Gold
        + ConsumableData@type= Currency

## **[UI 점검 필요]** 순찰 재화 순찰 시간 표기

* + 문제
    - [2시간 순찰 골드] 슬롯에 표시될 때 가시성 낮음
  + 개선
    - 순찰 아이템 슬롯에 시간 표기
  + 대상
    - BoxData@select\_type = SweepGold 또는 SweepExp
  + 처리
    - 슬롯에 표기
      * 시간 정보 : BoxPoolData@id=(해당항목)에 해당하는 select\_sweep\_reward\_hour 값
      * 시간 표기 포맷 : n시간 (정수 단위)

## **[UI 점검 필요]**수량 및 소지 한도 표기

* + 문제
    - 각 재화 소지량이 1,000 단위를 넘어가면 K단위 변환 적용.
    - 각 재화의 소지 한도가 정해져있으나, 표시되지 않음
  + 개선
    - 재화의 정확한 소지량을 툴팁에 표기
    - 소지 한도 안내
      * 현재 게임에서는 소지 한도 이상으로 획득해도 카운트 해주지 않음.
  + 처리
    - 참고
      * 수량 범위
        + int : -2,147,483,648 ~ **2,147,483,647** (최대)
      * 실제 적용할 수량 범위
        + ConstantDatas에서 제어 희망

1,000,000 / 10,000,000 식으로 식별 가능한 수준으로 제한 희망

      * 보유 수량이 한도 특정 % 도달하면 알림

## **[시안 불필요]** 상자 확률 표기

* + 문제
    - 상자 타입 별 구성품 확인 개선
  + 개선
    - 구성품 이름과 수량 표기
      * 수량이 1개일 때 : 구성품 이름만 표기
      * 수량이 2개 이상일 때 : "구성품 이름 x 수량" 형태 표기
    - 확률
      * 상자 타입이 Random 일 때 활성화
    - 안내 문구
      * 상자 타입이 Select 일 때 활성화
        + "아래 구성품 중 1개를 선택할 수 있습니다"
        + 로컬키 : local\_box\_probability\_type\_select
      * 상자 타입이 All 일 때 활성화
        + "아래 구성품을 모두 획득할 수 있습니다"
        + 로컬키 : local\_box\_probability\_type\_all
