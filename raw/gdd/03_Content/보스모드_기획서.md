## 2. 달성 목표

### 플레이어 경험 목표
- 다양한 보스를 연속 처치하는 성취감 및 웨이브 진행의 긴장감 제공
- 아웃게임 성장(특히 보스 전용 스탯 `mul_damage_target_grade_boss`)이 점수에 직결됨을 체감
- 리그 경쟁을 통한 비슷한 수준 유저 간 순위 경쟁 동기 부여

### BM 목표
- `mul_damage_target_grade_boss` 스탯 성장에 대한 과금 동기 강화
- 일일 1회 + 광고 1회 구조로 광고 수익 확보

### 운영 목표
- 매일 접속 루틴 형성 (일일 도전 횟수 제한)
- 시즌 교체 시 보스 풀·웨이브 구성·배율을 CSV 행 추가/수정만으로 갱신 가능한 구조 유지
- 신규 보스 추가 시 `BossModeWaveData` 행 추가만으로 즉시 반영 — 코드 변경 불필요

### 개발 목표
- 기존 랭킹 시스템(`RankScheduleData`, `RankListData`) 재활용으로 개발 비용 최소화
- CSV 드리븐: 보스 풀, 확정 보스 웨이브, 웨이브 가중치를 기획자가 직접 조정 가능

## 3. 수행 방향

### 핵심 설계 원칙
1. **아웃게임 성장이 점수를 결정한다** — 인게임 조작 비중은 낮게, 덱 전투력이 딜량에 직결
2. **전투력 기반 스탯 보정으로 초보~고수 모두 적정 도전감 보장** — 난이도 단계 없는 단일 모드, 웨이브 1부터 공통 시작
3. **기존 시스템 최대 재활용** — 신규 코드보다 CSV 행 추가 위주

### 제약 사항 / 금지 사항
- 인게임 카드 드로우 없음 (챕터 덱 그대로 사용)
- CSV는 직접 수정 불가 (Google Sheets 원본 보호 — 기획서 작성 후 Sheets에서 적용)
- 보스 처치 불가 설계(HP 무한) 금지 — 처치 가능, 웨이브 진행 구조

### 우선순위
| 우선순위 | 항목 | 이유 |
|---------|------|------|
| P0 (필수) | 웨이브 진행 인게임, 일일 도전 횟수, 주간 랭킹 | 콘텐츠 핵심 루프 |
| P0 (필수) | 전투력 기반 난이도 보정 (reference_chapter_id) | 단일 모드에서 전 구간 유저 대응 |
| P1 (중요) | 판 내 처치 수 달성 보상 | 매 판 동기 부여 |
| P1 (중요) | 인터랙션 오브젝트 (버프) | 인게임 재미 요소 |
| P2 (선택) | 주간 도전 횟수 누적 보상 | 접속 루틴 강화 보조 |

## 4. 개요

### 기본 구성
- **도전 횟수**: 일일 1회 제공 (서버 초기화 시 충전) + 광고 시청 1회 충전 가능 (최대 2회 / 일, 이월 없음)
  - 진입 시 잔여 횟수 체크, **게임 종료 시 소모 처리**
- **한 판 시간**: 고정 60초 (타이머 종료 시 정규 게임 종료)
- **시즌 주기**: 1주일 (매주 초기화)
- **랭킹 점수**: 시즌 내 전체 도전 누적 딜량 합산
- **리그**: 브론즈 / 실버 / 골드 / 플래티넘 / 다이아 (퍼센타일 기반, 기존 InfiniteRank 구조 재활용)

### 핵심 규칙
- **게임 시간**: 한 판 = 60초 고정. 60초 동안 최대한 많은 보스를 처치하여 딜량 누적
- 웨이브 단위 진행: 60초 내에서 보스 1마리 등장 → 처치 즉시 or 웨이브 시간 초과 시 다음 보스 스폰
- 시간 초과 시 해당 웨이브 딜량만 손실 (추가 페널티 없음)
- 웨이브마다 보스 능력치 배율 상승 (점점 어려워짐)
- 랜덤 보스 등장 (가중치 풀) + 특정 웨이브는 확정 보스 (CSV 지정)
- 맵은 일반 스테이지 대비 1.5배 넓은 전용 맵 사용
- 맵 내 인터랙션 오브젝트 배치 → 상호작용 시 버프 획득 [제작 단계 검토]

### 게임 종료 조건
| 종료 유형 | 조건 | 비고 |
|----------|------|------|
| 정규 종료 | 60초 타이머 만료 | 정상 종료, 누적 딜량 반영 |
| 조기 종료 | 전투 중 아군 전멸 → 부활 팝업 → 미선택 시 종료 | 기본 규칙 동일 적용 (1회 부활 시도 가능) |

### 리그 규칙
리그 구조는 기존 `InfiniteRank` 시스템과 동일한 방식을 적용한다 (`BossRank` 타입 추가).

**리그 배정**
| 리그 | 배정 기준 | 리그 내 정원 |
|------|----------|------------|
| 브론즈 | 전체 하위 60~100% (Percent) | 최대 50명 |
| 실버 | 전체 30~60% (Percent) | 최대 50명 |
| 골드 | 전체 10~30% (Percent) | 최대 50명 |
| 플래티넘 | 전체 상위 0~10% (Percent) | 최대 50명 |
| 다이아 | 전체 상위 100명 (Ordinal) | 최대 100명 |

**리그 배치**
- 이전 시즌 기록 없음 (최초 진입 포함): **브론즈 리그**로 시작
- 이전 시즌 기록 있음: 이전 시즌 최종 퍼센타일 기준 리그 테이블에 맞는 리그로 배치

## 5. 시스템 동작 흐름

### 전체 흐름 다이어그램
```
[로비 진입] → [도전권 확인] → [입장]
    → [웨이브 1부터 시작, 60초 타이머 시작]
    → [보스 등장] → [처치 즉시 or 웨이브 시간 초과] → [다음 보스 스폰 (배율 상승)]
    → [반복]
    → [종료 조건 체크]
         ├─ 60초 만료 → 정규 종료
         └─ 아군 전멸 → 부활 팝업 → 미선택 시 조기 종료
    → [결과: 누적 딜량 집계] → [시즌 랭킹 합산] → [달성 보상 체크]
```

### 단계별 상세 흐름

#### Step 1. 진입
- 로비에서 도전 버튼 클릭
- 잔여 도전 횟수 확인 (0이면 진입 불가)
- 도전 횟수는 **진입 시 체크, 게임 종료 시 소모 처리** (중간 이탈/크래시 시 횟수 보존)
- 도전 기회 충전: 일일 서버 초기화 시 1회 충전 + 광고 시청 시 1회 추가 충전 (일 1회 한정)

#### Step 2. 인게임 진행 (60초)
- 60초 타이머 시작
- 웨이브 단위로 진행: 보스 스폰 → 처치 즉시 or `end_time_sec` 경과 → 다음 웨이브
- `boss_pool_ids` 확인
  - 1개면 해당 보스 확정 등장
  - 복수면 `boss_pool_weights` 가중치 랜덤 선택
- 시간 초과 시 다음 웨이브 전환 (페널티 없음, 딜량 기회만 손실)
- **보스 최종 스탯 산출**:
  ```
  보스 최종 ATK(HP) = base_stat
                    × (플레이어 전투력 / 기준_챕터.recommend_power)   ← 전투력 보정
                    × wave_weight                                      ← 웨이브별 가중치
  ```
  - 기준 챕터: `BossModeLobbyData.reference_chapter_id` → `StageChapterData.recommend_power` 참조
  - `wave_weight`: `BossModeWaveData`에서 웨이브별 직접 설정
- 아군 전멸 시 부활 팝업 출력 → 부활 선택 시 전투 재개 (1회 한정) → 미선택 시 조기 종료

#### Step 3. 맵 내 인터랙션
- 맵에 Boss_Object (1001~1005) 배치
- 유닛이 오브젝트 영역에 머물면 스킬 이펙트(버프) 획득
- 세부 버프 스펙: [제작 단계 검토]

#### Step 4. 결과 처리
- 전투 종료 시 누적 딜량 집계
- 시즌 누적 딜량에 합산 (서버 갱신)
- 판 내 누적 처치 수 확인 → 달성 보상 마일스톤 체크 (매 판 초기화)

#### Step 5. 보상 지급
- 달성 보상 마일스톤 달성 시 즉시 지급
- 주간 마일스톤 보상은 해당 횟수 달성 즉시 지급
- 랭킹 보상은 시즌 종료 후 우편으로 발송

### 예외 처리 / 엣지 케이스
| 상황 | 처리 방식 |
|------|----------|
| 시즌 종료 후 미수령 달성 보상 | 우편 자동 발송 |
| 전투 중 네트워크 단절 | 기존 전투 종료 처리 방식 동일 적용 [TODO — 개발팀 확인] |
| 광고 로드 실패 | 광고 도전권 미지급, 재시도 버튼 노출 |
| 광고 중간 이탈 / 취소 | 광고 도전권 미지급 |
| 광고 시청 완료 후 네트워크 오류 | 서버 검증 실패 시 재시도 처리 (도전권 중복 지급 방지) |

## 7. 밸런스 계획

> ⚠️ 세부 수치는 확정값이 아니며, 테스트 후 조정될 수 있습니다.

### 설계 기준
- 모든 유저 웨이브 1부터 시작, 전투력 보정 공식으로 난이도 자동 조절
- 웨이브별 능력치 배율: 매 웨이브 약 10~15% 상승 (테스트 후 조정)
- 기준 챕터(`reference_chapter_id`): `BossModeLobbyData`에서 기획자가 직접 지정. 해당 챕터의 `recommend_power`가 스탯 보정 분모로 사용됨
- **[설계 의도] 전투력 높은 유저가 절대 딜량도 높다** — 명확한 성장 목표 제시가 목적. 리그 퍼센타일 구조로 같은 수준끼리 경쟁하므로 공정성은 리그 시스템이 담당

### 주요 수치 테이블 (초안)
| 항목 | 기준값 | 비고 |
|------|--------|------|
| 기본 도전 횟수 / 일 | 1 | 이월 없음, 매일 초기화 |
| 광고 추가 도전 | 1 | 최대 2회 / 일 |
| 한 판 전체 시간 | 60초 | 고정 |
| 웨이브별 보스 제한 시간 | [TODO] | 총 60초 안에서 배분, 테스트 후 조정 |
| 웨이브별 HP 배율 증가 | ~15% | 테스트 후 조정 |
| 웨이브별 ATK 배율 증가 | ~10% | 테스트 후 조정 |
| 확정 보스 등장 주기 | 5웨이브마다 | 테스트 후 조정 |
