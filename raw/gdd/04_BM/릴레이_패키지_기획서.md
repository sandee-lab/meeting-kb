# BM 릴레이 패키지

> **요약:** 단계별로 구매가 해금되는 연계형 상품 시스템으로, 이전 단계 상품을 구매해야 다음 단계 구매 자격이 부여된다. RelayGroupData와 RelayStepData 테이블을 신규 추가하여 그룹 단위 관리, 시간 제한(FOMO), 초기화 방식 등을 제어한다. 챕터 실패/클리어 등 특정 조건 만족 시 팝업으로 노출되며, 가로 스크롤 UI에 잠금/연결선/타이머 연출을 포함한다.
> **현재 상태:** 미적용

# **M8\_BM\_릴레이패키지**

## **1. 개요**

* 단계별로 구매가 해금되는 연계형 상품 시스템을 추가한다.
* 이전 단계 상품 구매 시 다음 단계 상품의 구매 자격이 주어진다.
* 특정 조건(챕터 실패, 클리어 등) 만족 시 팝업으로 노출되는 트리거 기능을 포함한다.

## **2. 릴레이 패키지 시스템**

### **개요**

* Step 개념을 도입하여 순차적 구매를 강제한다.
* 그룹 단위로 관리하여 여러 종류의 릴레이 패키지를 운용할 수 있게 한다.
* 시간 제한 기능을 두어 구매 욕구(FOMO)를 자극한다.

### **테이블 변경/추가**

**1) RelayGroupData (신규)**

릴레이 패키지 그룹의 기본 정보를 정의

| **컬럼명** | **적용 대상** | **목적** | **타입** | **비고** |
| --- | --- | --- | --- | --- |
| group\_id | Key | 릴레이 그룹 식별자 | Int | 예: 101 (초보자 릴레이) |
| condition\_type | 로직 | 노출 조건 타입 | Enum | ChapterFail, ChapterClear, LevelReach |
| condition\_value | 로직 | 노출 조건 값 | Int | 예: 10 (10챕터 실패 시) |
| duration\_hour | 로직 | 활성화 유지 시간 | Integer | 0일 경우 무제한, 값 존재 시 타이머 작동 |
| reset\_type | 로직 | 초기화 방식 | Enum | None(1회성), Daily, Weekly |

**2) RelayStepData (신규)**

그룹 내 각 단계별 상품 구성 정의

| **컬럼명** | **적용 대상** | **목적** | **타입** | **비고** |
| --- | --- | --- | --- | --- |
| group\_id | Key | 소속 그룹 ID | Integer | RelayGroupData 참조 |
| step | Key | 단계 순서 | Integer | 1, 2, 3, 4... |
| product\_id | 연결 | 실제 판매 상품 ID | String | IAP 및 재화 차감 상품 ID |
| display\_price | UI | 표기 가격 | String | "1,500원", "30,000원" |
| is\_final | UI | 마지막 단계 여부 | Boolean | True일 경우 UI 강조 연출 |

### **동작 로직**

**1. 활성화**

* 게임 플레이 중 RelayGroupData의 condition\_type과 condition\_value 조건 만족 시 해당 그룹 활성화.
* 활성화 시점부터 duration\_hour 카운트다운 시작 (DB에 expire\_time 저장 필요).

**2. 구매 상태 관리**

* 유저별 릴레이 상태 저장: (user\_id, group\_id, current\_step)
* Step 1은 기본적으로 구매 가능 (Unlock).
* Step N 구매 완료 시, 서버는 Step N+1 상태를 Locked → Available로 변경.
* 마지막 단계 구매 완료 시 그룹 전체 Complete 처리.

**3. 초기화 (Reset)**

* reset\_type에 따라 주기적으로 구매 기록 초기화.
  + BM 전략상 대부분 None(계정당 1회)으로 설정하나, 주간 릴레이는 Weekly 사용 가능.

### **UI 작업**

**1. 릴레이 팝업/페이지**

* **리스트:** 가로 스크롤 또는 계단식 배열.
* **잠금(Lock):** 구매하지 않은 상위 단계는 '자물쇠' 아이콘 및 딤드(Dimmed) 처리.
* **연결선:** 단계 사이에 사슬/화살표 그래픽 추가. 선행 단계 구매 시 연결선 점등 애니메이션.
* **타이머:** duration\_hour 설정 시 "남은 시간 HH:MM:SS" 표기.

**2. 레드닷**

* 트리거 발동 시 상점 탭에 레드닷 출력.

**3. 예외 처리**

* 구매 도중 시간 만료 시: 현재 결제창이 떠있다면 결제 허용, 그 외 진입 차단.
