# 던전 실패 시 성장 가이드 시스템

> **요약:** 던전 실패 시 유저의 덱 상태를 7개 지표(덱 저장 스테이지, 평점, 유닛/장비/지휘관 레벨, 업그레이드 레벨)로 자동 진단하여 우선순위별 맞춤형 성장 가이드를 제공하는 시스템이다. 덱 구성-업그레이드-장비-유닛-지휘관 순서의 5단계 우선순위로 분기하며, 각 분기에서 관련 컨텐츠 및 패키지로 자연스럽게 유도한다. 기준값은 기획자 디폴트-서버 데이터-하이브리드 3단계 방식으로 운영된다.
> **현재 상태:** 미적용

# **던전 실패 시 맞춤형 성장 가이드 시스템**

## **1. 개요**

### **1.1 목적**

* 던전 실패 시 유저에게 명확한 성장 방향 제시
* 병목 구간을 자동 진단하고 최적의 해결 경로 안내
* 컨텐츠 간 자연스러운 순환 유도

### **1.2 작동 시점**

유저가 던전 N레벨 도전 실패 시, 현재 덱 상태를 분석하여 맞춤형 가이드 제공

## **2. 진단 지표**

덱 상태를 다음 7개 지표로 종합 평가합니다.

| **지표** | **설명** |
| --- | --- |
| 덱 저장 스테이지 | 하드카피로 저장된 덱의 최고 클리어 스테이지 |
| 평점 | 덱 평가 시스템의 종합 점수 |
| 유닛 평균 레벨 | 덱 내 배치된 유닛들의 평균 레벨 |
| 장비 평균 레벨 | 덱 내 장착된 장비들의 평균 레벨 |
| 지휘관 레벨 | 덱 내 지휘관의 레벨 |
| 업그레이드 레벨 | 일반 업그레이드의 레벨(단계) |
| 고급 업그레이드 레벨 | 고급 업그레이드의 평균 레벨 |

## **3. 가이드 분기 로직**

### **3.1 우선순위 체계**

진단은 다음 우선순위에 따라 진행되며, **조건 충족한 상위 조건 3개를 우선시하여 출력**

1순위: 덱 구성 (덱 저장 스테이지, 평점)

2순위: 업그레이드 레벨

3순위: 장비 레벨

4순위: 유닛 레벨

5순위: 지휘관 레벨



### **3.2 분기별 가이드**

#### **1순위: 덱 구성 문제**

**진단 조건:**

* 덱 저장 스테이지 < 기준값
* 또는 평점 < 기준값

**가이드:**

****"덱 구성을 재검토해 보세요"

[모의전으로 이동]

**설명:**

* 덱 자체의 시너지나 조합이 부족한 상태
* 모의전에서 다양한 조합 테스트 권장

#### **2순위: 업그레이드 부족**

**진단 조건:**

* 업그레이드 레벨 < 기준값
* 또는 고급 업그레이드 레벨 < 기준값

**1단계 가이드:**

****"업그레이드를 진행하면 더 유리해질 거예요"

[업그레이드 탭으로 이동]

**2단계 가이드 (업그레이드 탭 내):**

**골드 부족 시:**

****"골드가 부족하네요"

[골드 던전으로 이동][골드 패키지로 이동]

**황금 디스크 부족 시:**

****"황금 디스크가 필요하네요"

[특별 패키지로 이동]



#### **3순위: 장비 부족**

**진단 조건:**

* 장비 평균 레벨 < 기준값

**1단계 가이드:**

****"장비를 강화하면 더 유리해질 거예요"

[장비 탭으로 이동]

**2단계 가이드 (장비 탭 내):** 유저가 장비 재료 부족으로 레벨업 불가 시

"장비 재료가 부족하네요"

[순찰로 이동][장비 패키지로 이동]

**골드 부족 시:**

****"골드가 부족하네요"

[골드 던전으로 이동][골드 패키지로 이동]



#### **4순위: 유닛 부족**

**진단 조건:**

* 유닛 평균 레벨 < 기준값

**1단계 가이드:**

****"유닛을 강화하면 더 유리해질 거예요"

[유닛 탭으로 이동]

**2단계 가이드 (유닛 탭 내):** 유저가 유닛 조각 부족으로 레벨업 불가 시

"유닛 조각이 부족하네요"

[유닛 던전으로 이동]



#### **5순위: 지휘관 부족**

**진단 조건:**

* 지휘관 평균 레벨 < 기준값

**가이드:**

****"지휘관을 강화하면 더 유리해질 거예요"

[지휘관 탭으로 이동]

**설명:**

* 지휘관 탭에서 레벨업 진행

**골드 부족 시:**

****"골드가 부족하네요"

[골드 던전으로 이동][골드 패키지로 이동]

**2단계 가이드 (지휘관 탭 내):** 유저가 에너지 쉘 부족으로 레벨업 불가 시

"에너지 셀이 부족하네요"

[지휘관 던전으로 이동]



## **4. 기준값 설정 방식**

### **4.1 3단계 하이브리드 방식 (권장)**

| **단계** | **방식** | **설명** |
| --- | --- | --- |
| **1단계: 초기값** | 기획자 디폴트 설정 | 서비스 초기 또는 데이터 부족 시 사용 |
| **2단계: 데이터 수집** | 서버 통계 기반 | 실제 클리어 유저 데이터 축적 |
| **3단계: 동적 갱신** | 하이브리드 운영 | 디폴트 + 서버 데이터 병합 |

### **4.2 방식별 상세**

#### **방식 1: 기획자 디폴트 (Dev Default)**

**장점:**

* 즉시 구현 가능
* 의도한 난이도 곡선 반영
* 초기 서비스에 필수

**단점:**

* 실제 플레이 데이터와 괴리 가능
* 메타 변화에 대응 어려움

**예시:**

****던전 레벨 10 기준값:

- 유닛 평균 레벨: 4

- 장비 평균 레벨: 12

- 지휘관 평균 레벨: 6



#### **방식 2: 서버 데이터 기반 (Server Analytics)**

**수집 데이터:**

* 던전 N레벨 클리어 성공 유저의 평균값
* 상위 50% 클리어 유저 기준 중앙값
* 최근 7일/30일 데이터

**산출 방식:**

****기준값 = MIN(클리어 유저 평균 × 0.9, 상위 50% 중앙값)

**장점:**

* 실제 플레이 데이터 반영
* 메타 변화 자동 추적
* 정확한 가이드 제공

**단점:**

* 초기 데이터 부족
* 극소수 고인물 영향
* 구현 복잡도 증가

#### **방식 3: 하이브리드 (권장)**

**작동 방식:**

****if 서버\_데이터\_충분 and 신뢰도\_높음:

기준값 = 서버\_데이터\_기반값

else:

기준값 = 기획자\_디폴트값

# 안전 장치

기준값 = CLAMP(기준값, 디폴트 × 0.7, 디폴트 × 1.3)

**신뢰도 조건:**

* 해당 던전 레벨 클리어 유저 100명 이상
* 데이터 수집 기간 7일 이상

**장점:**

* 초기 서비스 안정성
* 장기적 정확도 확보
* 급격한 변화 방지

### **4.3 기준값 갱신 주기**

| **방식** | **갱신 주기** | **비고** |
| --- | --- | --- |
| 디폴트 | 패치 시 수동 | 밸런스 패치 반영 |
| 서버 데이터 | 매일 00:00 자동 | 전날 데이터 집계 |
| 하이브리드 | 매일 00:00 자동 | 신뢰도 검증 후 적용 |

## **5. UI/UX 가이드라인**

### **5.1 가이드 노출 시점**

* 던전 실패 팝업에서 "성장 가이드" 버튼 제공
* 또는 자동으로 가장 우선순위 높은 가이드 노출

### **5.2 메시지 톤**

* 친근하고 격려하는 톤 유지
* "~하세요" 보다 "~하면 더 유리해질 거예요" 권장
* 부족함을 지적하기보다 성장 가능성 강조

### **5.3 시각적 표현**

* 부족한 지표: 빨간색 강조
* 충족한 지표: 회색 또는 비노출
* 권장 경로: 화살표 아이콘으로 시각화

## **6. 기대 효과**

* 수익화: 병목 구간에서 자연스러운 패키지 노출
