### [2024-12-04] 로딩 화면 리소스 변경

- **결론**: 인게임 트랜지션 로딩 전후 연결 처리에서 냥묘 리소스를 모두 제거.
- **출처**: [[raw/meetings/[1204] 회의록]]

### [2025-07-28] UI 디자인 방향 'Retro Futurism' 확정
- **맥락**: 게임 전체의 UI 비주얼 콘셉트를 확정해야 했다.
- **결론**: Retro Futurism 스타일로 UI 방향 확정. CRT 모니터 질감(스캔라인/노이즈), 채도 낮은 기본색 + 레인보우 컬러 포인트, 투박한 기계식/초기 디지털 폰트, 물리적 스위치/다이얼 형태 버튼, 아날로그 효과음 사용.
- **출처**: [[raw/meetings/[0728] 미니크래프트 UI 폴리싱]]

### [2025-07-28] UI 키 컬러 및 아웃라인 규칙 확정
- **맥락**: UI 컴포넌트 간 시각적 일관성을 확보할 규칙 필요.
- **결론**: 아웃라인 컬러를 가급적 통일하고, 종이 베이스에 블랙 계열 포인트, 선명한 컬러를 포인트로 활용.
- **출처**: [[raw/meetings/[0728] 미니크래프트 UI 폴리싱]]

### [2025-08-02] UI 제작툴 피그마로 변경
- **맥락**: XD는 향후 관리 이슈가 크기 때문에 전환 필요.
- **결론**: UI 제작툴을 XD → 피그마(Figma)로 변경. 기존 XD 이미지는 점진적 전환, Free 버전 사용.
- **출처**: [[raw/meetings/[0802] 미니크래프트 UI 개선 방향]]

### [2025-08-02] 이미지 최적화 방식 변경
- **맥락**: 디자인이 픽스되지 않은 상태에서 컬러 조립 방식은 변경 시 시간 리스크가 크다.
- **결론**: 버튼 및 등급 배경 등은 만들어진 이미지로 모두 변경. 컬러 지정 조립 방식 지양.
- **출처**: [[raw/meetings/[0802] 미니크래프트 UI 개선 방향]]

### [2025-08-02] UI 제작 담당자 배정
- **맥락**: UI 작업의 효율적 분담 필요.
- **결론**: 기존 폴리싱/방향성 정리는 정해룡, 신규 컨텐츠 레이아웃(상점, 던전 등)은 김수현, 드로잉 이미지/아이콘은 김수현+정승언 담당.
- **출처**: [[raw/meetings/[0802] 미니크래프트 UI 개선 방향]]
