# 랭킹 및 경쟁 시스템

> 출처: [[raw/gdd/02_System/랭킹_시스템_기획서]], [[raw/gdd/03_Content/명예의_전당_선착순_랭킹_기획서]], [[raw/gdd/02_System/채널링_운영정책_기획서]], [[raw/gdd/02_System/로컬_푸시_기획서]], [[raw/meetings/[1208] 이벤트 및 BM]]

## 개요

유저 간 경쟁심을 유발하고 플레이 지속 동기를 강화하기 위한 랭킹 및 경쟁 시스템이다. 스테이지 랭킹(주간 전체 + 일간 조별), 명예의 전당(선착순 스토리 클리어), 채널링 운영 정책이 설계되었다.

## 상세 내용

### 스테이지 랭킹

> 출처: [[raw/gdd/02_System/랭킹_시스템_기획서]]

소프트 런칭에 도입되는 기본 랭킹 시스템. 챕터 클리어 기록(챕터# 웨이브# 형식)을 기준으로 집계.

**주간 전체 랭킹 (前 명예의 전당):**
- 7일마다 갱신, 전체 채널 유저에서 상위 100위 집계
- 갱신 시점: 매주 서버 갱신일 오전 9시 (공개 기준 30분 전 = 오전 8:30 스냅샷)
- 직전 랭킹 순위와 비교하여 순위 상승/유지/하락 구분 연출

**일간 조별 랭킹 (前 라이벌 랭킹):**
- 매일 전날 성적 기준으로 갱신 (오전 9시 공개 / 오전 8:30 스냅샷)
- 50명 단위 그룹 경쟁 (`ConstantDatas@rank_stage_group_size`)
- 7일간 유지되는 조 내 경쟁 후 재편성

**리그 편성 (5개):**

| 리그 | 편성 기준 | 특성 |
|------|-----------|------|
| 다이아 | 1~50위 | 최상위 실력자 |
| 플래티넘 | 51~100위 | 중상위권 경쟁 |
| 골드 | 상위 30~60% | 중간권 |
| 실버 | 상위 60~90% | 중하위권 |
| 브론즈 | 하위 10% | 초보자/복귀 유저 |

- 조 당 인원: 50명
- 신규/복귀 유저는 대기 그룹 편성 후 다음 재편성에 편입
- 재편성 중 1시간 집계 중단
- 전체 유저 수 감소 시 리그 단계 축소 가능 (테이블에서 리그 비율 조정)

**등재/삭제 조건:**
- 등재: 챕터 2 클리어 (`ConstantDatas@rank_enroll_required_chapter`)
- 삭제: 제재 대상 유저 제외

**실시간 피드백:**
- 신기록 달성 시 조 내 순위 변동 실시간 연출 (공개 랭킹에는 즉시 미반영)
- 챕터 종료 후 클리어 연출 시퀀스에 포함
- "내일 예상 순위 2위!" 등 예고 메시지 포함

**보상:**
- 랭킹 갱신 직후 우편함으로 지급
- 미수령 시 소멸 (항상 최근 갱신 회차 보상만 제공)
- 보상 타입으로 Shard(조각 시스템) 지원 (일간 랭킹은 과도하게 높지 않도록 조정)

**데이터 테이블:**
- `RankListData`: rank_type(Stage_Channel / Stage_Group_1~5) / max_rank / linked_reward_id
- `RankRewardData`: rank_min / rank_max / reward_type / reward_id / reward_amount

**달성 조건 신규 추가:**
- 타입: `Acheive_Stage_Rank` (값1: 등수 / 값2: 횟수) — 프로필 해제 조건으로 활용 가능

**도전(경쟁)모드 랭킹 (추후 업데이트 예정):**
- 소규모 조 단위 경쟁 → 승급/유지/강등 시스템 도입
- 랭킹 결과에 따라 보상 및 리그 이동 가능

### 채널링 운영 정책

> 출처: [[raw/gdd/02_System/채널링_운영정책_기획서]]

서비스 확장성과 유저 유입을 채널 단위로 관리.

**채널 구조:**
- 최초 채널 1개 개설 ('신규' 상태)
- 채널 증설 기준: `ConstantDatas@channel_creation_capacity` (예: 1,000명 도달 시 자동 증설)
- 채널 상태: 신규(항상 1개) / 보통 / 점검

**운영 정책 변경:**

| 항목 | 기존 | 변경 |
|------|------|------|
| 콘텐츠 운영 기준 | 채널 | 계정 |
| 계정 생성 | 1서버 당 1계정 | 1채널 당 1계정 |
| 복수 계정 보유 | 불가 | 가능 (채널 당 1계정 허용) |

- 신규 유저: 최초 접속 시 신규 채널 자동 배치 (채널 선택 화면 없음)
- 동시 접속 불허: 최근 접속 기기 유지, 이전 기기 접속 종료 처리
- 튜토리얼 완료: 채널 계정별 독립 관리
- 중복 닉네임 검사 기준도 채널로 변경

**이벤트 관리 변경 (채널링 연동):**
- `EventManagerData@event_scope_type`: Global(기간 고정) / Personal(계정 생성 기준) / Channel(채널 오픈 기준)
- `ContentsUnlockData`에 조건 2 컬럼 추가: User_Create(계정 생성 n일차) / Channel_Create(채널 생성 n일차)

### 로컬 푸시

> 출처: [[raw/gdd/02_System/로컬_푸시_기획서]]

특정 조건 만족 시 로컬 푸시를 발송하여 리텐션 강화.

- 푸시 종류는 최소화, 우선순위 존재
- 하나 발송 후 지정 시간 내 추가 발송 없음

| 조건 | 제목 | 내용 |
|------|------|------|
| 보유 행동력 최대 도달 | 행동력이 가득찼습니다 | 행동력이 가득찼습니다. 접속해서 사용하세요! |
| 순찰 보상 최대치 도달 | 순찰 보상이 가득찼습니다 | 순찰 보상이 가득찼습니다. 접속해서 수령하세요! |
| 일일 초기화 시간 | 새로운 하루 | 새로운 하루가 시작되었습니다! |

- 이벤트/시즌 콘텐츠 추가에 따라 확장 가능

### 명예의 전당 (선착순 랭킹)

- 스토리 챕터 최초 클리어 유저에게 영구 기록 부여
- 선착순 경쟁을 통한 탐색 및 도전 동기
