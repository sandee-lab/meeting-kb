## 개요

미니크래프트(타이니캣어셈블)의 UI/UX 디자인 방향은 레트로 퓨처리즘을 기반으로, 밝은 톤과 캐주얼한 버튼 스타일을 유지하면서 시각적 폴리싱을 지속적으로 진행해왔다. 초기 UI 폴리싱(2025.07)부터 소프트런칭 폴리싱(2026.03)까지, 유저 피드백을 반영하여 시인성 개선, 연출 최적화, 전환 딜레이 축소 등이 주요 과제로 반복 등장하였다. 특히 유닛 가독성 문제는 3개 빌드 연속 최다 지적 사항으로, 게임의 코어 병목으로 인식되었다.

## 상세 내용

### 디자인 언어 및 아트 디렉션

#### 레트로 퓨처리즘 컨셉 (2025.07)
- CRT 모니터의 화면 왜곡, 스캔라인, 노이즈 효과 적용
- 전체적으로 채도가 낮은 색상을 기본으로 하되, 레인보우 컬러 등으로 색상 대비를 포인트로 활용
- 투박한 기계식 또는 초기 디지털 폰트 사용으로 시대적 느낌 전달
- 버튼과 메뉴를 물리적인 스위치나 다이얼 모양으로 제작
- 버튼 클릭 시 기계식 사운드, 노이즈 등 아날로그 효과음 활용
- 키 컬러: 종이 베이스에 블랙 계열 포인트, 선명한 컬러 포인트
- 아웃라인 컬러 가급적 통일

#### 디자인 기본 원칙 (2025.08)
- 기존 디자인 언어 유지: 밝은 톤 베이스, 버튼 및 팝업 창에서 라운드와 볼록한 형태
- 캐주얼 방향: 큰 버튼, 밝은 톤과 선명한 컬러
- 게임에 어울리는 디자인 코드: 종이 박스, 스타크래프트(SF), 레트로
- 출시 전 최소 2회의 디자인 폴리싱이 필요하다고 판단

#### 폰트 규칙
- 아웃라인이 있는 경우 자간 값을 적게, 아웃라인이 없는 경우 값을 크게
- 한국어, 영어 폰트에 따라 다르게 적용할 수 있도록 시스템 지원
- 강조 폰트 컬러 룰 정리 필요 (고유명사, 강화 스탯 등)

### 제작 프로세스 변경 (2025.08)

- **제작툴 피그마로 변경**: XD는 향후 관리 이슈가 크기 때문에 피그마로 전환 결정
  - XD에서 작업된 이미지들을 점진적으로 피그마로 전환 예정
  - 당분간 Free 버전 사용 (Ingame / Outgame / Shop 영역 구분)
- **이미지 최적화 방식 변경**: 버튼 및 등급 배경 등을 만들어진 이미지로 모두 변경, 컬러를 지정해서 조립하는 방식 지양
  - 디자인이 픽스되지 않은 상태에서 조립 형태 사용 시 디자인 변경에 너무 많은 시간이 소요되는 리스크
- **담당자 배분**:
  - 현재 제작된 중요 페이지 폴리싱 및 디자인 방향성 정리 — 정해룡
  - 신규 콘텐츠 레이아웃 (상점, 던전 등) — 김수현
  - 드로잉 이미지 및 아이콘 — 김수현, 정승언

### UI 폴리싱 대상 화면 (2025.07)

골드 성장, 메인 화면, 프로필, 순찰, 공용 보상창, 기록, 7일 이벤트, 퀘스트, 출석부, 파츠, 지휘관, 종료창 등 전체 UI 화면에 대한 폴리싱이 진행되었다.

### 시인성 개선 — 핵심 반복 과제

시인성 문제는 M9(2025.11)부터 소프트런칭(2026.03)까지 3개 빌드 연속 최다 지적 사항이었다.

#### M9 폴리싱 (2025.11)
- SCV 피격 시 스크린 이펙트 추가, 낮은 HP 상태 표기
- 대미지 폰트 속도 조정 (더 빠르게 사라지도록)
- 골드 및 미네랄 획득 연출 간소화
- 적군 프로젝타일 컬러 통일, 장판 통일
- 강조 폰트 컬러 룰 정리 필요
- 아이콘 전면 정비 (종합 파워, 탐사, 시즌패스, 7일 미션, 설정, 하단 아이콘 전체)
- 상점 패키지 로드 시 진입 렉 해결

#### 2월 폴리싱 리스트 (2026.01)
- 지휘관 강조: 아웃라인 오브젝트 추가로 최상위 렌더링
- 지휘관 주변 콜라이더로 여유 공간 확보
- 지휘관 피격 시 카메라 쉐이크 약화 + 30% 이하 스크린 이펙트
- 전체 유닛 범위 동적 바닥 표시
- 이펙트 끄기 / 유닛 반투명도 옵션 제공
- 설치형 스킬 아군/적군 컬러 구분
- 몬스터 사망 이펙트 과다 제거
- 맵 내 불필요 오브젝트 제거
- 경험치 보석 획득 리듬감 개선 (10/30 단위 묶음 처리, 이징 적용)
- 재화 획득 연출 변경 (화면 날아가는 연출 삭제, 위치 표기로 변경)

#### 소프트런칭 폴리싱 리스트 (2026.03)
- **[P1]** 지휘관 아웃라인 오브젝트 추가 — 근거 5/5 (B2에서 "1웨이브 끝날 때까지 중심 캐릭터인지 몰랐다"는 피드백 대응)
- **[P0]** 지휘관 콜라이더 확대 — 근거 4/5
- **[P0]** 지휘관 피격 연출 개선 — 근거 4/5
- **[P0]** 전체 유닛 범위 동적 표현 — 근거 4/5
- **[P1]** 이펙트 끄기 / 유닛 반투명도 옵션 — 근거 4/5
- **[P1]** 몬스터 표현 약화로 아군 유닛 시각 강조 — 근거 3/5
- **[P0]** 몬스터 사망 이펙트 과다 제거 — 근거 3/5
- **[P1]** 맵 내 불필요 오브젝트 제거 — 근거 3/5 (B2에서 "노란 꽃이 재화로 인식된다"는 피드백)

### 연출 및 전환 딜레이 최적화

연출 딜레이 축소는 Alpha 2(2025.01)부터 소프트런칭(2026.03)까지 지속적으로 요구된 사항이다.

#### Alpha 1 폴리싱 (2025.01)
- 동전 올라가는 애니메이션 속도 전체적으로 20~30% 빠르게
- 머지 연출에 이펙트 및 연출 추가 (유닛 아이콘 등록, 유닛마다 대사 차별화)
- 유닛 피격 시 플래시 효과 추가
- 튜토리얼 앞 부분 구현

#### Alpha 2 폴리싱 (2025.01)
- 인트로 단축 (스킵 기능 추가, 카메라 이동 시간 축소)
- 인게임 플로우 디테일 정리 (UI 등장 후 0.5초 대기, 대사 타이밍 조정)
- 광폭화 게이지 및 워닝 연출 추가
- 카메라 쉐이크 (기지 파괴, 레이지 모드 시작 시)
- 사운드 채우기, 내 기지 폭파 연출 이펙트 추가

#### 소프트런칭 폴리싱 (2026.03)
- **[P0]** 레벨업 카드 선택 시 렉 현상 수정 — 근거 4/5
- **[P1]** 시작 전 연출 길이 짧게
- **[P1]** 레벨업 슬로우 최소화 (이펙트는 정상속도, 진입 직전에만 짧게 슬로우)
- **[P1]** 웨이브 시작 연출 바로 나오게
- **[P1]** 미네랄 채집 속도 지수형으로 변경
- **[P1]** 게임 종료 후 보상 받기까지 시간 줄이기

### 스파인 쉐이더 — 유닛 겹침 시각 처리 (2026.03)

유닛 시인성 문제 해결을 위해 Spine 전용 스텐실 쉐이더 구현 방안이 정리되었다.

- **구조**: 유닛 프리팹에 본체 SkeletonAnimation(Shader A, 스텐실 기록용) + 자식 HighlightLayer(Shader B, 겹침 표시용)
- **Shader A**: 모든 유닛에 적용하여 스텐실 값 기록 (Ref 1, Always, Replace)
- **Shader B-1**: 단색 실루엣 방식 — 겹치는 영역을 불투명한 단색으로 표시
- **Shader B-2**: 반투명 오버레이 방식 — 겹치는 영역에 반투명 색상 적용
- Sorting Layer 설정 필수 (하이라이트 레이어가 본체보다 나중에 그려져야 함)
- URP 사용 시 셰이더 문법이 달라지므로 렌더 파이프라인 확인 필요
- MaterialPropertyBlock을 사용하면 인스턴싱 유지 가능 (100마리 유닛도 성능 문제 없음)

### 편의성 관련 UI 개선

- 도감 보상 일괄 받기
- 일괄 강화 및 추천 장착
- 일괄 머지 기능 (논의 필요 — 전략적 재미와 편의성의 밸런스)
- UI 렉 개선 (상점 등 리스트가 많은 화면 사전 로드)

### 필수 체크 리스트 (2026.01)

2월 폴리싱 시점에서 설정된 5가지 핵심 평가 기준:
1. 게임 진입부터 3분 내에 게임의 매력이 느껴지는가
2. 4챕터 전후로 가치 있는 재화나 유닛 등이 명확히 인지되는가
3. 해당 재화 혹은 유닛이 가지고 싶다는 생각이 드는가
4. 첫 1시간 동안 플레이의 흥미가 유지되는가
5. 신경 쓰일 정도로 불편하거나 불만족스러운 부분은 없는가

### 아웃게임 메뉴 레이아웃 (M8, 2026.03, 미적용)

> 출처: [[raw/gdd/05_UX/아웃게임_메뉴_레이아웃_기획서]]

로비 화면 아이콘 배치와 햄버거 메뉴 구조를 정의한다.

**아이콘 배치 규칙:**
- **좌측 바로가기**: 종료 일자가 세팅되는 기간제 콘텐츠 배치 (멤버십, 배틀패스, 초보자 미션). 햄버거 메뉴 내부에 포함되지 않음
- **우측 바로가기**: 상시 열람 빈도가 높은 콘텐츠 배치 (퀘스트, 랭킹). 햄버거 메뉴 내부에도 포함됨

**햄버거 메뉴 정렬 규칙 (3열, F형 시선 동선):**

| 행 | 좌 | 중 | 우 |
|----|----|----|-----|
| 1행 | 우편함 | 친구 | 퀘스트 |
| 2행 | 출석부 | 도감 | 랭킹 |
| 3행 | — | — | 설정 |

- 우편(각종 보상), 퀘스트(보상), 친구(스태미나) → 1행 (게임 플레이 루프 핵심)
- 랭킹, 도감 → 2행 (조회/기록 중심)
- 설정 → 맨 끝/맨 아래 (UI 표준)
- 메뉴 열기 시 우측에서 좌측으로 들어오는 연출

**중앙 배너**: 첫 구매 이벤트, 특별 패키지 등 배치 예정. 전용 테이블로 관리 (이미지, 텍스트, 순서, 종료 기준, 클릭 이동 대상)

| 기록 없을 때 | '최고 기록 없음' 또는 비표시 (개발팀 확인) |
| 갱신 시점 | 플레이 결과 화면 확인 후 즉시 반영 |

## 이전 설계 (대체됨)

아래 기획서들은 초기 설계 문서로, 이후 개선된 기획으로 대체되었다.

> 출처: [[raw/gdd/05_UX/소환_메뉴_분리]], [[raw/gdd/05_UX/스테이지_결과창]], [[raw/gdd/05_UX/시스템_메시지_규격]], [[raw/gdd/05_UX/일시정지_이어하기]], [[raw/gdd/05_UX/전투_시작_연출]], [[raw/gdd/05_UX/콘텐츠_언락_연출]], [[raw/gdd/05_UX/툴팁_표시_정의]]

### 소환 메뉴 분리 (M6, 대체됨)

소환을 상점에서 분리하여 별도 메뉴로 독립. 유닛 소환·장비 소환 2탭 구조, 레드닷 처리 규칙 정의.
- 기본 탭: 유닛 소환 (이용 빈도 높음)
- 비용: 소환권(1번) / 다이아(2번) 중 만족 조건에 따라 표시
- 연속 소환 시 유닛은 초반 연출 생략, 장비는 기본 연출 유지
- 레드닷: 소환 가능 시 진입 아이콘 → 해당 탭 → 가능한 버튼 계층 표시

### 스테이지 결과창 (M6, 대체됨)

→ [[raw/gdd/05_UX/폴리싱_결과창_MVP유닛_기획서]]로 대체

구성 요소: 승리/패배 텍스트, 챕터 정보, 최고 기록/신기록 표시, 결과 웨이브, 처치 몬스터 수, 획득 보상.
- 최고 기록: 도전 결과 중 가장 높은 웨이브. 도중 이탈 시 이탈 웨이브가 기록
- 신기록: 결과가 이전 최고기록보다 높을 때 (동률은 신기록 아님)

### 시스템 메시지 규격 (M6, 대체됨)

토스트 메시지 타입 정의:
- 타입1: 텍스트만 / 타입2: 텍스트 + 아이콘
- 공통 규격: 최소 300 / 최대 500 너비, 폰트 32pt, 2초 유지
- 신규 메시지 발생 시 이전 메시지 즉시 종료 후 신규 출력

주요 출력 대상: 재화 부족(아웃게임), 잠긴 콘텐츠 접근, 가스 부족(인게임), 인게임 유닛 인벤토리 만원·배치 만원·초과 분해

비고 — 컨텐츠 언락: 로비 복귀 시 2초간 인풋 차단 후 출력. 동시 다중 해제 시 `ContentsUnlockData@order` 순서로 순차 출력.

비고 — 인게임 보상 연출: 몬스터 킬 수 달성 시 화면 상단에서 내려오는 메시지 + 오브젝트 드롭 + 인벤토리 플라이 연출 순서

### 일시정지·이어하기 (M6, 대체됨)

일시정지 팝업 구성:
- 택틱 목록: 획득 순서대로 표시. 스택 방식은 `StageModeTacticPoolData@group_id` 기준
- 피해량 기록: 유닛 별 누적 피해량 / 평균 DPS(소수점 1자리, 2자리 이하 절사). 최상단 유닛을 100% 기준으로 게이지 표시
- 나가기: "이번 게임 보상을 얻을 수 없습니다" 확인 팝업 → 나가기(보상 없음) / 계속하기 분기

이어하기: 강제 종료 후 재접속 시 이어하기 제공. 스테이지는 마지막 웨이브 아머리 화면에서 재개.

### 전투 시작 연출 (M7, 대체됨)

비행선 → 지휘관·상자 드롭 → 상자 개봉 → 택틱 아이콘 플라이 → 'Survive' 문구 시퀀스.
- 로딩 화면 중앙에 비행선 이미지 배치하여 연속성 부여
- Wave 1은 기존 웨이브 시작 연출 생략

### 콘텐츠 언락 연출 (M7, 대체됨)

신규 콘텐츠 해제 시점 연출:
- 로비 복귀 후 딤드 화면 → 흑백 아이콘 + 자물쇠 → 해제 연출(컬러 전환, 스케일 효과, 타이틀 출력) → 아이콘 플라이
- 동시 해제 시 `ContentsUnlockData@order` 순서로 순차 출력
- `ContentsUnlockData@show_popup_on_unlock=true` 시 해당 UI 강제 오픈
- 잠김 상태 숨김 처리: 메인 화면 양쪽 배치 콘텐츠(출석부, 퀘스트 등)는 아이콘 미출력

### 툴팁 표시 정의 (M6, 대체됨)

툴팁 구성 항목 정의 (2025-08-21 시안 기준):
- **인게임 유닛 툴팁**: 유닛명, 스탯(기본값·추가값), 아이콘, 강화레벨, 등급, 공격타입, 스킬탭(잠금 조건 포함), 설명란
- **아웃게임 유닛 스킬**: 스킬명, 아이콘, 조건(강화 레벨 미달 시 자물쇠), 스킬 정보, 이동화살표(upgrade_skill 시)
- **아웃게임 지휘관 택틱**: 택틱카드(종류/아이콘/등급/이름/설명/승급 조건), 하단정보창(이름, 스타트카드, 설명)
  - `add_tactic_type=Default`: "택틱 카드는 전투 중 등장합니다"
  - `add_tactic_type=Pool`: "택틱 카드는 전투 중 등장합니다. 시작 카드는 전투 시작 시 즉시 적용됩니다"

## NCsoft 공식 피드백 — UI/UX 이슈 (2026-03-03, Ver.0.0.10)

> 출처: [[raw/feedback/NCsoft_TinyCatAssemble 피드백_20260303]]

**전황 파악 방해 요소:**
- 웨이브 안내창 + 대형 개체 팝업 동시 출현으로 전투 화면 가림
- 장비/유닛/적 공격 연출 혼재로 피아 구분 어려움
- 설치형 스킬의 아군/적 구분 불명확

**유닛 배치 및 조작:**
- 8방향 진형 배치로 조작 중앙점 혼란 (3시 방향 클리버가 메인으로 오인)
- 조작 즉시 이동 → 딜레이 후 따라오는 형태로 변경 제안 (탕탕 드론 방식)

**덱 정보 가시성:**
- 현재 덱 강도(세트효과, 등급, 특성 효과 등) 파악 어려움
- 속성 연결이 플레이에 미치는 영향 직관성 부족
- 유닛 전투력 수치 표시 / 자동배치 버튼 추가 / 속성 연결 효과 가시성 개선 제안

**기타 확인된 UI 버그 및 개선 요청:**

| 이슈 | 개선 제안 |
|------|-----------|
| Skip 버튼 오른쪽 위 구석 (한손 조작 불편) | Skip 버튼 하단 이동 |
| 유닛 강화 화면 화살표·강화 버튼 거리 멀음 | 화살표를 강화 버튼 좌우에 배치 |
| 장비 아이템 부위 이미지로 인지 어려움 | 장비 이미지로 슬롯 부위 식별 가능하도록 |
| 전설 유닛 공격 피격음 자극적 | 사운드 교체 또는 믹싱 |
| 등급 부스트 효과 불명확, 물음표 클릭 안 됨 | 설명 추가 및 클릭 활성화 |
| 챕터 패키지 할인율 오류 | 수정 |
| 순찰 골드 구매 불가 (버그) | 버그 수정 |
| 보스전 시 레벨 경험치바 미표시 | 표시 유지 수정 |
| 영웅 2개 조합 시 반응 없음 | 안내 추가 |
| 10회 소환 300크리스탈 vs 건전지 10개 500크리스탈 가격 충돌 | 일일 상점 300 미만으로 조정 |
