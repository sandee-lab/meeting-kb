# 장비 시스템 밸런스

> 출처: [[raw/gdd/02_System/장비_밸런스_가이드]], [[raw/gdd/02_System/2026-03-30-장비시스템개편]], [[raw/gdd/02_System/재화_가치_정의]]

## 개요

장비(Parts) 시스템은 9종 라인 × 15단계 진화 구조로 구성되며, 일반 소환과 스페셜 소환의 이중 소환 구조를 통해 장비를 획득한다. 2026-03-30 기준 스페셜 소환에 S급 장비가 추가되었으며, 이를 위한 CSV 구조 변경이 확정되었다.

---

## 1. 장비 종류 및 구조

### 1-1. 9종 장비 라인

| 대분류 | 소분류 | 한글명 |
|:------:|:------:|:-----:|
| BombardDrone | Buster | 폭격기 |
| BombardDrone | Napalm | 위성 |
| BombardDrone | Thunder | 요격 드론 |
| DefenceDrone | Guardian | 이동 드론 |
| DefenceDrone | Repair | 회복 드론 |
| DefenceDrone | Shock | 자기장 드론 |
| AttackDrone | Rapid | 레이저 드론 |
| AttackDrone | Sonic | 미사일 드론 |
| AttackDrone | Heavy | 네이팜 드론 |

- **9종** 장비 라인 (3대분류 × 3종), 각 라인이 **15단계** 진화
- 총 **135개** 장비 아이템
- 일반 풀 크기: **16종**, S급 풀 크기: **9종** (3대분류 × 3종, 고정)

### 1-2. 15단계 진화 경로

```
Common(t0) → Uncommon(t0) → Rare(t0) → Epic(t0→t1→t2) → Legendary(t0→t1→t2→t3) → Mythic(t0→t1→t2→t3) → Ancient(t0)
```

## 2. 소환 시스템 — 일반 소환 vs 스페셜 소환

### 2-1. 이중 소환 구조

| 항목 | 일반 소환 | 스페셜 소환 |
|:----:|:--------:|:---------:|
| 비용 | **100 다이아** | **300 다이아** |
| 일반(Common) | 86% | 62% |
| 고급(Uncommon) | 11.5% | 25% |
| 희귀(Rare) | 2% | 9% |
| 영웅(일반 Epic) | 0.5% | 3.4% |
| 영웅(S급 Epic) | — | 0.6% |
| 일반영웅 천장 | **80회** | **10회** |
| S급영웅 천장 | — | **60회** |

> S급 영웅은 스페셜 소환에만 존재. 두 천장 카운터는 **별도 카운팅**.

### 2-2. Pity 적용 유효확률 (스페셜 소환)

| 구분 | 기본 확률 | 천장 | E[주기] | **유효확률** | 배율 |
|:----:|:-------:|:----:|:-------:|:-----------:|:----:|
| 일반 영웅 | 3.4% | 10회 | 8.60회 | **11.63%** | 3.4x |
| S급 영웅 | 0.6% | 60회 | 50.51회 | **1.98%** | 3.3x |

> MC 검증(100K × 1000회): 천장이 에픽 출현의 **~71%** 를 차지.

### 2-3. 소환 1회 기대 가치

등급 기준 가치 (Common = 70 dia, 진화비용 기반 3배 체계):

| 등급 | 기준 가치 |
|:----:|:--------:|
| Common | 70 dia |
| Uncommon | 210 dia |
| Rare | 630 dia |
| Epic | 1,890 dia |
| S급 Epic | 1,890 dia |

- 일반 소환 기대 가치: **106.4 dia** (비용 100 dia → +6.4%)
- 스페셜 소환 기대 가치: **375.1 dia** (비용 300 dia → +25.0%, MC 100K×1000회 실측)

## 3. S급 장비 개편 (2026-03-30)

> 현재 상태: **미적용**
> 출처: [[raw/gdd/02_System/2026-03-30-장비시스템개편]]

### 3-1. 개편 목적 및 설계 방침

스페셜 소환에 S급 장비를 추가하기 위한 데이터 구조 및 클라이언트 변경.

- `ShopDrawData`에 `epic_s_rate` 컬럼 추가 (S급 Epic 확률 분리)
- `ShopDrawPoolData`에 일반/S급 장비 ID를 모두 등록
- 서버가 풀에서 아이템 선택 시 `PartsData.is_special`로 일반/S급 분기하여 해당 확률 적용
- 확률 계산은 **서버에서 수행**. 클라이언트 CSV는 UI 표시용 + 서버 동기화 기준

### 3-2. CSV 구조 변경

**PartsData.csv**
- `is_special` bool 컬럼 추가. S급 장비 행에 `TRUE` 설정

**ShopDrawData.csv**
- 스페셜 소환 행 추가 (draw_type=`Special`, product_id=50002)
- `epic_s_rate` 컬럼 추가 (S급 Epic 확률, 만분율)
- 일반 소환(50001): `epic_s_rate=0` (S급 없음)
- 스페셜 소환(50002): `epic_rate=340`, `epic_s_rate=60`
- `fail_count_max` = 일반 영웅 천장(10회)
- `fail_count_max_s` 컬럼 추가 = S급 영웅 천장(60회)
- `success_grade_s` 컬럼 추가 = S급 천장 보장 등급

**ShopDrawPoolData.csv**
- product_id=50002 풀에 일반 장비 + S급 장비 ID 모두 등록
- 풀 자체 구분 컬럼 불필요 — `PartsData.is_special`로 런타임 판별

### 3-3. 주요 클라이언트 변경 파일

| 구분 | 파일 | 변경 내용 |
|:---:|------|----------|
| CSV | `PartsData.csv` | `is_special` 컬럼 |
| CSV | `ShopDrawData.csv` | 스페셜 행 + `epic_s_rate`, `fail_count_max_s`, `success_grade_s` |
| C# | `ShopDrawDatas.cs` | 신규 필드 3종 추가 |
| C# | `SummonPopUpUI.cs` | 일반/스페셜 탭 분기 + 이중 천장 UI |
| C# | `UserShopDatas.cs` | 이중 천장 카운터 |
| C# | `PartsSlot.cs` | S급 뱃지/테두리 표시 |
| C# | `PartsSummonResultSlot.cs` | S급 획득 연출 |
| 서버 | WAS API | 이중 천장 카운터 응답 구조 |
