# M5 아웃게임 던전

> **요약:** 메인 전투 외 반복 플레이 콘텐츠. 골드/유닛/장비 던전 종류와 소탕, 단계 구조 설계.
> **현재 상태:** 미적용

M5\_아웃게임\_던전

## 개요

* 메인 전투 외 반복 플레이 콘텐츠를 제공한다.
  + 성장 콘텐츠의 필요 재화를 공급한다.
  + 스테이지 해금으로 성취감을 제공한다.
  + 도전 횟수 제한으로 과금을 유도한다.
* 던전의 종류가 여러 개고, 던전마다 다양한 규칙이 존재한다.
  + 기본 플레이와 유사한 던전 1개를 구현하고, 나중에 나머지를 구현
  + 추후에 개발이 추가하더라도, 추가될 수 있는 구조 필요
* 어려운 던전은 하위 탭으로 따로 분리 (추후에 개발)
* 던전마다 각각 단계가 존재하며, 1부터 도전 가능
* 클리어하면 클리어 보상을 바로 지급한다.
* 클리어 한 단계 던전은 소탕을 통해 즉시 보상을 획득할 수 있다.
  + 각각 던전 별로 소탕 회수가 reset\_type 시점마다 초기화된다.
  + DungeonLobbyData@sweep\_count

## 던전 종류

* 일반 스테이지
  + 게임 스토리 진행, 기본 시스템을 학습시킨다.
  + 클리어 위주로 진행하며, 유저 성장의 기준이 된다.
* 골드 던전
  + 골드 중심의 재화 수급처
  + 짧은 플레이의 돌파형 콘텐츠로 지속적 참여 유도
* 유닛 던전 (추후에 구현)
* 장비 던전 (추후에 구현)
* 이벤트 던전 (추후에 구현)
  + 특정 기간에만 열릴 수 있도록 시스템 필요
* 던전 타입
  + 스테이지 전투 타입 (StageWave)
    - 일반 스테이지 모드의 규칙을 활용한 전투 방식으로 던전을 진행한다.
    - dungeon\_battle\_id는 StageWaveData의 id를 참조한다.

## 동작

1. 메인 화면

![](data:image/png;base64...)

1. 던전 콘텐츠 해금
   * 콘텐츠 타입
     + Contents\_Type = Dungeon
     + Contents\_Type = GoldDungeon
2. 튜토리얼
   * 콘텐츠 최초 해금 시 튜토리얼 진행 (던전 튜토리얼)
     + (메인 화면) 던전 버튼 하이라이트
     + (던전 리스트) 골드 던전 하이라이트
     + (상세 팝업) 도전 버튼 하이라이트
3. 레드닷 출력
   * 소탕 회수 0보다 큰 던전이 하나라도 있으면
     + 메인 화면 던전 버튼 레드닷
4. 던전 리스트

![](data:image/png;base64...)

1. 진행 불가 던전
   * 하단 배치
     + 진행 불가 던전끼리 오더값 정렬 (DungeonLobbyData@order)
     + 해금 조건이 완료되지 않아 플레이할 수 없는 던전 상태
   * 딤 처리 및 자물쇠 아이콘으로 터치 불가 표시
   * 해금 조건 (DungeonLobbyData@unlock\_condition\_desc)텍스트로 출력
2. 레드닷
   * 소탕 횟수 0보다 크면 레드닷

* 던전 정보
  + 오더 (DungeonLobbyData@order)순으로 정렬
  + 던전 이름 (DungeonLobbyData@local\_name) 출력
  + 남은 소탕 횟수 출력
    - 소탕 초기화 횟수 (DungeonLobbyData@sweep\_count) 참조해서 계산
  + 남은 광고보고 소탕 횟수 출력
    - 소탕 횟수가 0 남은 경우 광고 보고 소탕 횟수를 출력 (마
    - DungeonLobbyData@ad\_sweep\_count 참조해서 계산
  + 던전 배너 (DungeonLobbyData@banner\_image) 출력
  + 초기화까지 남은 시간 출력
    - 초기화 타입 참조 (DungeonLobbyData@reset\_type)
  + 던전 보상 도전 가능한 스테이지의 보상을 정보로 출력한다.
    - DungeonStageData의 보상 타입 참고
    - 최대 4개의 보상 종류 (최초 클리어 2종, 반복 2종)

1. 스테이지 정보
   * 클리어한 최고 스테이지에 따른 처리
     + 최고 스테이지 소탕 가능
     + 남은은 소탕 횟수
     + 클리어한 최고 스테이지보다 아래 단계는 소탕 불가능
   * 스테이지 정보 출력
     + 상태에 따른 표시 처리
       - 클리어한 최고 스테이지 아래 단계 = 소탕 불가능
         * 스테이지 체크 표시
         * result\_reward 출력
         * 보상 아이콘 체크 표시
         * 소탕 버튼 및 그 외 소탕 관련 정보 숨김
       - 클리어한 최고 스테이지 = 소탕 가능
         * result\_reward 출력
         * 소탕 버튼 및 그 외 소탕 관련 정보 표시
       - 클리어한 최고 스테이지 한 단계 위 = 도전 가능
         * first\_clear\_reward 와 result\_reward 둘 다 출력
         * result\_reward 우선적으로 출력 (왼쪽)
         * 클리어 보상과 반복 보상은 최대 2종류 까지 가능
         * 최초 클리어 마크 보상 아이콘에 표시
2. 도전 버튼 터치
   * 스테미나를 소진하지 않는다.
   * 던전 전투 종류 (DungeonLobbyData@dungeon\_type & dungeon\_battle\_id)
     + 스테이지 전투 타입 (StageWave)
     + 일반 스테이지 모드의 규칙을 활용한 전투 방식으로 던전을 진행한다.
     + dungeon\_battle\_id는 StageWaveData의 id를 참조한다.
   * 몬스터 데이터 참조
     + DungeonStageData@max\_hp, atk, def 등등
   * 최고 클리어 스테이지 한 단계 위에서만 버튼이 활성화 된다.
3. 던전 클리어
   * 한 단계 더 높은 스테이지가 해금된다. (최고 단계면 Pass)
   * 최초 클리어 보상을 추가로 획득한다.
     + first\_clear\_reward 와 result\_reward 둘 다 획득
     + 클리어 후 보상 창 출력 및 연출
4. 소탕 버튼 터치
   * 최고 클리어 스테이지 한 단계 위에서만 버튼이 활성화 된다.

|  | 도전 버튼 | 소탕 버튼 | 첫 클리어 보상 | 소탕 보상 |
| --- | --- | --- | --- | --- |
| 도전 스테이지 이하 스테이지 | 비활성화 | 비활성화 | 수령 체크 표시 출력 | 표시 |
| 도전 스테이지 (이전 스테이지 클리어) | 활성화 | 활성화 | 보여줌 | 표시 |
| 도전 스테이지 이상 스테이지 | 비활성화 | 비활성화 | 보여줌 | 표시 |

* + 소탕 횟수를 소모하여 반복 보상을 획득할 수 있다.
  + 소탕 횟수가 없는 상태
    - 소탕 남은 횟수의 숫자 부분이 붉은 색으로 출력
    - 버튼 색깔이 회색으로 변경, 소탕 텍스트가 교체 “소탕 횟수 부족”
    - 소탕 버튼은 터치 불가능
    - 광고 보고 충전 기능 추가
      * 소탕 횟수 충전
        + 광고 보고 소탕 횟수를 충전할 수 있다. (가득)
      * 소탕 횟수 증가
        + VIP 멤버십을 소유한 플레이어일 경우, 소탕 횟수 2에서 3으로 증가
        + VIP 멤버십을 소유한 플레이어가 광고 보고 소탕 횟수를 충전할 경우
