# 스테이지 결과 창

> **요약:** 스테이지 클리어·패배 결과 팝업의 구성 요소와 최고 기록·신기록 표시 규칙을 정의한 UX 설계 문서.
> **현재 상태:** 이후 버전으로 대체됨

**결과창**

구성

![](data:image/png;base64...)

* + 결과
    - 20웨이브를 모두 클리어 시 성공
    - 도중에 사망 시 패배
    - 상단단에 “승리”, “패배” 텍스트 출력!
      * Local\_Popup\_Title\_Stage\_Result\_Victory 승리
      * Local\_Popup\_Title\_Stage\_Result\_Failed 패배
      * 이미지에 글자 포함?
    - 배너 이미지는 결과에 따라 달라진다
      * 성공 이미지 출력
      * 실패 이미지 출력
  + 현재 챕터
    - Local\_Popup\_Title\_Stage\_Result\_Chapter 챕터 {0}
  + 최고 기록 / 신기록 표시
    - 현재 챕터에 처음 도전한 경우 최초 도전
    - 최고 기록이란? 도전한 결과 중 가장 높은 웨이브
    - 최고 기록은 왼쪽 “Best” “Wave 14”로 따로 출력된다.
      * Local\_Popup\_Title\_Stage\_Result\_Best 최고 기록 (Best)
      * Local\_Popup\_Title\_Stage\_Result\_Wave 웨이브 {0}
    - 플레이 도중에 나간 경우
      * 나갔을 때의 웨이브가 최고 기록으로 저장된다.
    - 신기록이란? 도전한 결과가 최고기록 보다 높은 경우
      * 동일할 경우, 신기록이 아니다.
    - 신기록 텍스트 추가
      * 신기록일 경우 UI 우상단에 마크가 표기된다.
      * Local\_Popup\_Title\_Stage\_Result\_New\_Record 신기록!
  + 결과 웨이브
    - “웨이브 {0}”로 출력된다.(Local\_Popup\_Title\_Stage\_Result\_Wave)
    - 승리 시 웨이브 20으로 출력된다.
    - 준비 페이즈에서 나간 경우 해당 웨이브가 출력된다.
      * 예시) 시작하자마자 나간 경우 1웨이브

| 케이스 / 결과 | 성공 | 실패 |
| --- | --- | --- |
| 최초 도전 | 최고기록 X  신기록 O | 최고기록 X  신기록 O |
| 반복 도전 | 최고기록 O(20웨이브)  신기록 X | 최고기록 O - 지난 기록보다 높으면 갱신  신기록 - 지난 기록보다 높으면 O |

* + 처치한 몬스터 수 (진행)
    - 인게임에서 처치한 몬스터의 수를 표시한다.
    - 아이콘과 숫자로 출력한다.
    - 몬스터 아이콘 필요
  + 획득 보상
    - 획득 상자 (상자는 까서 줍니다)
      * 깐 상자 아이콘 필요
    - 획득 보상 세부 내용
      * 재화
      * 상자 깐 결과
  + 상세 버튼
    - 선택 시[상세] 팝업 출력
    - Local\_Popup\_Title\_Stage\_Result\_Report 상세세

아트 필요

**배너 이미지**: 승리/패배 결과에 따른 2가지 배너

**아이콘**: 몬스터 시체 아이콘, 깐 상자 아이콘

**UI 레이아웃**: 결과창 전체 디자인 (텍스트 배치, 버튼 등)
