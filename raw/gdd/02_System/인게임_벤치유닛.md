# 인게임 벤치 유닛 시스템

> **요약:** 몬스터 처치 경험치로 레벨업하여 벤치 유닛을 소환하는 시스템. 벤치 유닛은 인벤토리에 보관되며 보유 효과를 부여하고, 승급은 가능하나 포메이션 배치와 리롤은 불가능하다.
> **현재 상태:** 이후 버전으로 대체됨

# M6\_2\_인게임\_벤치 유닛

# 목표

* 몬스터 처치에 대한 목표를 강화한다.
* 성장과 보상 선택 요소를 통해서 전략의 재미를 더한다.
* 인벤토리에 추가되는 유닛의 효과로 구현하여, 인벤토리 관리 요소를 추가한다.

# 개요

* 몬스터 처치 시 경험치 획득, 레벨업 시스템을 활용한다.
* 각 레벨 도달 시 마다 데이터시트에서 지정한 아이템을 지급한다.
* 아이템 타입에 ‘유닛 소환’ 타입을 추가한다.
  + ‘유닛 소환’ 타입 아이템 획득 시 지정한 유닛을 소환한다.
  + 포메이션에 자리가 없거나 is\_item 값이 true 이면, 인벤토리로 즉시 보낸다.
  + 만약, 인벤토리에 자리가 없을 경우 자동으로 판매 처리한다.
* 벤치 유닛은 아머리의 인벤토리 슬롯을 1칸 차지한다.
  + 일반 유닛과 동일하게 승급, 판매를 할 수 있다.
  + 리롤은 불가능하며, 포메이션 슬롯에 배치할 수 ㅇ벗다.
* 벤치 유닛은 보유하고 있을 때, 효과의 종류에 따라 각각 특별한 혜택을 부여한다.

# 시스템 동작 구조

## 경험치

### 경험치 획득

* + - 몬스터 처치 시 경험치를 획득한다.
    - 경험치는 드랍하지 않고, 자동으로 획득 처리한다.

### 몬스터 별 경험치 적용

* + - 각 몬스터의 난이도에 따른 경험치 값을 적용한다. (unitdata@exp\_value)
      * 획득 경험치 값은 float 형식으로 처리한다.

## 레벨업

### 조건

* + - 각 레벨 별 필요 경험치 이상의 값을 누적하면, 레벨업 처리한다.

### 보상

* + - 레벨업 보상으로 데이터시트에서 지정한 아이템을 지급한다.
      * ConstantDatas@stagemode\_reward\_item\_id\_by\_level\_up

## 아이템 효과

### 신규 타입 ‘유닛 소환’

* + - 신규 아이템 효과로 **Spawn\_Unit** 타입을 추가한다.

### ‘유닛 소환’ 아이템 획득 시 처리

* + - 포메이션에 빈 슬롯이 있다면, 대상 유닛을 포메이션에 즉시 배치한다.
    - 포메이션에 빈 슬롯이 없거나, is\_item = TRUE이면, 인벤토리로 넣어준다.
    - 만약, 인벤토리에도 빈 슬롯이 없는 경우 해당 유닛을 즉시 판매 처리하여, 재화로 넣어준다.

## 벤치 유닛

### 벤치 유닛 설정

* + - 유닛 중 is\_item 값이 TRUE인 경우 ‘벤치 유닛’으로 처리한다.
    - 벤치 유닛은 포메이션 슬롯에 배치할 수 없다.

### 벤치 유닛 관리

#### 승급

* + - * 일반 유닛 승급 시 승급 대상은 일반 유닛 중에서만 선택할 수 있다.
      * 벤치 유닛 승급 시 승급 대상은 벤치 유닛 중에서만 선택할 수 있다.

| ***Note)***  추후 벤치 유닛의 종류가 추가되는 경우 승급 시 동일한 효과의 유닛으로만 승급이 되도록 규칙이 추가 될 수 있다. 현재는 종류가 1종이므로 is\_item = true 인 유닛으로만 승급되도록 예외 처리 한다. |
| --- |

#### 리롤

* + - * 값을 -1로 처리하여 리롤 할 수 없도록 설정한다.

#### 판매

* + - * 규칙에 따라 판매할 수 있도록 값을 설정한다.

#### 포에미션 슬롯 배치

* + - * 벤치 유닛은 포메이션 슬롯에 배치할 수 없다.

### 벤치 유닛 보유 효과의 종류

#### 클리어 보상 아이템(Clear\_Reward\_Consumable)

* + - * 데이터 설정

| effect\_type | Clear\_Reward\_Consumable |
| --- | --- |
| effect\_value\_1 | consumable id |
| effect\_value\_2 | 수량 |

* + - * 규칙
        + 스테이지 클리어 시 Clear\_Reward\_Consumable 타입의 벤치 유닛을 보유하고 있는 경우 데이터시트에서 지정한 보상을 추가로 지급한다.

#### 클리어 보상 박스(Clear\_Reward\_Box)

* + - * 데이터 설정

| effect\_type | Clear\_Reward\_Box |
| --- | --- |
| effect\_value\_1 | box id |
| effect\_value\_2 | 수량 |

* + - * 규칙
        + 스테이지 클리어 시 Clear\_Reward\_Box 타입의 벤치 유닛을 보유하고 있는 경우 데이터시트에서 지정한 보상을 추가로 지급한다.
        + 박스 아이템으로 지급하며, 박스 아이템은 추후 인벤토리에서 유저가 직접 오픈한다.

    - 레벨 기준
  + exp\_for\_next\_level
    - 다음 레벨에 필요한 경험치

## ConstantDatas

* + stagemode\_reward\_item\_id\_by\_level\_up
    - 레벨업 시 지급할 아이템 ID

## UnitData

* + is\_item
    - 값이 TRUE이면, 벤치 유닛(아이템)으로 처리
    - 값이 FALSE이면, 일반 유닛으로 처리

* + effect\_type
    - Spawn\_Unit

## UnitItemEffectData

* + id
    - 아이템 유닛 id
  + effect\_type
    - Clear\_Reward\_Box
      * 클리어 시 보상 지급하는 타입
  + effect\_value\_1
    - Clear\_Reward\_Box인 경우 지급할 박스 아이템 id
  + effect\_value\_2
    - Clear\_Reward\_Box인 경우 지급할 보상 수량
