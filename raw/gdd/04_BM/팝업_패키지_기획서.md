## 2. 달성 목표

### 플레이어 경험 목표
- 성장에 필요한 아이템이 필요한 시점에 적절한 가격으로 제공되어, "지금 사면 딱 좋다"는 타이밍 만족감을 준다
- 초저가 장비 핫딜로 첫 결제 심리적 장벽을 최소화하고, "다이아만으로도 본전"이라는 인식을 제공한다
- 전설 유닛 성장 패키지로 구매 즉시 의미 있는 성장 단계에 도달하는 확정 성장 경험을 제공한다

### BM 목표
- **PUR 전환**: 장비 핫딜(1,100~2,200원)로 무과금 유저의 첫 결제 전환율 극대화
  > 단, 우리 게임에서 장비 직접 지급 핫딜이 실제로 얼마나 유효할지는 미지수. 핫딜 형태로 개발은 하되, 장비보다 **승급 재료를 푸시하는 구성이 더 가치 있어 보임** — 승급 구간이 실질적인 성장 허들이기 때문. 인스턴스 A 구성에 이를 반영.
- **LTV 확장**: 전설 유닛 성장 패키지(1,100~14,000원)로 마일스톤마다 촘촘한 과금 루프 형성
- **핵심 판매 방식 확립**: PUR 전환 외에도, 니즈 발생 시점 기반 효율적 구매 유도가 가능한 범용 판매 채널 확보

### 운영/개발 목표
- **범용 시스템**: 등장 조건/상품 구성/기간을 모두 CSV 데이터로 설정 가능한 범용 팝업 패키지 시스템으로 개발
- **확장성**: 장비·유닛 외에도 전술·소모품·이벤트 등 다양한 상품에 동일 시스템 재활용
- **기존 인프라 활용**: ShopPackageData/ShopPackagePoolData 구조를 확장하여 최소 개발 비용으로 구현

## 3. 수행 방향

### 핵심 설계 원칙
1. **니즈 시점 노출**: 유저 행동(스테이지 클리어, 장비 장착, 유닛 성장 등) 기반으로 니즈가 발생하는 정확한 시점에 팝업 노출
2. **고민 제거 구성**: 패키지 가격에 준하는 캐시 재화(다이아) + 핵심 상품을 함께 지급하여 "효율 계산" 과정을 없앰
3. **데이터 드리븐**: 트리거 조건, 상품 구성, 기간, 가격을 모두 CSV에서 관리. 코드 변경 없이 신규 팝업 패키지 추가 가능
4. **초보자 패키지와 차별화**: 초보자 패키지는 소환권/샤드/Box 중심, 팝업 패키지(핫딜)는 장비/유닛 조각 중심으로 매력 영역을 분리
5. **확정 성장 보장**: 전설 유닛 패키지는 다음 마일스톤까지 정확히 도달 가능한 수량을 제공하여 구매 = 즉시 성장 체감

### 제약 사항 / 금지 사항
- 동시에 노출되는 팝업 패키지는 최대 1개 (팝업 우선순위 큐로 충돌 방지)
- 동일 패키지 중복 구매 불가 (각 패키지 1회 한정, `purchase_limit = 1`)
- 팝업 기간 만료 후 재노출 불가 (동일 조건 재달성 시에도 미노출)
- 클라이언트 단독 보상 지급 불가 (서버 검증 필수)

| 우선순위 | 항목 | 이유 |

| P0 (필수) | 팝업 패키지 UI (시간 제한 표시 포함) | 유저 접점 |

| P1 (중요) | 전설 유닛 획득 패키지 (미보유 시 트리거) | 유닛 BM 진입점 |

| P2 (선택) | 미구매 만료 시 할인 재노출 | 전환율 보완 |

## 4. 개요

### 기본 구성

본 시스템은 **하나의 범용 팝업 패키지 프레임워크** 위에 두 가지 핵심 인스턴스를 운영합니다.

#### 인스턴스 A: 장비 핫딜 (PUR 전환)

| 단계 | 트리거 | 핵심 상품 | 캐시 재화 | 가격 | 기간 | 태그 |
|------|--------|----------|----------|------|------|------|
| 1단계 | 챕터 1 클리어 | Epic 장비 1개 (유저 선택) | 다이아 100 | 1,100원 | 72시간 | 90% OFF |
| 2단계 | 챕터 2 클리어 | Epic T1 장비 1개 + 동종 Epic 1개 | 다이아 200 | 1,200원 | 72시간 | 85% OFF |
| 3단계 | 챕터 4 클리어 | Epic T2 장비 1개 + 동종 재료 2개 | 다이아 300 | 2,200원 | 72시간 | 80% OFF |

> **설계 의도**: 1~3단계를 통해 장비 진화 플로우(Epic → Epic T1 → Legendary)를 직접 체험하게 하여, 장비 시스템에 대한 이해 + 성장 체감을 동시 제공
> **피드백 반영**: 장비 자체보다 **합성 재료(승급 재료) 지급이 핵심 가치**라는 판단 하에, 3단계 구성을 "재료 중심"으로 설계. 승급 허들 구간 유저에게 정확히 필요한 재료를 제공하는 방향이 PUR 전환에 더 효과적일 것으로 판단.

#### 인스턴스 B: 전설 유닛 BM

**B-2. 성장 패키지 (마일스톤 기반 촘촘 배치)**

각 패키지는 다음 특수 마일스톤(Stat/Add_Tactic 해금)까지 **정확히 도달 가능한 수량**을 제공합니다.

| 패키지 | 트리거 | 전용 조각 | 도달 목표 | 해금 효과 | 가격 | 기간 |
|--------|--------|----------|----------|----------|------|------|
| 1성 스타터 | 전설 유닛 최초 획득 | 4개 + 골드 | Lv0 → Lv3 | Stat 해금 | 1,100~2,200원 | 72시간 |
| 2성 부스트 | enchant Lv3 달성 | 7개 + 골드 | Lv3 → Lv6 | **전술 슬롯 해금** | 3,300원 | 72시간 |
| 3성 돌파 | enchant Lv6 달성 | 11개 + 골드 | Lv6 → Lv9 | Stat 해금 | 5,500원 | 72시간 |
| 4성 진화 | enchant Lv9 달성 | 17개 + 골드 | Lv9 → Lv12 | **전술 슬롯 해금** | 11,000원 | 72시간 |
| 5성 완성 | enchant Lv12 달성 | 16개 + 골드 | Lv12 → Lv15 | Stat 해금 | 14,000원 | 72시간 |

> **근거**: UnitEnchantData 기준, 전설 유닛(예: 제다이 1003)의 enchant_level별 전용 조각(consumable 11003) 소모량을 정확히 계산하여 구성
> - Lv0→3: 1+1+1+1 = 4개, Lv3→6: 1+2+2+2 = 7개, Lv6→9: 2+3+3+3 = 11개, Lv9→12: 3+4+5+5 = 17개 [TODO: Lv10~12 정확한 소모량 재확인 필요], Lv12→15: 6+10+0 = 16개

### 핵심 규칙

**트리거 분류**
- **동시 트리거 복수 만족 (최초)**: 가장 마지막에 만족한 트리거부터 마감 임박 순으로 전체 순차 팝업 노출 (1회 한정)
- **이후 추가 트리거 만족**: 마지막 트리거 1개만 팝업 노출

**팝업 [닫기] 행동 분기**
- 동시 트리거 최초 순차 노출 중: 다음 팝업 노출
- 마지막 팝업 또는 단일/이후 추가 트리거: 팝업이 **스페셜 오퍼 숏컷 버튼으로 소멸하는 연출** → 유저는 메인화면을 보게 됨 (상점 자동진입 없음)

**팝업 [구매] 행동 분기**
- 미구매 오퍼가 있으면 다음 마감 임박 오퍼 팝업
- 미구매 오퍼 없으면 종료

**메인 복귀 트리거**
- 스페셜 오퍼 상점 이탈 후 신규 트리거 만족 시 → 메인 화면 복귀 시 팝업 노출

**공통 규칙**
- 팝업 노출 후 **기간 한정** 카운트다운 시작 (72시간 또는 7일)
- 각 패키지 **1회 구매 한정** (purchase_limit = 1)
- 미구매로 기간 만료 시 패키지 소멸 (재노출 없음)
- 구매 시 즉시 보상 지급 (CommonRewardUI)
- 전설 유닛 성장 패키지는 **해당 유닛 전용 조각**으로 지급 (범용 재료 아님)

### 진입 조건 / 해금 조건
- **자동 팝업**: 트리거 조건 달성 + 로비 진입 시 자동 노출
- **수동 재진입**: 활성 오퍼가 있을 경우 메인 화면에 **스페셜 오퍼 숏컷 버튼** 노출 (가장 빨리 마감되는 오퍼의 잔여 시간 표시)
- **스페셜 오퍼 상점**: 숏컷 버튼 탭 또는 팝업 [닫기] 시 진입 — 활성 오퍼 전체 리스트 확인 가능
- **팝업 우선순위**: 기존 팝업 큐에서 **초보자 패키지 바로 뒤**에 위치

**팝업 우선순위 큐 (업데이트)**

| 순서 | 팝업 |
|------|------|
| 1 | 레벨 업 |
| 2 | 콘텐츠 언락 |
| 3 | 튜토리얼 |
| 4 | 선착순 랭킹 안내 |
| 5 | 최초 클리어 상품 구매 팝업 |
| 6 | 엔딩 챕터 클리어 축하 팝업 |
| 7 | 초보자 패키지 |
| **8** | **팝업 패키지 (핫딜)** ← NEW |
| 9 | 시즌 패스 시작 연출 |

### 보상 구조 요약
- 기존 `ShopPackagePoolData`의 `package_pool_id → (reward_type, reward_id, reward_amount)` 구조를 그대로 활용
- 장비 핫딜: Parts + Currency(다이아)
- 전설 유닛 성장: Consumable(전용 조각) + Gold + Currency(다이아)
- 전설 유닛 획득: Unit(확정) 또는 Box(선택권) + Consumable(전용 조각)

## 5. 시스템 동작 흐름

### 전체 흐름 다이어그램

#### A. 동시 트리거 복수 만족 시 (최초 1회)

```
동시에 A, B, C 트리거 만족 (최초)
→ C 팝업 노출 (가장 마지막 트리거 / 마감 임박 순)
  ├─ [닫기] → B 팝업 노출
  │            ├─ [닫기] → A 팝업 노출
  │            │            └─ [닫기] → 팝업 숏컷 버튼으로 소멸 연출 → 메인화면
  │            └─ [구매] → CommonRewardUI → 다음 마감 임박 오퍼 팝업
  └─ [구매] → CommonRewardUI → 다음 마감 임박 오퍼 팝업
```

#### B. 단일 트리거 / 이후 추가 트리거 만족 시

```
단일 트리거 만족 (또는 이후 추가 트리거)
→ 해당 팝업 1개 노출
  ├─ [닫기] → 팝업 숏컷 버튼으로 소멸 연출 → 메인화면
  └─ [구매] → CommonRewardUI
               ├─ 미구매 오퍼 있음 → 다음 마감 임박 오퍼 팝업
               └─ 미구매 오퍼 없음 → 종료
```

#### C. 메인 복귀 트리거

```
스페셜 오퍼 상점 이탈
→ [신규 트리거 만족] → 메인 화면 복귀 시 팝업 노출 (B 흐름 적용)
```

### 단계별 상세 흐름

#### Step 1. 트리거 조건 달성
- 유저가 특정 행동(스테이지 클리어, 장비 장착, 유닛 enchant 달성 등)을 수행
- 서버에서 `PopupPackageData`의 `popup_trigger_type` + `trigger_value`와 매칭하여 판정
- 조건 달성 시 `UserPopupPackageData`에 활성화 기록 (활성화 시각 포함)
- 동시 트리거 여부: 기존 활성 팝업 존재 여부를 서버에서 함께 응답

#### Step 2. 팝업 노출
- 클라이언트: 로비 진입 시 `UserDataManager`에서 활성 팝업 패키지 목록 확인
- 기간 내 미구매 팝업이 있으면 팝업 우선순위 큐에 삽입
- `PopupPackagePopUpUI` 팝업 표시: 상품 정보 + 남은 시간 카운트다운 + 할인 태그
- 동시에 스페셜 오퍼 숏컷 버튼 활성화 (가장 빨리 마감되는 오퍼의 잔여 시간 표시)

#### Step 3. 구매 처리
- [구매] 버튼 → `WASClient.ShopBuySend()` (기존 상점 구매 API 재활용)
- 서버 검증: 기간 내 + 미구매 확인 → 결제 처리 → 보상 지급
- 클라이언트: `CommonRewardUI`로 보상 팝업 표시
- 패키지 상태를 "구매 완료"로 갱신
- 보상 팝업 닫힘 후 → 미구매 오퍼가 있으면 다음 마감 임박 오퍼 팝업 / 없으면 종료

#### Step 4. 기간 만료 처리
- 서버에서 만료 시각 관리
- 클라이언트: 로비 진입 시 만료 확인 → 자동 제거
- 만료된 패키지는 재노출 불가
- 모든 활성 오퍼 만료 시 스페셜 오퍼 숏컷 버튼 비노출

#### Step 5. 팝업 닫기 연출
- 팝업 [닫기] 시: 팝업이 **스페셜 오퍼 숏컷 버튼 아이콘으로 소멸**하는 연출 재생
  - 예외: 동시 트리거 최초 순차 노출 중 — 소멸 연출 없이 다음 팝업으로 전환
- 닫기 후 유저는 메인화면을 보게 됨 (상점 자동진입 없음)
- 숏컷 버튼에 활성 오퍼가 있음을 인지한 유저가 자발적으로 탭하여 상점 진입

#### Step 6. 메인 복귀 트리거 처리
- 스페셜 오퍼 상점 이탈(닫기) 후 유저가 다른 콘텐츠를 이용하는 중 신규 트리거 만족
- 메인 화면 복귀 시(로비 진입 시) 해당 팝업 1개 자동 노출 (B 흐름 적용)

### 예외 처리 / 엣지 케이스
| 상황 | 처리 방식 |
|------|----------|
| 팝업 노출 중 앱 종료 | 다음 로비 진입 시 남은 기간 내라면 재노출 (단일 트리거 흐름으로 재개) |
| 동시 트리거 복수 달성 (최초) | 마감 임박 순으로 순차 팝업 노출 → 마지막 [닫기] 시 숏컷 버튼 소멸 연출 → 메인화면 |
| 이후 추가 트리거 달성 | 마지막 1개 트리거만 팝업 노출 → [닫기] 시 숏컷 버튼 소멸 연출 → 메인화면 |
| 결제 중 네트워크 실패 | 기존 WAS 재시도 로직(3회) 적용 |
| 결제 환불 | 보상 회수 정책 적용 (서버 처리) |
| 전설 유닛 성장 패키지 — 해당 유닛 미보유 시 | 트리거 자체가 enchant 달성이므로 미보유 상황 불가 |
| 장비 핫딜 — 이미 상위 등급 장비 보유 시 | 노출은 하되, 구매는 유저 판단에 맡김 (다이아 가치 있음) |
| 초보자 패키지와 동시 노출 | 팝업 큐 순서에 따라 초보자 패키지 먼저, 핫딜은 다음 로비 진입 시 |
| 스페셜 오퍼 상점 열람 중 오퍼 만료 | 상점 내 해당 오퍼 비활성화 표시 (만료 배지), 구매 불가 |

## 7. 밸런스 계획

### 설계 기준

> **다이아 기준 단가:** ShopCurrencyData 최소 단위 (20001) 기준 1,400원/150개 = **1다이아 ≈ 9.33원**

- **C·D (전설 성장):** Lv5→6, Lv11→12 한 스텝 비용 정확히 지급 — 서버가 유닛별 동적 계산
- **E (장비 합성):** Epic T2 1개 추가 지급으로 즉시 Legendary 합성 가능 상태 확정
- **F (골드 고급 성장):** 1,100원 소액 + 황금 디스켓으로 고급 성장 시스템 첫 진입 유도
- **G (무한모드):** 다이아보다 입장 재화(코드) 중심 구성 — 콘텐츠 몰입 강화
- **공통:** 가격 소액(1,100원) → 중가(5,500원+) 스케일로 자연스러운 과금 습관 형성

### 검증 기준

- **무과금:** 모든 팝업 미구매. 스테이지 드롭 + 무한모드 자연 수급으로 점진 성장
- **소과금 (1만원 이하):** F + G + E → 고급 성장·무한모드·장비 합성 첫 경험
- **중과금 (1~5만원):** C + E + F + G → 메인 유닛 Lv6 전술 슬롯 해금, 장비 Legendary 달성
- **고과금 (5만원+):** D(복수 유닛) → 최종 전술 슬롯 해금 완료, 유닛 빌드 완성

## 8. UI / UX

### 화면 목록
| 화면명 | 진입 경로 | 역할 |
|--------|----------|------|
| PopupPackagePopUpUI | 로비 자동 팝업 / 스페셜 오퍼 숏컷 버튼 | 팝업 패키지 상세 + 구매 |
| **SpecialOfferShopUI** | 팝업 [닫기] / 스페셜 오퍼 숏컷 버튼 탭 | 활성 오퍼 전체 리스트 열람 + 구매 |
| CommonRewardUI (기존) | 구매 완료 시 자동 | 보상 획득 표시 |

### 화면별 상세

#### PopupPackagePopUpUI (팝업 패키지 팝업)
- **레이아웃:**
  - 상단: 패키지명 + 태그 (HOT / BEST / N% OFF)
  - 중앙-좌: 핵심 상품 3D/2D 비주얼 (장비 아이콘 또는 유닛 일러스트)
  - 중앙-우: 보상 목록 (아이콘 + 이름 + 수량) 세로 나열
  - 하단-좌: 카운트다운 타이머 ("남은 시간 HH:MM:SS")
  - 하단-우: 구매 버튼 (가격 + 취소선 원가)
- **주요 요소:**
  - 할인 태그: `fake_krw` → `krw` 취소선 표시 + `value_text` 배지
  - 남은 시간: 활성화 시각 + `duration_hours` 기준 실시간 카운트다운
  - 보상 미리보기: `ShopPackagePoolData` 기반 보상 아이콘 렌더링
  - 전설 유닛 성장 패키지의 경우 "현재 Lv → 목표 Lv" 성장 미리보기 UI 포함
- **인터랙션:**
  - [구매] → IAP 결제 → 서버 → CommonRewardUI → 미구매 오퍼 팝업 또는 종료
  - [닫기] → 동시 트리거 순차 중이면 다음 팝업 / 마지막이면 팝업이 숏컷 버튼으로 소멸 연출 → 메인화면

#### SpecialOfferShopUI (스페셜 오퍼 상점) ← 신규
- **진입 경로:**
  - 메인 화면 스페셜 오퍼 숏컷 버튼 탭 (유일한 진입점)
- **레이아웃:**
  - 상단: 타이틀 "스페셜 오퍼" + [닫기] 버튼
  - 중앙: 활성 오퍼 리스트 (세로 스크롤)
  - 각 오퍼 카드: 상품 비주얼 + 보상 요약 + 남은 시간 + 가격 + [구매] 버튼
- **정렬 기준:** 마감 임박 순 (남은 시간 오름차순, 빨리 마감되는 상품이 최상단)
- **만료 오퍼 표시:** 만료 배지 표시 + [구매] 버튼 비활성화 (열람은 가능)
- **인터랙션:**
  - [구매] → IAP 결제 → 서버 → CommonRewardUI → 미구매 오퍼 팝업 또는 상점 갱신
  - [닫기] → 메인 화면으로 복귀

#### 스페셜 오퍼 숏컷 버튼 (메인 화면 상시 노출)
- **위치:** 메인 HUD 좌측 (초보자 패키지 버튼 근처)
- **표시 조건:** 활성 팝업 패키지 1개 이상 존재
- **주요 요소:**
  - 스페셜 오퍼 아이콘 + AlarmUI 레드닷
  - 가장 빨리 마감되는 오퍼의 잔여 시간 소형 표시 ("02:15:30")
- **비표시 조건:** 활성 오퍼 없음 (또는 비활성화 연출)
- **탭 시 동작:** SpecialOfferShopUI 오픈

### UX 주의사항
- 자동 팝업은 로비 진입 1회만 (닫은 후 재자동 노출 없음, 숏컷 버튼으로 재진입)
- 초보자 패키지 팝업과 동시 노출 방지 (큐 순서 준수) — 초보자 패키지는 독립 시스템으로 숏컷 버튼 소멸 연출 없음
- 팝업 [닫기] 시 팝업이 숏컷 버튼 아이콘 위치로 날아가며 소멸하는 연출 적용 (상점 자동진입 없음)
- 카운트다운 0 도달 시 숏컷 버튼 자동 소멸 (모든 오퍼 만료 시) + 가벼운 소멸 연출

## 부록: 범용 시스템 재활용 가이드

### 신규 팝업 패키지 추가 시 (코드 변경 없이)
1. `PopupPackageData` CSV에 신규 row 추가 (트리거 조건, 가격, 기간 세팅)
2. `ShopPackagePoolData` CSV에 보상 풀 추가
3. 배포 → 자동 적용

### 재활용 시나리오 예시
| 시나리오 | popup_trigger_type | trigger_value | 상품 구성 |
|---------|-------------------|---------------|----------|
| 장비 핫딜 (초반) | Stage_Clear | 챕터 1/2/4 | Parts + 다이아 |
| 전설 유닛 성장 | Unit_Enchant_Level | Lv 3/6/9/12 | 전용 조각 + 골드 |
| 전설 유닛 획득 | Unit_Not_Owned_Grade | Legendary | 선택권/확정 유닛 |
| 복귀 핫딜 | Comeback | 7일 | 골드 + 다이아 + 소환권 |
| 정체 돌파 | Stage_Fail_Count | 3회 | 미장착 카테고리 장비 |
| 가챠 실패 보상 | Draw_Fail_Count | N회 | 확정 획득권 |

> 새로운 `PopupTriggerType`이 필요한 경우에만 Enum 추가 + 서버 판정 로직 추가 필요.
> 기존 트리거 타입 재활용 시 CSV만으로 운영 가능.
