# 인게임 지휘관 부활 시스템

> **요약:** 지휘관 사망 시 재화 또는 광고 시청으로 부활하는 시스템. 부활 횟수 제한과 BM 연계 구조를 설계한다.
> **현재 상태:** 이후 버전으로 대체됨

**M7\_인게임\_지휘관부활**

# 의도

* 게임 중에 지휘관 사망 시 재화 소모해서 부활하는 시스템
* 게임에서 성장이 부족한 상황에서 자연스럽게 BM으로 이어주는 장치

# 구성 요소

## 부활 시스템 해제 시점

* + 초반 챕터는 기본 플레이 사이클 안내를 위해 웨이브 수와 컨텐츠가 구성되어 있어 지휘관 부활 필요성이 낮을 수 있다.
  + ![](data:image/png;base64...)![](data:image/png;base64...)ContentsUnlockData에서 시스템 해제 시점을 제어한다.

## 전투 당 부활 횟수 제한

* + 부활 횟수 제한이 없는 경우, 전투 긴장감을 유지하기 어려우며 성장을 적절하게 유도할 수 없다.
  + 1회 전투 시도 당 부활 가능 횟수를 제어한다.
    - ConstantDatas
      * stage\_revive\_limit\_per\_battle

## 일일 광고 시청을 통한 부활 횟수 제한

* + 일일 광고 시청을 통한 부활 횟수를 부여하여, 아이템 부족 시에도 부활을 이용할 수 있도록 한다.
  + 광고 제거 상품 비활성화 유저에게는 광고 제거 상품을 노출하는 창구로 이용할 수 있다.
    - ConstantDatas
      * stage\_revive\_ads\_daily\_limit\_per\_battle

## 부활 스킬

* + 전투 상황에 따라 부활 직후 사망할 수 있으므로, 부활 시 스킬을 1회 부여한다.
    - 스킬 아이디 = ? (미정)
    - 스킬 효과
      * 주변 적 넉백
      * n초 동안 지휘관 무적
        + passive\_none / once 로 부활 시 스킬 한번 효과 발동하고 소멸
    - ConstantDatas
      * stage\_revive\_linked\_skill\_id
* 부활 아이템
  + Consumable id = 36
  + 지급처
    - 일일 상점

1. 전투 중 지휘관 사망

      1. 그리고 일일 광고 시청 기회 남음 > 3 으로
      2. 그리고 일일 광고 시청 횟수 소진 & 부활 아이템 소지함 > 3으로
      3. 그리고 일일 광고 시청 횟수 소진 & 부활 아이템 없음 > **5** 으로
   2. [전투 당 부활 가능 횟수] 잔여 기회 없음 > **4** 으로

# 데이터 시트

## ConstantDatas

* + stage\_revive\_limit\_per\_battle
    - 1회 전투 시도 당 부활 가능 횟수
      * 광고 시청 기회 사용 시 차감하지 않는다.
  + stage\_revive\_ads\_daily\_limit\_per\_battle
    - 1일 광고 시청을 통한 부활 가능 횟수
  + stage\_revive\_linked\_skill\_id
    - 부활 시 부여해줄 스킬 id
  + stage\_revive\_linked\_consumable\_id
    - 부활 시 필요한 아이템 id (갯수는 회당 1회로 고정)
  + stage\_revive\_countdown\_sec
    - 부활 팝업 제한 시간(초)

## CommanderData

* + local\_death\_speech 추가
    - 사망 시 스피치 버블에 쓰일 스트링 앞 부분
  + death\_speech\_count 추가
    - 사망 시 스피치 종류 수
    - local\_death\_speech 와 결합하여 사용됨
    - 현재 local\_tactic\_speech와 처리 동일함함
