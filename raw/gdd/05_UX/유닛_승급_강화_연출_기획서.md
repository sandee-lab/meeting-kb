# 유닛 승급 강화 연출

> **요약:** 유닛이 특별 강화 등급(3/6/9/12강화)에 도달할 때 별도 팝업으로 해금 효과를 안내하고, 완료 후 TotalPower 연출을 순차 표시하는 피드백 시스템. 기존 강화 흐름 변경 없이 별도 팝업으로 분리 구현하며, 연출은 짧게 자동 등장/퇴장 처리한다.
> **현재 상태:** 개발 중

## 2. 달성 목표

### 플레이어 경험 목표
- 특별 강화(3강화, 6강화, 9강화, 12강화…) 달성 시 일반 강화와 차별화된 피드백을 제공하여 성취감 강화
- 특수 효과가 해금되었다는 사실을 명확히 인지시킴

### 운영/개발 목표
- 기존 강화 흐름 변경 없이, 특별 강화 안내를 별도 팝업으로 분리 구현
- 특별 강화 안내 팝업 완료 후 TotalPower 연출을 순차 실행하여 UI 겹침 없이 관리

---

## 3. 수행 방향

### 핵심 설계 원칙
1. **별도 팝업으로 분리:** 특별 강화 안내는 `SystemMessageUI`와 독립된 별도 팝업으로 구현한다. UI 위치 겹침 없이 순차 표시한다.
2. **순차 실행:** 특별 강화 안내 팝업 → (완료 후) TotalPower 연출 순서로 실행한다. SystemMessageUI는 특별 강화 팝업 완료 시점을 인지한 뒤 `AddTotalPower()`를 호출한다.
3. **짧은 연출:** 별도 스킵이 필요 없을 정도로 짧게.
4. **연출 통일:** `special_enchant_type`이 `Stat`이든 `Add_Tactic`이든 동일한 연출로 처리한다.

### 제약 사항 / 금지 사항
- 기존 일반 강화 흐름(`UnitEnchantUpdateSend` → `UpdateUnitData` → 결과 UI)을 변경하지 않는다
- 연출 중 입력 차단(모달)을 하지 않는다

### 우선순위
| 우선순위 | 항목 | 이유 |
|---------|------|------|
| P0 (필수) | 특별 강화 판별 후 별도 팝업 안내 연출 재생 | 핵심 기능 |
| P0 (필수) | 해금된 특수 효과 내용 텍스트 표시 | 유저 인지 필수 |
| P1 (중요) | 전용 이펙트(VFX) 및 효과음(SFX) | 몰입감 향상 |
| P2 (선택) | 강화 슬롯 UI에 특별 등급 강조 표시 | 추후 확장 가능 |

---

## 4. 개요

### 기본 구성
- **특별 강화 판별 기준:** `UnitEnchantData.enchant_type == EnchantType.Special`
- **해금 효과 종류:**
  - `UnitEnchantType.Stat` — 추가 스탯 부여
  - `UnitEnchantType.Add_Tactic` — 추가 전술 카드 부여
- **적용 조건:** 해당 유닛을 전투에 출동시켰을 때만 효과 적용
- **연출 방식:** 특별 강화 안내 팝업이 먼저 재생되고, 완료 후 TotalPower 연출이 순차 표시됨

### 핵심 규칙
- 특별 강화 등급에서만 추가 안내 팝업 재생 (일반 강화 시에는 TotalPower만 표시)
- 해금 효과 내용은 [세부 효과] 탭의 강화효과 슬롯과 동일한 텍스트/구성으로 노출
  - `Stat` 타입: `"{스탯명} + {수치}"` (예: "공격력 + 15%")
  - `Add_Tactic` 타입: 전술 아이콘 + 전술 카드명 슬롯
- 팝업 연출 완료 후 TotalPower 연출 시작

### 진입 조건
- 유닛 강화 서버 응답 성공 후, 해당 강화 등급의 `enchant_type == Special`일 때 자동 진입

## 5. 시스템 동작 흐름

### 전체 흐름 다이어그램
```
[유저: 강화 버튼 탭]
  → [WAS: UnitEnchantUpdateSend]
    → [서버 응답 수신]
      → [UserDataManager.UpdateUnitData]
        → [기존 강화 결과 UI 갱신]
          → [enchant_type == Special 확인]
            ├─ [일반] → SystemMessageUI.AddTotalPower() 즉시 실행
            └─ [특별] → ★ 특별 강화 안내 팝업 재생
                          → (팝업 완료 콜백)
                            → SystemMessageUI.AddTotalPower() 실행
```

### 단계별 상세 흐름

#### Step 1. 강화 실행
- 유저가 유닛 강화 UI에서 강화 버튼을 탭
- `WASClient.UnitEnchantUpdateSend(unitidx)` 호출

#### Step 2. 서버 응답 처리
- 기존 로직 그대로: 서버 응답 → `UserDataManager.UpdateUnitData()` → 유닛 데이터 갱신
- 강화 결과 UI 갱신 (별 표시, 슬롯 상태 등)

#### Step 3. 특별 강화 판별
- 강화된 등급의 `UnitEnchantData`를 조회
- `enchant_type == EnchantType.Special`이면 Step 4로 진행
- 아니면 `SystemMessageUI.AddTotalPower()` 즉시 호출 후 종료

#### Step 4. 특별 강화 안내 팝업 재생
- 별도 팝업 표시 및 연출 재생
- 안내 내용:
  - 타이틀 텍스트 (예: "강화 효과 해제!")
  - 해금 효과 내용: **[세부 효과] 탭의 강화효과 슬롯 텍스트를 그대로 노출** (`UnitEnchantInfoSlot` 동일 포맷)
    - `Stat` 타입: `specialStatText` 텍스트 그대로 → `"{스탯명} + {수치}"` (예: "공격력 + 15%")
      - 스탯명: `CSVDataManager.GetStatName()` 반환값
      - 수치: `CommonUtil.GetStatValueText()` 반환값 (mul/rate 계열은 %, 나머지는 정수)
    - `Add_Tactic` 타입: `unitEnchantSkillSlot` 슬롯 그대로 → 전술 아이콘 + 전술 카드명
- 이펙트(VFX) 재생 (기존 에셋 활용, 이펙터 연결 작업 필요)
- 효과음(SFX) 재생

#### Step 5. TotalPower 순차 실행
- 팝업 연출 완료 콜백 → `SystemMessageUI.AddTotalPower()` 호출
- 이후 TotalPower 연출은 기존 동작과 동일

## 6. UI / UX

### 화면 목록
| 화면명 | 진입 경로 | 역할 |
|--------|----------|------|
