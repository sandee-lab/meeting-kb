# GM툴 기능 추가 요청서 — 공허의 균열

> **요약:** 공허의 균열 콘텐츠의 QA 테스트 및 라이브 운영/CS 대응을 위한 GM툴 기능 추가 요청서. 티어/리그 강제 변경, 정수 누적 소모량 변경, 강제 정산 트리거, 어뷰저 랭킹 삭제, 상점 구매 이력 초기화, 플레이 로그 및 스냅샷 조회 등의 기능을 포함한다.
> **현재 상태:** 미적용

**운영툴(GM툴) 기능 추가 요청서**

공허의 균열 (무한 모드) 관리 기능

| **작성일** | 2026년 3월 | **버전** | v2 (기획서/개발명세서 반영) |
| --- | --- | --- | --- |
| **요청 부서** | 기획팀 | **수신** | 서버팀 / 웹팀 담당자 |

# **1. 요청 배경 및 목적**

공허의 균열(무한 모드)은 주간 시즌제 랭킹, 이중 조건 해금 상점(리그 등급 AND 누적 정수 소모량), 실시간 티어 배정 등 복잡한 조건 로직이 포함된 콘텐츠입니다. 아래 두 가지 목적을 위해 운영툴 기능 추가를 요청드립니다.

* QA 테스트 효율화: 시간 경과 조건(일간 정산, 시즌 정산 등)을 즉각 트리거하고, 특정 상태값을 직접 세팅하여 빠른 검증이 가능하도록 합니다.
* 라이브 운영 및 CS 대응: 어뷰저 랭킹 삭제, 결제·구매 오류 복구, 유저 플레이 로그 확인 등 실시간 운영 대응에 필요한 기능입니다.

# **2. 유저 데이터 조작**

특정 유저(UID 기준)의 공허의 균열 관련 상태값을 강제 변경하는 기능입니다.

## **2-1. 티어/리그 강제 변경**

| **기능** | 대상 유저의 현재 소속 티어(다이아, 플레티넘, 골드, 실버, 브론즈)를 임의로 변경 |
| --- | --- |

| **사유** | 상점의 티어 달성 해금 조건 QA 및 강등/승급 연출(VFX\_Tier\_LevelUp) 테스트 |
| --- | --- |

| ⚠️ 상점 구매 자격 판정은 '실시간 티어'가 아닌 '해당 시즌 최고 달성 티어' 기준으로 적용됩니다. (기획서 5.3 예외처리 참조) |
| --- |

## **2-2. 공허의 정수 누적 소모량 변경**

| **기능** | 이번 시즌 해당 유저의 공허의 정수 누적 소모량(unlock\_spend) 수치를 임의의 값으로 덮어쓰기 |
| --- | --- |

| **사유** | 상점 조건부 해금(Spend Lock) 로직 QA — 티어 조건 AND 누적 소모량 조건 동시 만족 여부 검증 |
| --- | --- |

| ⚠️ 상점 해금은 리그 등급 조건과 누적 소모량 조건을 AND로 동시에 만족해야 활성화됩니다. (InfiniteModeShopData 참조) |
| --- |

## **2-3. 최고 달성 단계(Level) 변경**

| **기능** | 유저가 클리어한 최고 단계(Lv.1~Lv.10) 수치 조작 |
| --- | --- |

| **사유** | UI 페이지 자동 포커스 로직 및 단계별 달성 보상 획득 상태 테스트 |
| --- | --- |

| ⚠️ 페이지 포커스 우선순위: ① 미수령 달성 보상이 있는 가장 높은 단계 → ② 도전 가능한 최고 단계. (개발명세서 2. 디폴트 페이지 로직 참조) |
| --- |

## **2-4. 달성 보상 수령 상태 초기화** - 보류(시즌 초기화면 해결)

| **기능** | 특정 유저의 단계별 달성 보상 수령 기록을 초기화 (시즌 단위 또는 특정 단계 선택) |
| --- | --- |

| **사유** | 달성 보상 수령 로직 반복 테스트 및 CS 보상 오류 복구 |
| --- | --- |

| ⚠️ 달성 보상 조건 시간은 고정값이 아닌 데이터 테이블(InfiniteModeStageData@clear\_survive\_time\_1/2/3)로 제어됩니다. 조회 시 참조 필요. |
| --- |

## **2-5. 입장 티켓(재화) 지급/차감 -** 이미 완료됨 (아이템 치트)

| **기능** | 무한 모드 전용 입장 티켓(Consumable 38) 수량 증감 |
| --- | --- |

| **사유** | 티켓 부족/최대 보유(1개) 상태에서의 UX 흐름 및 다이아 대체 결제 연동 테스트 |
| --- | --- |

| ⚠️ 일일 무료 충전: 매일 09:00 서버 기준, 보유량 0개일 때만 1개 충전. 티켓 부족 시 도전 버튼 재화 아이콘이 티켓→다이아로 자동 전환됩니다. |
| --- |

# **3. 시즌 및 랭킹 제어**

시즌 스케줄과 랭킹 보드를 운영툴에서 강제로 제어하는 기능입니다.

## **3-1. 일간/시즌 강제 정산 트리거**

| **기능** | 버튼 클릭 시 즉시 정산 스냅샷을 찍고 일간/시즌 랭킹 보상을 우편으로 발송 | *📎 개발/QA 환경 전용* |
| --- | --- | --- |

| **사유** | 정산 보상 로직, 우편 분할 지급(4슬롯), 축하하기 대상자 전환, 역전 방지 밸런스 검증 |
| --- | --- |

| ⚠️ 정산 중(매주 월요일 23:30~00:00 UTC)에는 랭킹 점수가 기록되지 않습니다. 강제 정산 트리거 실행 시 이 예외처리 로직이 정상 동작하는지도 함께 검증 필요합니다. |
| --- |

| ⚠️ 정산 중 상태에서 오토 클리어 팝업 우선순위 처리도 확인 필요: 오토 클리어 팝업이 먼저 뜨고, 스킵 선택 시 정산 팝업 미출력 / 도전 선택 시 정산 팝업 출력. |
| --- |

## **3-2. 랭킹 기록 임의 삭제 (어뷰징 처리) -** 어뷰저 삭제 관리

| **기능** | 랭킹 보드에 등록된 특정 UID의 점수 기록을 삭제(무효화)하고 순위 재정렬 |
| --- | --- |

| **사유** | 핵/비정상 플레이어 적발 시 라이브 랭킹 보드에서 즉시 처리 |
| --- | --- |

| ⚠️ 동점자 처리 규칙: 동일 점수 시 먼저 달성한 Timestamp 기준으로 순위 부여. 삭제 후 순위 재정렬 시 이 규칙이 올바르게 적용되는지 확인 필요. |
| --- |

| ⚠️ 랭킹 공개는 집계 시점 기준 30분 전 스냅샷 데이터 기준입니다. 삭제 후 반영 타이밍 주의. |
| --- |

## **3-3. 시즌 버프 강제 설정**

| **기능** | 이번 시즌 버프(InfiniteModeSeasonBuffData@buff\_type)를 테이블 데이터와 무관하게 수동으로 덮어쓰기 |
| --- | --- |

| **사유** | 시즌 버프 조합에 따른 밸런스 및 도전 탭 상단 배너 연출 사전 검증 |
| --- | --- |

# **4. 교환소(상점) 제어**

## **4-1. 상점 구매 이력 초기화**

| **기능** | 특정 UID의 공허의 교환소 구매 카운트(Purchase\_Limit)를 주기별(일간/주간/시즌/영구)로 선택하여 0으로 초기화 |
| --- | --- |

| **사유** | 상품 구매 관련 CS 발생 시 복구 처리 및 품절(Sold Out) 상태 해제 테스트 |
| --- | --- |

| ⚠️ 시즌 교체 시 시즌 한정 상품은 자동 교체되나, 계정 한정(영구) 상품의 구매 횟수는 시즌이 지나도 초기화되지 않습니다. 초기화 주기 선택에 주의 필요. |
| --- |

# **5. 데이터 조회 및 로그 확인**

CS 인입 시 상황 파악을 위해 운영툴에서 조회가 필요한 로그입니다.

## **5-1. 무한 모드 플레이 로그 조회**

출력 필요 항목:

* 발생 일시
* 도달한 단계(Level)
* 생존 시간
* 획득한 기본 점수 / 피버 추가 점수
* 소모한 기어(칩) 총량
* 자동 클리어(스킵) 여부 — 랭킹 미반영 케이스 구분용

## **5-2. 스냅샷(덱) 정보 조회**

| **기능** | 해당 유저가 최고 점수 기록 당시 장착한 유닛/배틀슈트 세팅(스냅샷) 조회 |
| --- | --- |

| ⚠️ 랭킹 리스트 및 시즌 결산의 '덱 보기' 기능도 동일 스냅샷 데이터를 참조합니다. 조회 시 기존 프로필 팝업 덱 보기 로직과 연계됩니다. |
| --- |

## **5-3. 교환소 재화(정수) 획득/소모 로그**

| **기능** | 공허의 정수 획득처(일간 랭킹 보상, 시즌 보상, 달성 보상, 축하하기 등) 및 소모처(상점 구매 물품) 타임라인 상세 조회 |
| --- | --- |

| ⚠️ 공허의 정수는 시즌이 지나도 소멸하지 않습니다. CS 문의 시 시즌 간 이월 내역도 함께 추적 필요. |
| --- |

## **5-4. 레드닷 상태 조회**

| **기능** | 특정 유저의 레드닷 트리거 상태값 조회 (달성 보상 미수령 여부, 티켓 보유 수량, 무료 상품 재고, 축하하기 수령 가능 여부) |
| --- | --- |

| **사유** | 레드닷 미출력/오출력 CS 인입 시 상태 확인 |
| --- | --- |

# **6. 개발팀 전달 코멘트**

| **서버팀/웹팀 담당자님,**  무한 모드는 이중 조건 상점 해금(리그 등급 AND 누적 정수 소모량), 주간 시즌 정산, 자동 클리어/정산 중 예외처리 등 조건 분기가 많은 콘텐츠입니다.  QA 기간에 시간을 기다려서 테스트할 수 없는 구조라, 아래 두 기능만큼은 테스트 빌드 운영툴에 최우선으로 붙여주셔야 정상적인 QA가 가능합니다:  **① 티어/누적 소모량 임의 변경 ② 강제 정산 트리거 및 시즌 종료, 다음 시즌 시작**  그 외 로그 조회 기능은 라이브 이후 CS 대응에 필수적이므로, 우선 순위가 낮습니다. 확인 감사드립니다! |
