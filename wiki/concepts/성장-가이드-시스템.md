# 성장 가이드 시스템

> 출처: [[raw/gdd/05_UX/성장_가이드_시스템_기획서]]

## 개요

던전 실패 시 유저의 덱 상태를 7개 지표로 자동 진단하여 우선순위별 맞춤형 성장 가이드를 제공하는 시스템이다. 덱 구성 → 업그레이드 → 장비 → 유닛 → 지휘관 순서의 5단계 우선순위로 분기하며, 각 분기에서 관련 콘텐츠 및 패키지로 자연스럽게 유도한다. 상태: 미적용.

## 목적

- 던전 실패 시 유저에게 명확한 성장 방향 제시
- 병목 구간을 자동 진단하고 최적의 해결 경로 안내
- 콘텐츠 간 자연스러운 순환 유도

## 작동 시점

유저가 던전 N레벨 도전 실패 시, 현재 덱 상태를 분석하여 맞춤형 가이드 제공.

## 진단 지표 (7개)

| 지표 | 설명 |
|------|------|
| 덱 저장 스테이지 | 하드카피로 저장된 덱의 최고 클리어 스테이지 |
| 평점 | 덱 평가 시스템의 종합 점수 |
| 유닛 평균 레벨 | 덱 내 배치된 유닛들의 평균 레벨 |
| 장비 평균 레벨 | 덱 내 장착된 장비들의 평균 레벨 |
| 지휘관 레벨 | 덱 내 지휘관의 레벨 |
| 업그레이드 레벨 | 일반 업그레이드의 레벨(단계) |
| 고급 업그레이드 레벨 | 고급 업그레이드의 평균 레벨 |

## 가이드 분기 로직

### 우선순위 체계

조건 충족한 상위 조건 3개를 우선시하여 출력.

| 순위 | 진단 대상 | 진단 조건 |
|------|----------|---------|
| 1순위 | 덱 구성 | 덱 저장 스테이지 < 기준값, 또는 평점 < 기준값 |
| 2순위 | 업그레이드 레벨 | 업그레이드 레벨 < 기준값, 또는 고급 업그레이드 레벨 < 기준값 |
| 3순위 | 장비 레벨 | 장비 평균 레벨 < 기준값 |
| 4순위 | 유닛 레벨 | 유닛 평균 레벨 < 기준값 |
| 5순위 | 지휘관 레벨 | 지휘관 평균 레벨 < 기준값 |

### 분기별 가이드 메시지 및 유도 경로

**1순위: 덱 구성 문제**
- 가이드: "덱 구성을 재검토해 보세요"
- 버튼: [모의전으로 이동]

**2순위: 업그레이드 부족**
- 1단계: "업그레이드를 진행하면 더 유리해질 거예요" → [업그레이드 탭으로 이동]
- 2단계 (업그레이드 탭 내 골드 부족 시): "골드가 부족하네요" → [골드 던전으로 이동][골드 패키지로 이동]
- 2단계 (황금 디스크 부족 시): "황금 디스크가 필요하네요" → [특별 패키지로 이동]

**3순위: 장비 부족**
- 1단계: "장비를 강화하면 더 유리해질 거예요" → [장비 탭으로 이동]
- 2단계 (재료 부족 시): "장비 재료가 부족하네요" → [순찰로 이동][장비 패키지로 이동]
- 2단계 (골드 부족 시): "골드가 부족하네요" → [골드 던전으로 이동][골드 패키지로 이동]

**4순위: 유닛 부족**
- 1단계: "유닛을 강화하면 더 유리해질 거예요" → [유닛 탭으로 이동]
- 2단계 (유닛 조각 부족 시): "유닛 조각이 부족하네요" → [유닛 던전으로 이동]

**5순위: 지휘관 부족**
- 1단계: "지휘관을 강화하면 더 유리해질 거예요" → [지휘관 탭으로 이동]
- 2단계 (골드 부족 시): "골드가 부족하네요" → [골드 던전으로 이동][골드 패키지로 이동]
- 2단계 (에너지 셀 부족 시): "에너지 셀이 부족하네요" → [지휘관 던전으로 이동]

## 기준값 설정 방식

### 3단계 하이브리드 방식 (권장)

| 단계 | 방식 | 설명 |
|------|------|------|
| 1단계: 초기값 | 기획자 디폴트 설정 | 서비스 초기 또는 데이터 부족 시 사용 |
| 2단계: 데이터 수집 | 서버 통계 기반 | 실제 클리어 유저 데이터 축적 |
| 3단계: 동적 갱신 | 하이브리드 운영 | 디폴트 + 서버 데이터 병합 |

**서버 데이터 기반 산출식:**
```
기준값 = MIN(클리어 유저 평균 × 0.9, 상위 50% 중앙값)
```

**하이브리드 안전 장치:**
```
기준값 = CLAMP(기준값, 디폴트 × 0.7, 디폴트 × 1.3)
```
- 신뢰도 조건: 해당 던전 레벨 클리어 유저 100명 이상, 데이터 수집 기간 7일 이상

**갱신 주기:**
- 기획자 디폴트: 패치 시 수동 갱신
- 서버 데이터 / 하이브리드: 매일 00:00 자동 갱신

## UI/UX 가이드라인

**가이드 노출 시점:** 던전 실패 팝업에서 "성장 가이드" 버튼 제공. 또는 자동으로 최우선 가이드 노출

**메시지 톤:**
- 친근하고 격려하는 톤 유지
- "~하세요" 보다 "~하면 더 유리해질 거예요" 권장
- 부족함 지적보다 성장 가능성 강조

**시각적 표현:**
- 부족한 지표: 빨간색 강조
- 충족한 지표: 회색 또는 비노출
- 권장 경로: 화살표 아이콘으로 시각화

## 기대 효과

- 막막함 해소 및 이탈 방지: 실패 후 즉각적인 해결책 제공
- 컨텐츠 순환: 던전 → 성장 컨텐츠 → 던전 재도전
- 수익화: 병목 구간에서 자연스러운 패키지 노출
