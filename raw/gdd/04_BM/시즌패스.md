# 아웃게임 시즌패스

> **요약:** 기간제 시즌패스의 레벨·보상·과금 구조를 설계. 기본 보상과 과금 보상의 이중 트랙 구조와 스태미나 추가 보상을 정의한다.
> **현재 상태:** 이후 버전으로 대체됨

**M7\_아웃게임\_시즌패스**

# 개요

* 시즌패스는 **일정 기간**동안 운영된다.
* 시즌패스는 운영 기간 동안 달성한 **[레벨]에 따라 보상을 지급**한다.
  + 레벨은 각 단계 별로 요구 조건을 설정한다.
  + 레벨 보상은 **[기본 보상] [과금 보상]**으로 나뉜다.
    - 기본 보상 : 누구나 수령 가능
    - 과금 보상 : 패스 구매자에 한해 수령 가능
  + 최종 레벨 도달 후에는 추가 1레벨 달성 시마다 **[추가 보상]을 반복 수령**할 수 있다.
    - 최초 시즌 패스는 스태미나 사용량으로 레벨을 설계한다.
    - 이를 통해 스태미나 충전 및 멤버십 구매 기회를 확대한다.
* **미달성 레벨은 재화로 구매**할 수 있다.
  + 해당 재화 구매 기회 확대.

# 구성 요소

## 운영 기간

* + 이벤트 관리 테이블(EventManagerData)에서 관리한다. ([이동](https://docs.google.com/spreadsheets/d/1C8IoWcVPT4VTWCiiwfY40fgjd965RaYJmDFQK2kXSo8/edit?gid=116431303#gid=116431303&range=B2))

## 레벨

* + 특정 조건을 기준으로 레벨을 설정한다.
    - 최초 설계 패스는 스태미나 사용량을 기준으로 한다.
  + 레벨별 조건 요구 값은 테이블에서 설정한다.
  + 각 패스의 최종 레벨은 테이블을 참조한다.
    - 해당 패스 레벨에 -1 값이 있는 경우, 최대 레벨 이상으로 레벨을 쌓을 수 있다.
    - -1 값이 없는 경우, 최대 레벨까지만 레벨을 쌓을 수 있다.

## 레벨 보상

* + 기본 보상
    - 패스 진행 중인 모든 유저 대상
    - 패스 레벨 달성 시 해제된다.
  + 과금 보상
    - 각 패스 연결 상품 구매 유저 대상
    - 패스 상품 구매 시 보상이 해제된다.

## 추가 보상

* + 추가 보상 해제 조건
    - 과금 유저
      * PassListData@linked\_product\_id=(product\_id) 구매 시에만
    - 최대 레벨 달성 시
      * 예 : 해당 패스 최대 레벨이 30이고, 이후 -1 단계로 처리된 경우
        + 30레벨까지는 각 레벨에 설정된 레벨 보상을 획득한다.
        + 31레벨부터 매 1레벨 달성할 때 마다 설정된 반복 보상을 획득한다.
  + 추가 레벨 달성 조건
    - PassDetailData@step=-1 행 데이터 참조
  + 반복 보상 규칙
    - 보상 내용
      * PassDetailData@step=-1 행 데이터 참조
        + special\_ 보상만 참조
    - 보상 획득 가능 횟수
      * PassDetailData@max\_repeat\_amount
        + 상품 효율 관리를 위해 반복 수령 횟수 제한한다.
