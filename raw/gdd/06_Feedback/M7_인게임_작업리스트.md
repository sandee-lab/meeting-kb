# 인게임 작업 방향성 및 To Do 리스트

> **요약:** M7 인게임 개발 및 밸런스 작업 항목 목록. 웨이브 밸런스, 전투 템포, 로테이션 속도 등 작업 방향성과 완료/보류 항목을 정리한다.
> **현재 상태:** 이후 버전으로 대체됨

# 작업 방향성

* 전투 정보의 복잡성 완화
  + 지휘관과 유닛의 체력 문제 해소
* 전투 템포 상승 및 게임 전반적으로 답답하게 느껴지는 감정 해소
  + 게임 전체 플레이 시간 대비 순수 전투 시간의 비중을 늘린다.
  + 순수 전투 시간 대비 몬스터를 압도하는 상황의 비중을 늘린다.
* 전설 유닛 중심의 덱빌딩 과정을 직관적으로 구성

# [완료]밸런스 이슈

* 한판의 웨이브 구성을 20 > 15 웨이브 수준으로 압축.
  + 순수 전투 시간은 비슷하게 유지하되 아머리 비중을 낮춰서 불필요한 플레이 시간 압축
  + 아머리에서 승급을 빠르게 할 수 있는 편의성 기능도 고려 필요
  + 유닛 승급이 재미있지만, 반복적으로 행위를 해야 하므로 지루하게 느껴지는 순간이 있음
* 웨이브를 압축하게되면, 가스 이자, 택틱 수급 등의 기준이 바뀌면서 기존과 다르게 잘 워킹하지 않는 요소가 생기게 되므로 전체적으로 고려가 필요함.

# 개발 이슈

## ~~[보류]체력 통합 Test [영섭]~~

* 의도
  + 다수의 유닛이 개별 체력을 갖는 상황이 전투 상황을 정확하게 인지하기 힘들게 한다. 또한 체력이라는 스탯을 다룰 때, 지휘관 체력인지 유닛의 체력인지 유저가 이해하기 어려워지는 문제가 있다.
  + 부대 단위의 통합 체력 시스템을 도입하는 것이 좋다고 판단하지만, 구현 이슈 및 예상하지 못한 사이드 이펙트가 클 수 있으므로 기존의 기능을 활용해서 플레이 느낌을 테스트 해보도록 한다.
* 필요 기능
  + 유닛이 받은 피해를 지휘관에게 공유
* 이슈
  + 개별 콜라이더일 때, 의도하지 않게 여러 번의 피격을 받는 상황.

## [완료]몬스터 남기기

* 의도
  + 웨이브 방식의 룰에서 웨이브 시작 시에는 항상 몬스터가 소량 존재하고 있음.
  + 유저 성향에 따라서 이 구간이 전반적으로 루즈하다는 피드백이 다수 있음.
  + 여러 가지 솔루션으로 대응이 필요하며, 그 중에 쉽게 해볼 수 있는 것들을 우선 적용 해본다.
* 필요 기능
  + 웨이브 종료 시 잔여 몬스터를 kill 처리하지 않고, 남겨둔다.
* 이슈
  + 이어하기 시점에 대한 정보 저장 문제 정리 필요
    - 중간 저장 타이밍을 전투 시작 타이밍으로 제한.
    - 이어하기 시점에는 잔여 몬스터는 hp, 종류, 위치만 저장

## [완]택틱 구조 정리

* 의도
  + 유닛과 택틱이 하나의 덱빌딩 과정에서 의미있게 동작할 수 있도록 정리가 필요하다.
  + 개별 택틱의 개성 보다는 직관적으로 잘하고 있다라는 감각을 제공하는 것이 중요하고, 전설 유닛과 조합되는 택틱을 제공하여, 게임에서 전설 유닛을 소환하고 싶은 욕구를 강화하는 방향으로 동작하는 것을 목표로 한다.
  + 동작 방식 예시
    - 기본 택틱의 등급은 통일하고, 그룹 택틱 방식을 채택
    - 특정 그룹 택틱을 MAX로 획득 시 [전설 택틱] 발견
    - [전설 택틱]은 특정 [전설 유닛]과 같이 있을 때, 효과를 발현
* 필요 기능
  + [완]웨이브 타입에서 택틱 타입을 등급 무관하게 랜덤하게 나오는 타입 추가
    - Reward\_Tactic\_Random
      * 이 경우 등급 별로 선택 확률을 제어한다.
      * 등급 별 확률을 관리하는 데이터를 추가할지 택틱 별 개별 확률로 제어할지 논의 필요
        + StageModeTacticPoolData@select\_weight 컬럼 추가, 택틱 개별로 확률 제어.
  + [완]택틱 그룹 만렙 달성 시 보너스 선택 추가
    - 특정 택틱 그룹을 최대 레벨로 도달하면, 해당 택틱 그룹에 속한 Mythic 등급의 택틱을 보너스로 추가 지급한다.
      * StageModeTacticPoolData@max\_level 추가하여 택틱 레벨 최대 값 설정
      * 해당 그룹의 max\_level 이상의 Mythic 등급 카드를 보너스 카드로 처리
    - Mythic 카드 선택 UI 팝업
      * 조건에 맞는 택틱 중 3개를 골라서 보여준다.
      * 수량이 3개 이하이면, 해당 수량 만큼만 보여준다.
      * 제시된 택틱 중 하나를 선택해서 획득할 수 있다.
      * 만약, 조건에 맞는 택틱이 없다면, 보너스 선택 UI를 출력하지 않는다.

* + - Mythic 카드 리롤
      * 조건에 맞는 택틱이 3개를 초가하는 경우 리롤 UI를 활성화 한다.
      * 택틱 리롤 횟수를 동일하게 차감하여 Mythic 카드 3장을 다시 뽑는다.
      * Pool이 3개 이하인 경우 리롤이 의미 없으므로 이 경우에는 리롤 UI를 비활성화 처리한다.
  + [중]UI 개선 이슈
    - 택틱 그룹에 연결될 보너스 택틱이 있으면, 택틱 선택지 UI에서 알 수 있도록 툴팁 형태의 UI 연결 필요
      * Mythic 카드 보유한 택틱 드룹이면, [미리보기] 버튼 배치
